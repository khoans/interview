0. Given the following table containing employees, some of which are managers
```sql
TABLE employee
    id INTEGER NOT NULL PRIMARY KEY
    managerId INTEGER REFERENCES employee(id)
    name VARCHAR(255) NOT NULL
```
which quer(ies) return the list of employees who are not managers?

Practice:














Option 1: Using NOT IN
```SQL
SELECT * FROM employee
WHERE id NOT IN (SELECT managerId FROM employee WHERE managerId IS NOT NULL)
```
=> An employee is not a manager if their `id` never appears in the `managerId` column

Option 2: using NOT EXISTS
```SQL
SELECT * FROM employee e
WHERE NOT EXISTS (SELECT 1 FROM employee m WHERE m.managerId = e.id)
```
=> An employee is not a manager if there does not exist any employee whose `managerId` is equal to that employee's `id`

Option 3: using LEFT JOIN
```SQL
SELECT e.* FROM employee e
LEFT JOIN employee m ON e.id = m.managerId
WHERE m.managerId IS NULL
```
=> An employee is not a manager if there is no match in the join, which means that the `managerId` in the joined table is NULL
