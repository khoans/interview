# BPMN là gì? Temporal có hỗ trợ không?

## 1. BPMN là gì?

### Định nghĩa

**BPMN** (Business Process Model and Notation) là một chuẩn **ký hiệu đồ họa** để mô hình hóa các quy trình nghiệp vụ (business processes).

- **Chuẩn**: Được quản lý bởi **OMG** (Object Management Group)
- **Phiên bản hiện tại**: BPMN 2.0 (ISO 19510)
- **Mục đích**: Cung cấp ngôn ngữ chung giữa business analysts và developers

### 4 thành phần cơ bản của BPMN

```
┌─────────────────────────────────────────────────────────────┐
│                    BPMN 2.0 Elements                         │
├─────────────────────────────────────────────────────────────┤
│  1. Flow Objects      → Events, Activities, Gateways        │
│  2. Connecting Objects → Sequence Flow, Message Flow        │
│  3. Swimlanes         → Pools, Lanes                        │
│  4. Artifacts         → Data Objects, Groups, Annotations   │
└─────────────────────────────────────────────────────────────┘
```

### Các ký hiệu cơ bản

```
┌─────────────────────────────────────────────────────────────┐
│                    Flow Objects                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Events (Sự kiện) - Hình tròn:                               │
│  ○  Start Event      → Bắt đầu process                       │
│  ○  Intermediate     → Xảy ra giữa process                   │
│  ○  End Event        → Kết thúc process                      │
│                                                              │
│  Activities (Hoạt động) - Hình chữ nhật bo tròn:             │
│  ┌──────────┐                                                │
│  │  Task    │  → Công việc cần thực hiện                     │
│  └──────────┘                                                │
│  ┌──────────┐                                                │
│  │ Sub-Proc │  → Process con (nested)                        │
│  └──────────┘                                                │
│                                                              │
│  Gateways (Cổng quyết định) - Hình thoi:                     │
│     ◇  Exclusive Gateway (XOR) → Chọn 1 nhánh               │
│     ◇  Parallel Gateway (AND)  → Tất cả nhánh               │
│     ◇  Inclusive Gateway (OR)  → Một hoặc nhiều nhánh       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Ví dụ BPMN trực quan

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         Loan Application Process                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   ○ ──→ ┌──────────────┐ ──→ ◇ ──→ ┌──────────────┐ ──→ ○              │
│  Start  │  Validate    │     │     │  Approve     │    End             │
│         │  Application │     │     │  Loan        │    (Approved)      │
│         └──────────────┘     │     └──────────────┘                     │
│                              │                                           │
│                              │ No                                        │
│                         Credit │                                         │
│                         Check  │   ┌──────────────┐                     │
│                              └──▶│   Reject     │──→ ○                 │
│                                  │  Application │   End                │
│                                  └──────────────┘   (Rejected)          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Các loại Events trong BPMN

| Loại | Ký hiệu | Mô tả |
|------|---------|-------|
| **Start Event** | ○ | Bắt đầu process |
| **End Event** | ● | Kết thúc process |
| **Intermediate Event** | ○ (double border) | Xảy ra giữa process |
| **Timer Event** | ○ 🕐 | Kích hoạt theo thời gian |
| **Message Event** | ○ ✉ | Gửi/nhận message |
| **Error Event** | ○ ⚡ | Xảy ra lỗi |
| **Signal Event** | ○ 📡 | Broadcast signal |

## 2. Temporal có hỗ trợ BPMN không?

### Câu trả lời ngắn gọn:

**❌ KHÔNG trực tiếp**, nhưng **✅ CÓ THỂ** gián tiếp thông qua các giải pháp bên thứ ba.

### Giải thích chi tiết:

### a. Temporal native support

Temporal **không hỗ trợ BPMN native** vì:

| Lý do | Giải thích |
|-------|-----------|
| **Code-first approach** | Temporal ưu tiên viết workflow bằng code (Go, Java, Python, TypeScript, PHP) |
| **Dynamic workflows** | Code linh hoạt hơn so với BPMN static |
| **Versioning** | Code versioning dễ hơn BPMN XML versioning |
| **Testing** | Unit test code dễ hơn test BPMN |

### b. Có cách nào dùng BPMN với Temporal không?

**✅ CÓ!** Một số giải pháp:

#### Option 1: BPMN → Temporal Code Generator

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   BPMN XML   │────▶│   Code       │────▶│   Temporal   │
│   (.bpmn)    │     │   Generator  │     │   Workflow   │
└──────────────┘     └──────────────┘     └──────────────┘
```

**Công cụ:**
- [Camunda](https://camunda.com/) + Custom exporter
- [Eclipse BPMN Modeler](https://www.eclipse.org/bpmn2-modeler/) + Template engine
- Tự build parser BPMN XML → Temporal SDK code

#### Option 2: BPMN Engine + Temporal Integration

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   BPMN       │     │   Temporal   │     │   Workers    │
│   Engine     │────▶│   Activities │◀────│   (Code)     │
│   (Camunda,  │     │              │     │              │
│    Activiti) │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
```

**Cách hoạt động:**
- BPMN Engine chạy BPMN workflows
- Khi cần thực thi activity, gọi đến Temporal Workers
- Temporal lo phần distributed execution, retry, durability

#### Option 3: Sử dụng nền tảng trung gian

Một số nền tảng cung cấp BPMN-to-Workflow conversion:

| Nền tảng | Mô tả |
|----------|-------|
| **Camunda 8** | Sử dụng Zeebe engine (tương tự Temporal), hỗ trợ BPMN |
| **Orkes** | Hỗ trợ cả BPMN và workflow engine |
| **Conductor** | JSON-based nhưng có BPMN converters |

## 3. So sánh: BPMN vs Temporal Workflow

### a. Cách định nghĩa workflow

**BPMN (XML):**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<process id="loanApplication" name="Loan Application">
  
  <startEvent id="start"/>
  
  <sequenceFlow id="flow1" sourceRef="start" targetRef="validateTask"/>
  
  <serviceTask id="validateTask" name="Validate Application">
    <extensionElements>
      <temporal:activity>validateApplication</temporal:activity>
    </extensionElements>
  </serviceTask>
  
  <sequenceFlow id="flow2" sourceRef="validateTask" targetRef="creditGateway"/>
  
  <exclusiveGateway id="creditGateway"/>
  
  <sequenceFlow id="flow3" sourceRef="creditGateway" targetRef="approveTask">
    <conditionExpression>${creditScore >= 700}</conditionExpression>
  </sequenceFlow>
  
  <sequenceFlow id="flow4" sourceRef="creditGateway" targetRef="rejectTask">
    <conditionExpression>${creditScore < 700}</conditionExpression>
  </sequenceFlow>
  
  <serviceTask id="approveTask" name="Approve Loan"/>
  <serviceTask id="rejectTask" name="Reject Loan"/>
  
  <endEvent id="end1"/>
  <endEvent id="end2"/>
  
</process>
```

**Temporal (TypeScript):**

```typescript
import { proxyActivities, sleep } from '@temporalio/workflow';
import type * as activities from './activities';

const { validateApplication, checkCreditScore, approveLoan, rejectLoan } = 
  proxyActivities<typeof activities>({
    startToCloseTimeout: '1 minute',
  });

export async function loanApplicationWorkflow(applicationId: string) {
  // Step 1: Validate
  const validation = await validateApplication(applicationId);
  
  // Step 2: Check credit score
  const creditScore = await checkCreditScore(applicationId);
  
  // Step 3: Decision (Exclusive Gateway)
  if (creditScore >= 700) {
    await approveLoan(applicationId);
    return { status: 'approved' };
  } else {
    await rejectLoan(applicationId);
    return { status: 'rejected' };
  }
}
```

### b. Bảng so sánh

| Tiêu chí | BPMN | Temporal Native |
|----------|------|-----------------|
| **Định nghĩa** | XML (visual diagram) | Code (TypeScript, Go, Java, Python, PHP) |
| **Học curve** | Thấp (kéo thả visual) | Trung bình (cần biết code) |
| **Flexibility** | Hạn chế (phải theo chuẩn) | Cao (code làm gì cũng được) |
| **Versioning** | XML versioning | Code versioning (Git) |
| **Testing** | Integration test | Unit test + Integration test |
| **Debugging** | BPMN viewer | Debugger, Web UI, replay |
| **Collaboration** | ✅ Business + Dev cùng hiểu | ❌ Chỉ developers |
| **Tooling** | Camunda Modeler, Eclipse | VS Code, IDEs |

## 4. Khi nào dùng BPMN? Khi nào dùng Temporal native?

### Dùng BPMN khi:

```
✅ Phù hợp:
├── Cần sự hợp tác giữa business và technical teams
├── Quy trình nghiệp vụ phức tạp, nhiều nhánh
├── Cần visual documentation rõ ràng
├── Compliance yêu cầu process documentation
├── Low-code/No-code preference
└── Process hay thay đổi bởi business users

Ví dụ:
├── Loan approval process
├── Insurance claim processing
├── Order fulfillment workflow
├── HR onboarding process
└── Customer complaint handling
```

### Dùng Temporal native khi:

```
✅ Phù hợp:
├── Team có developers mạnh
├── Cần flexibility cao trong logic
├── Workflow có nhiều tính toán phức tạp
├── Cần unit test coverage cao
├── Prefer code-over-config
└── Microservices orchestration

Ví dụ:
├── Data pipeline orchestration
├── Microservices coordination
├── Event-driven architectures
├── Distributed transaction management
└── Real-time processing workflows
```

## 6. Giải pháp thay thế BPMN cho Temporal

Nếu bạn muốn visual workflow nhưng dùng Temporal:

### a. Temporal Web UI

```
Temporal cung cấp Web UI để:
├── Xem workflow executions
├── Xem history events
├── Debug và replay
└── Xem workflow structure (dạng tree)
```

### b. Workflow Visualization Tools

| Tool | Mô tả |
|------|-------|
| **Temporal UI** | Built-in, xem workflow execution |
| **Phlare** | Profiling và visualization |
| **Custom Dashboard** | Tự build từ Temporal APIs |

### c. DSL-based (Domain Specific Language)

Thay vì BPMN, dùng YAML/JSON DSL:

```yaml
# temporal-workflow.yaml
name: order-fulfillment
version: "1.0"

tasks:
  - id: validate-order
    type: activity
    name: validateOrder
    timeout: 30s
    retry:
      max: 3
      
  - id: charge-payment
    type: activity
    name: chargePayment
    depends_on: validate-order
    
  - id: ship-order
    type: activity
    name: shipOrder
    depends_on: charge-payment
```
