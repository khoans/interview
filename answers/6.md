# Tích hợp Third-Party trong BPMN

## 1. Tổng quan

Khi sử dụng BPMN, có nhiều cách để tích hợp với các hệ thống third-party (ngân hàng, payment gateway, credit bureau, email service, etc.)

```
┌─────────────────────────────────────────────────────────────────┐
│                    BPMN Process                                  │
│                                                                  │
│   ○ ──→ ┌──────────────┐ ──→ ┌──────────────┐ ──→ ┌──────────┐ │
│  Start  │   Service    │     │   Service    │     │  Third   │ │
│         │   Task 1     │     │   Task 2     │     │  Party   │ │
│         └──────────────┘     └──────────────┘     └──────────┘ │
│                                                      API       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │  Third Party    │
                    │  - Payment      │
                    │  - Email        │
                    │  - SMS          │
                    │  - Credit Score │
                    │  - Bank API     │
                    └─────────────────┘
```

## 2. Các cách tích hợp Third-Party trong BPMN

### Phương pháp 1: Service Task với Java Delegate (Camunda)

**Đây là cách phổ biến nhất trong Camunda BPMN.**

### a. BPMN XML

```xml
<?xml version="1.0" encoding="UTF-8"?>
<bpmn:definitions xmlns:bpmn="http://www.omg.org/spec/BPMN/20100524/MODEL"
                  xmlns:camunda="http://camunda.org/schema/1.0/bpmn">

  <bpmn:process id="paymentProcess" isExecutable="true">
    
    <bpmn:startEvent id="start"/>
    
    <bpmn:sequenceFlow sourceRef="start" targetRef="paymentTask"/>
    
    <!-- Service Task gọi Payment Gateway -->
    <bpmn:serviceTask id="paymentTask" name="Process Payment">
      <bpmn:extensionElements>
        <!-- Cách 1: Delegate Expression -->
        <camunda:delegateExpression>${paymentGatewayDelegate}</camunda:delegateExpression>
        
        <!-- Cách 2: Class Expression -->
        <!-- <camunda:class>com.example.PaymentGatewayDelegate</camunda:class> -->
        
        <!-- Cách 3: External Task -->
        <!-- <camunda:externalTaskId>process-payment</camunda:externalTaskId> -->
      </bpmn:extensionElements>
      <bpmn:incoming>flow1</bpmn:incoming>
      <bpmn:outgoing>flow2</bpmn:outgoing>
    </bpmn:serviceTask>
    
    <bpmn:sequenceFlow id="flow1" sourceRef="start" targetRef="paymentTask"/>
    <bpmn:sequenceFlow id="flow2" sourceRef="paymentTask" targetRef="end"/>
    
    <bpmn:endEvent id="end"/>
    
  </bpmn:process>
</bpmn:definitions>
```

### b. Java Delegate Implementation

```java
// PaymentGatewayDelegate.java
@Component
public class PaymentGatewayDelegate implements JavaDelegate {
    
    @Autowired
    private PaymentGatewayService paymentGatewayService;
    
    @Override
    public void execute(DelegateExecution execution) {
        // Lấy variables từ process
        String orderId = (String) execution.getVariable("orderId");
        BigDecimal amount = (BigDecimal) execution.getVariable("amount");
        String currency = (String) execution.getVariable("currency");
        String customerId = (String) execution.getVariable("customerId");
        
        try {
            // Gọi Third-Party Payment API (Stripe, PayPal, etc.)
            PaymentResponse response = paymentGatewayService.charge(
                customerId,
                amount,
                currency,
                orderId
            );
            
            // Set kết quả vào process variables
            execution.setVariable("paymentStatus", "SUCCESS");
            execution.setVariable("transactionId", response.getTransactionId());
            execution.setVariable("paymentTime", response.getTimestamp());
            
            System.out.println("Payment successful: " + response.getTransactionId());
            
        } catch (PaymentException e) {
            // Xử lý lỗi
            execution.setVariable("paymentStatus", "FAILED");
            execution.setVariable("errorMessage", e.getMessage());
            execution.setVariable("errorCode", e.getCode());
            
            // Ném exception để BPMN handle qua Error Event
            throw new BpmnError("PAYMENT_FAILED", e.getMessage());
        }
    }
}
```

### c. Third-Party Service Implementation

```java
// PaymentGatewayService.java
@Service
public class PaymentGatewayService {
    
    @Value("${stripe.api.key}")
    private String apiKey;
    
    @Value("${stripe.api.url}")
    private String apiUrl;
    
    private final RestTemplate restTemplate = new RestTemplate();
    
    public PaymentResponse charge(String customerId, BigDecimal amount, 
                                   String currency, String orderId) {
        
        // Prepare request
        Map<String, Object> request = new HashMap<>();
        request.put("customer_id", customerId);
        request.put("amount", amount.multiply(new BigDecimal("100"))); // cents
        request.put("currency", currency);
        request.put("order_id", orderId);
        request.put("idempotency_key", UUID.randomUUID().toString());
        
        // Headers
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + apiKey);
        headers.set("Content-Type", "application/json");
        headers.set("Idempotency-Key", request.get("idempotency_key").toString());
        
        HttpEntity<Map<String, Object>> entity = new HttpEntity<>(request, headers);
        
        // Call Stripe API
        ResponseEntity<Map> response = restTemplate.postForEntity(
            apiUrl + "/v1/charges",
            entity,
            Map.class
        );
        
        Map<String, Object> responseBody = response.getBody();
        
        // Parse response
        PaymentResponse paymentResponse = new PaymentResponse();
        paymentResponse.setTransactionId((String) responseBody.get("id"));
        paymentResponse.setStatus((String) responseBody.get("status"));
        paymentResponse.setTimestamp(Instant.now());
        
        return paymentResponse;
    }
}
```

## 3. Phương pháp 2: External Task Pattern

**Phù hợp khi third-party service chạy độc lập (microservices).**

### a. BPMN XML với External Task

```xml
<bpmn:serviceTask id="sendEmailTask" name="Send Email">
  <bpmn:extensionElements>
    <camunda:externalTaskId topic="send-email"/>
    <camunda:inputParameter name="email">${customerEmail}</camunda:inputParameter>
    <camunda:inputParameter name="subject">${emailSubject}</camunda:inputParameter>
    <camunda:inputParameter name="body">${emailBody}</camunda:inputParameter>
  </bpmn:extensionElements>
</bpmn:serviceTask>
```

### b. External Task Worker (Node.js)

```javascript
// email-worker.js
const { Client } = require('camunda-external-task-client-js');

const client = new Client({
  baseUrl: 'http://localhost:8080/engine-rest',
  use: logger: true,
  asyncResponseTimeout: 10000
});

// Subscribe to 'send-email' topic
client.subscribe('send-email', async ({ task, taskService }) => {
  const email = task.variables.get('email');
  const subject = task.variables.get('subject');
  const body = task.variables.get('body');
  
  try {
    // Gọi Third-Party Email Service (SendGrid, SES, etc.)
    await sendEmailViaSendGrid(email, subject, body);
    
    // Complete task với success
    await taskService.complete(task);
    console.log(`Email sent to ${email}`);
    
  } catch (error) {
    // Handle error với retry
    await taskService.handleFailure(task, {
      errorMessage: error.message,
      retries: 3,
      retryTimeout: 5000
    });
  }
});

async function sendEmailViaSendGrid(to, subject, body) {
  const sgMail = require('@sendgrid/mail');
  sgMail.setApiKey(process.env.SENDGRID_API_KEY);
  
  await sgMail.send({
    to: to,
    from: 'noreply@example.com',
    subject: subject,
    text: body,
    html: body
  });
}
```

### c. External Task Worker (Java)

```java
// SmsWorker.java
@Component
public class SmsWorker {
    
    @Autowired
    private ExternalTaskService externalTaskService;
    
    @PostConstruct
    public void startWorker() {
        // Subscribe to 'send-sms' topic
        externalTaskService.subscribe()
            .topic("send-sms", 10) // fetch 10 tasks
            .handler((externalTask, externalTaskService) -> {
                try {
                    String phoneNumber = externalTask.getVariable("phoneNumber");
                    String message = externalTask.getVariable("message");
                    
                    // Gọi Third-Party SMS Service (Twilio, etc.)
                    sendSmsViaTwilio(phoneNumber, message);
                    
                    // Complete task
                    externalTaskService.complete(externalTask);
                    
                } catch (Exception e) {
                    externalTaskService.handleFailure(
                        externalTask,
                        "SMS send failed: " + e.getMessage(),
                        3, // retries
                        5000 // retry timeout
                    );
                }
            })
            .open();
    }
    
    private void sendSmsViaTwilio(String phoneNumber, String message) {
        Twilio.init(ACCOUNT_SID, AUTH_TOKEN);
        
        Message.creator(
            new com.twilio.type.PhoneNumber(phoneNumber),
            new com.twilio.type.PhoneNumber("+1234567890"),
            message
        ).create();
    }
}
```

## 4. Phương pháp 3: Camunda Connectors

**Cách nhanh nhất - không cần code, chỉ cần configuration.**

### a. REST Connector (trong BPMN XML)

```xml
<bpmn:serviceTask id="callCreditBureau" name="Check Credit Score">
  <bpmn:extensionElements>
    <camunda:connector>
      <camunda:inputOutput>
        <!-- Input mapping -->
        <camunda:inputParameter name="url">
          https://api.creditbureau.com/v1/score
        </camunda:inputParameter>
        <camunda:inputParameter name="method">POST</camunda:inputParameter>
        <camunda:inputParameter name="headers">
          <camunda:map>
            <camunda:entry key="Authorization">Bearer ${creditBureauApiKey}</camunda:entry>
            <camunda:entry key="Content-Type">application/json</camunda:entry>
          </camunda:map>
        </camunda:inputParameter>
        <camunda:inputParameter name="payload">
          {
            "customer_id": "${customerId}",
            "national_id": "${nationalId}"
          }
        </camunda:inputParameter>
      </camunda:inputOutput>
      
      <!-- Connector type -->
      <camunda:connectorId>http-connector</camunda:connectorId>
    </camunda:connector>
  </bpmn:extensionElements>
</bpmn:serviceTask>
```

## 5. Phương pháp 4: Message Events cho Async Integration

**Dùng khi third-party gọi lại (callback/webhook).**

### a. BPMN với Message Intermediate Event

```xml
<bpmn:process id="paymentWithCallback" isExecutable="true">
  
  <bpmn:startEvent id="start"/>
  
  <!-- Gọi Payment Gateway -->
  <bpmn:serviceTask id="initiatePayment" name="Initiate Payment">
    <bpmn:extensionElements>
      <camunda:delegateExpression>${paymentInitDelegate}</camunda:delegateExpression>
    </bpmn:extensionElements>
  </bpmn:serviceTask>
  
  <!-- Wait for callback từ Payment Gateway -->
  <bpmn:intermediateCatchEvent id="waitForCallback" name="Wait for Payment Callback">
    <bpmn:messageEventDefinition messageRef="paymentCallbackMsg"/>
  </bpmn:intermediateCatchEvent>
  
  <!-- Timeout nếu không nhận callback -->
  <bpmn:boundaryEvent id="timeoutEvent" attachedToRef="waitForCallback">
    <bpmn:timerEventDefinition>
      <bpmn:timeDuration>PT30M</bpmn:timeDuration>
    </bpmn:timerEventDefinition>
  </bpmn:boundaryEvent>
  
  <bpmn:serviceTask id="processCallback" name="Process Callback"/>
  <bpmn:endEvent id="end"/>
  
  <bpmn:sequenceFlow sourceRef="start" targetRef="initiatePayment"/>
  <bpmn:sequenceFlow sourceRef="initiatePayment" targetRef="waitForCallback"/>
  <bpmn:sequenceFlow sourceRef="waitForCallback" targetRef="processCallback"/>
  <bpmn:sequenceFlow sourceRef="processCallback" targetRef="end"/>
  
  <!-- Timeout handling -->
  <bpmn:serviceTask id="checkPaymentStatus" name="Check Payment Status"/>
  <bpmn:sequenceFlow sourceRef="timeoutEvent" targetRef="checkPaymentStatus"/>
  
</bpmn:process>

<!-- Message Definition -->
<bpmn:message id="paymentCallbackMsg" name="paymentCallback"/>
```

### b. Webhook Controller nhận callback

```java
// PaymentWebhookController.java
@RestController
@RequestMapping("/webhooks/payment")
public class PaymentWebhookController {
    
    @Autowired
    private RuntimeService runtimeService;
    
    @Autowired
    private PaymentSignatureVerifier signatureVerifier;
    
    /**
     * Nhận webhook từ Payment Gateway (Stripe, PayPal, etc.)
     */
    @PostMapping("/callback")
    public ResponseEntity<?> handlePaymentCallback(
            @RequestBody PaymentWebhookPayload payload,
            @RequestHeader("X-Signature") String signature) {
        
        // Verify webhook signature
        if (!signatureVerifier.verify(payload, signature)) {
            return ResponseEntity.status(401).build();
        }
        
        // Correlate message với BPMN process đang wait
        runtimeService.correlateMessage("paymentCallback", 
            Variables.createVariables()
                .putValue("transactionId", payload.getTransactionId())
                .putValue("paymentStatus", payload.getStatus())
                .putValue("amount", payload.getAmount())
                .putValue("timestamp", payload.getTimestamp())
        );
        
        return ResponseEntity.ok().build();
    }
    
    /**
     * Correlate với specific process instance
     */
    @PostMapping("/callback/{processInstanceId}")
    public ResponseEntity<?> handlePaymentCallbackForInstance(
            @PathVariable String processInstanceId,
            @RequestBody PaymentWebhookPayload payload) {
        
        runtimeService.createMessageCorrelation("paymentCallback")
            .processInstanceById(processInstanceId)
            .setVariables(Variables.createVariables()
                .putValue("transactionId", payload.getTransactionId())
                .putValue("paymentStatus", payload.getStatus())
            )
            .correlate();
        
        return ResponseEntity.ok().build();
    }
}
```

## 6. Xử lý lỗi khi tích hợp Third-Party

### a. Error Boundary Event

```xml
<bpmn:serviceTask id="callThirdParty" name="Call External API">
  <bpmn:extensionElements>
    <camunda:delegateExpression>${thirdPartyDelegate}</camunda:delegateExpression>
  </bpmn:extensionElements>
  
  <!-- Error Boundary Event -->
  <bpmn:boundaryEvent id="errorBoundary" attachedToRef="callThirdParty">
    <bpmn:errorEventDefinition errorRef="thirdPartyError"/>
  </bpmn:boundaryEvent>
</bpmn:serviceTask>

<!-- Error Definition -->
<bpmn:error id="thirdPartyError" errorCode="THIRD_PARTY_ERROR" name="Third Party Error"/>

<!-- Error handling flow -->
<bpmn:serviceTask id="retryLogic" name="Retry Logic"/>
<bpmn:sequenceFlow sourceRef="errorBoundary" targetRef="retryLogic"/>
```

### b. Retry Configuration

```java
// Trong Java Delegate
@Component
public class ThirdPartyDelegate implements JavaDelegate {
    
    @Override
    public void execute(DelegateExecution execution) {
        try {
            // Call third-party API
            callExternalService();
            
        } catch (ConnectException e) {
            // Network error - retry
            throw new BpmnError("RETRYABLE_ERROR", "Connection failed", e);
            
        } catch (AuthenticationException e) {
            // Auth error - don't retry
            throw new BpmnError("AUTH_FAILED", "Authentication failed", e);
            
        } catch (BusinessException e) {
            // Business error - don't retry
            throw new BpmnError("BUSINESS_ERROR", e.getMessage(), e);
        }
    }
}
```

### c. Retry Configuration trong application.yml

```yaml
camunda:
  bpm:
    default-retry-time-to-live: PT1M  # Thời gian chờ giữa các retry
    job-execution:
      retries: 3  # Số lần retry mặc định
      
# Custom retry cho từng loại lỗi
third-party:
  retry:
    max-attempts: 3
    backoff:
      initial-interval: 1s
      max-interval: 30s
      multiplier: 2.0
```

## 8. Best Practices

### ✅ Nên làm:

```
1. Sử dụng External Task Pattern cho microservices
2. Implement retry logic với exponential backoff
3. Sử dụng Circuit Breaker pattern
4. Log đầy đủ requests/responses cho audit
5. Sử dụng Message Events cho async callbacks
6. Implement timeout handling
7. Sử dụng Connectors cho các dịch vụ phổ biến
8. Test với mock third-party services
```

### ❌ Tránh:

```
1. Blocking calls quá lâu trong Service Tasks
2. Không xử lý timeout
3. Không có retry mechanism
4. Hardcode API credentials trong BPMN XML
5. Không validate response từ third-party
6. Không handle partial failures
```

## 9. Kết luận

| Phương pháp | Khi nào dùng | Độ phức tạp |
|-------------|--------------|-------------|
| **Java Delegate** | Logic phức tạp, cần control toàn bộ | Trung bình |
| **External Task** | Microservices, độc lập deploy | Thấp |
| **Connectors** | Dịch vụ phổ biến (REST, Email, SMS) | Rất thấp |
| **Message Events** | Async callbacks, webhooks | Trung bình |

**Tóm lại:**
- BPMN hỗ trợ nhiều cách tích hợp third-party
- Chọn phương pháp phù hợp với kiến trúc hệ thống
- Luôn xử lý lỗi và retry properly
- Sử dụng Connectors khi có sẵn để tiết kiệm thời gian
