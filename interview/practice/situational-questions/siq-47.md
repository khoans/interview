**47.** How do you keep up with new technologies? What did you learn recently?

========== THIS SECTION IS THE ANSWER ==========

I learn driven by need, not hype. When a real problem requires a new tool, I go deep. Otherwise, I stay aware at a surface level through tech blogs and conference talks.

Recently, I learned AWS Bedrock for building the natural language search feature on our automotive marketplace. Users wanted to type things like "reliable SUV under 30k with good gas mileage" instead of using filters. I had to learn prompt engineering, Bedrock's API integration patterns, and how to map LLM output to structured Solr queries. Before that, I deepened my knowledge of Next.js when we needed server-side rendering for SEO optimization — our search results pages needed to rank well on Google, which required SSR with proper meta tags and structured data.

The trade-off: you can't learn everything. I prioritize technologies that solve a current or near-term problem for my team. I follow a few curated sources — the Spring blog, AWS What's New, and Martin Fowler's articles — rather than trying to track every new framework.
