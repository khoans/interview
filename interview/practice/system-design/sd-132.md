**132.** What is a bastion host (jump box)? Why do you use one instead of exposing servers directly?

========== THIS SECTION IS THE ANSWER ==========
A bastion host is a hardened server in a public subnet that serves as the single entry point for SSH/RDP access to private resources. Instead of exposing every server to the internet, you expose only the bastion — it acts as a gateway, reducing the attack surface to one tightly controlled machine with audit logging and strict access controls.

**Real example:** On our automotive marketplace, our ECS instances, RDS databases, and ElastiCache clusters were all in private subnets with no public IPs. When engineers needed to debug a production PostgreSQL issue or check Redis state, they SSH'd into the bastion host (in the public subnet, accessible only from our VPN IPs via Security Group rules), then connected to internal resources from there. The bastion logged all SSH sessions to CloudWatch for audit. We later replaced the bastion with AWS Systems Manager Session Manager, which provides shell access without opening any inbound ports and logs every command to CloudTrail.

**Trade-off:** A bastion is a single point of failure and a high-value target — if compromised, the attacker has access to the entire private network. Hardening is critical: disable password auth (SSH keys only), enable MFA, restrict to specific source IPs, patch aggressively, and log everything. Session Manager is the modern replacement — no inbound ports, IAM-based access control, automatic session logging — but requires the SSM agent on every target instance.
