**252.** What is traceroute/tracert? How do you diagnose network latency issues?

Traceroute maps the path packets take between your machine and a destination, showing each hop (router) and the round-trip time at each point. It works by sending packets with incrementing TTL values -- each router decrements TTL and responds when it hits zero.

On our marketplace, when users in certain US regions reported slow vehicle search responses, we used traceroute and New Relic distributed tracing to diagnose the issue. The problem turned out to be that some users were hitting the us-west-2 region when us-east-1 was closer. We fixed it by switching Route 53 from weighted routing to latency-based routing, which automatically directed users to the nearest region. We also used CloudFront which has edge locations across the US, reducing the first-hop latency significantly.

The trade-off with traceroute: many routers block ICMP packets (which traceroute uses), showing timeouts for those hops, making diagnosis incomplete. In cloud environments, you often rely more on application-level tracing (New Relic, X-Ray) than network-level traceroute since you do not control the underlying network infrastructure.
