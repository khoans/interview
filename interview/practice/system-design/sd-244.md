**244.** What is HTTP vs HTTPS? What is the default port for each?

HTTP (port 80) transmits data in plaintext; HTTPS (port 443) encrypts traffic using TLS (Transport Layer Security). HTTPS ensures confidentiality (data cannot be read in transit), integrity (data cannot be tampered with), and authentication (you are talking to the real server via its certificate).

On our marketplace serving 30M users/month, HTTPS was mandatory everywhere. CloudFront terminated TLS at the edge using ACM (AWS Certificate Manager) certificates, then forwarded traffic to our ALB over HTTPS. We enforced HSTS (HTTP Strict Transport Security) headers so browsers would never downgrade to HTTP. Our Spring Boot services communicated internally over HTTP within the VPC (ALB to ECS) since traffic was already inside a private network -- this reduced TLS overhead for internal calls.

The trade-off is TLS termination placement. Terminating at CloudFront edge gives users the fastest TLS handshake (geographic proximity), but means traffic between CloudFront and origin could theoretically be intercepted. We used HTTPS end-to-end to the ALB for defense-in-depth, accepting the small latency cost.
