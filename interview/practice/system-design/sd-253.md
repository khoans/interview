**253.** What is a proxy server? Explain forward proxy, reverse proxy, and transparent proxy.

A **forward proxy** sits between clients and the internet, acting on behalf of clients (e.g., corporate proxy filtering outbound traffic). A **reverse proxy** sits in front of servers, acting on behalf of servers (e.g., load balancer, CDN). A **transparent proxy** intercepts traffic without client configuration (e.g., ISP caching proxy).

On our marketplace, we used multiple reverse proxies in layers: CloudFront was the outermost reverse proxy (CDN, TLS termination, caching static assets like vehicle images), then the ALB was a reverse proxy for our ECS services (load balancing, health checks, SSL termination). Clients never communicated directly with our Spring Boot applications. For outbound traffic from our ECS tasks (calling third-party APIs for vehicle valuation data), we routed through a NAT Gateway which acted like a forward proxy, giving all outbound traffic a consistent IP for whitelisting.

The trade-off: each proxy layer adds latency (typically 1-5ms per hop) but provides security benefits (hiding backend topology, filtering requests) and operational benefits (caching, load balancing). Our three-layer setup (CloudFront -> ALB -> ECS) added about 10ms total but handled 30M users/month seamlessly.
