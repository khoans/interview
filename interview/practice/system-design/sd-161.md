**161.** What is Kubernetes? Explain its architecture (control plane, nodes, kubelet, etcd).

========== THIS SECTION IS THE ANSWER ==========

Kubernetes is a container orchestration platform that automates deployment, scaling, and management of containerized applications. The architecture has two parts: the control plane (brain) and worker nodes (muscle). The control plane runs the API server (entry point for all commands), etcd (distributed key-value store holding all cluster state), the scheduler (assigns pods to nodes), and controller managers (maintain desired state). Each worker node runs kubelet (agent ensuring containers run as specified), kube-proxy (network routing), and the container runtime (Docker/containerd).

In our marketplace, we ran ECS rather than Kubernetes, but the concepts map directly -- ECS has a control plane managing task placement across EC2 instances. When I evaluated Kubernetes for our team, I set up an EKS cluster in staging. The etcd-backed state store was critical -- when a node died, the controller manager detected it and rescheduled pods to healthy nodes automatically. The API server was the single entry point, so we secured it with RBAC and private endpoint access within our VPC.

The trade-off: Kubernetes gives you powerful orchestration but adds operational complexity. For our team of 8 developers, ECS was simpler to manage with AWS handling the control plane. Kubernetes makes more sense when you need multi-cloud portability or advanced scheduling features that ECS lacks.
