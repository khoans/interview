**220.** What is AWS Inspector? How does it scan for vulnerabilities in EC2 and container images?

========== THIS SECTION IS THE ANSWER ==========

AWS Inspector is an automated vulnerability scanning service that continuously assesses EC2 instances, Lambda functions, and container images in ECR for software vulnerabilities (CVEs) and network exposure. It uses the SSM agent on EC2 to inventory installed packages and compares them against the NVD (National Vulnerability Database). For containers, it scans images pushed to ECR and running in ECS/EKS.

In our marketplace: (1) ECR image scanning -- every time our GitHub Actions pipeline pushed a new Docker image, Inspector scanned it and reported CVEs by severity. Critical findings blocked the deployment pipeline. (2) EC2 scanning -- Inspector detected an outdated OpenSSL version on our EC2 instances that was vulnerable to a known exploit; we patched it within 24 hours. (3) Findings were exported to Security Hub for centralized dashboard. We configured EventBridge rules to send critical findings to Slack and create Jira tickets automatically. Inspector also detected network reachability issues -- an EC2 instance with a security group allowing port 22 from the internet was flagged.

The trade-off: Inspector scanning is continuous and mostly accurate, but it only detects known CVEs -- zero-days are not covered. It also adds cost based on scan volume. Combine Inspector with SAST in your CI/CD pipeline for code-level vulnerabilities, and use GuardDuty for runtime threat detection. Inspector is one piece of a layered vulnerability management strategy.
