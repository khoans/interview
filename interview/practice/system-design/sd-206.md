**206.** What is AWS CloudTrail? How does it help with security auditing?

========== THIS SECTION IS THE ANSWER ==========

CloudTrail records every API call made in your AWS account -- who made the call, from which IP, at what time, on which resource, and the result. It is the audit log for all AWS activity. Events are delivered to S3 and optionally to CloudWatch Logs for real-time alerting. You can create trails that capture management events (control plane operations) and data events (S3 object access, Lambda invocations).

In our marketplace, CloudTrail was essential for security and compliance: (1) Incident investigation -- when an unauthorized IAM policy change was detected, we traced it to a specific user, IP address, and timestamp within minutes. (2) Compliance auditing -- we proved to auditors that production database access was limited to specific roles at specific times. (3) Real-time alerts -- CloudTrail events flowed to CloudWatch Logs, where metric filters triggered alarms on suspicious activity: root account login, IAM policy changes, security group modifications, and S3 bucket policy changes. (4) We enabled CloudTrail in all regions and all accounts via Organizations, ensuring no blind spots.

The trade-off: CloudTrail data events (S3 object-level logging, Lambda invocations) generate massive volume and cost. We enabled data events only on sensitive buckets (customer data, secrets) and used S3 lifecycle policies to archive old trail logs to Glacier after 90 days. Management events are free for the first trail per account.
