**251.** What is DNS round-robin? How is it used for basic load balancing?

DNS round-robin distributes traffic by returning multiple IP addresses for a single domain name, cycling through them in order. Each client gets a different IP first in the list, spreading requests across servers.

On our marketplace, we did not rely on DNS round-robin alone -- we used Route 53 with weighted routing and health checks pointing to ALBs in us-east-1 and us-west-2. Route 53 is smarter than basic round-robin because it can factor in health status (removing unhealthy targets), geographic proximity (latency-based routing), and weights (sending 80% to primary region). Basic DNS round-robin has no health awareness -- if one server dies, DNS still sends traffic to it until the TTL expires.

The trade-off: DNS round-robin is free and simple but offers no health checking, no session persistence, and relies on TTL-based propagation which can take minutes. For a 30M user/month platform, that delay during an outage means thousands of failed requests. Real load balancing needs ALB/NLB with active health checks, not just DNS.
