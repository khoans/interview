**158.** What is a WAF (Web Application Firewall)? What rules would you configure?

========== THIS SECTION IS THE ANSWER ==========

A WAF inspects HTTP/HTTPS requests at the application layer (Layer 7) and blocks malicious traffic based on configurable rules. It sits in front of your application and can block SQL injection, XSS, bad bots, and other OWASP Top 10 attacks before they reach your code.

In our marketplace, we used AWS WAF attached to our CloudFront distribution and ALB. Our rule configuration: (1) AWS Managed Rules -- Core Rule Set for OWASP Top 10, SQL injection rule group, and Known Bad Inputs. (2) Rate limiting -- max 2000 requests per 5 minutes per IP to prevent scraping of vehicle listings. (3) Geo-blocking -- restricted to US since the marketplace only served US customers. (4) Custom rules -- blocked requests with suspiciously long query strings (over 2048 chars) and requests missing required headers. (5) Bot control -- allowed legitimate bots (Googlebot) but blocked scraper patterns stealing our listing data.

The lesson: start with AWS Managed Rules (they catch 80% of attacks), then add custom rules based on your traffic patterns. We initially set rate limits too low, which blocked legitimate dealer bulk uploads -- tune rules based on real traffic, not theory.
