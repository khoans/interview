**354.** How did you protect your application against the OWASP Top 10 vulnerabilities? Which ones were most relevant to your project?

========== THIS SECTION IS THE ANSWER ==========

The three most relevant to our marketplace were injection, broken access control, and security misconfiguration. We addressed each with specific technical controls.

**Injection (A03)**: All database queries used Spring Data JPA with parameterized queries — zero string concatenation. SonarCloud flagged any raw SQL concatenation in PRs. AWS WAF rules blocked common SQL injection patterns at the edge. **Broken Access Control (A01)**: Spring Security `@PreAuthorize` annotations on every endpoint enforced role-based access. Dealers could only modify their own listings — we validated ownership at the service layer, not just the URL. Horizontal privilege escalation was tested in our integration tests. **Security Misconfiguration (A05)**: We hardened Spring Boot defaults — disabled actuator endpoints in production, removed default error pages that leaked stack traces, enforced HTTPS with HSTS headers, set secure cookie flags, and configured CORS to allow only our domains. **XSS (A07)**: Input sanitization on all user-submitted content (listing descriptions) and Content-Security-Policy headers. **Sensitive Data Exposure (A02)**: PII encrypted at rest via KMS, never logged, and masked in API responses.

Lesson learned: OWASP protections are not one-time — SonarCloud in CI caught regressions continuously, which was more valuable than a one-time security audit.
