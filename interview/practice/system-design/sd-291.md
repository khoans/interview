**291.** Your load balancer logs show a massive spike from a single IP. What do you do?

========== THIS SECTION IS THE ANSWER ==========

First, verify whether it is legitimate or an attack by checking request patterns in the logs. Uniform intervals hitting one endpoint suggests a bot or DDoS; diverse endpoints might be a power user.

On our automotive marketplace, we spotted a single IP making 50K requests/hour to our vehicle search API. We analyzed CloudFront and ALB access logs in Kibana, confirmed it was a scraper harvesting listing data. We immediately added a WAF rate-limiting rule on AWS WAF attached to CloudFront (1,000 requests/minute per IP), which blocked the traffic within minutes. We also set up an SNS alert for future spikes exceeding threshold.

The trade-off: aggressive rate limiting can block legitimate users behind corporate NATs sharing one IP. We used tiered limits — stricter for unauthenticated requests, lenient for authenticated dealer accounts — so real high-volume users were never affected.
