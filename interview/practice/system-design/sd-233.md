**233.** What is Docker content trust? How does image signing work?

Docker Content Trust (DCT) uses digital signatures to verify the integrity and publisher of Docker images. When enabled, Docker only pulls images that have been signed by trusted publishers, preventing tampered or malicious images from running.

On our platform, we pushed images to Amazon ECR which supports image scanning and immutable tags. Image signing works via The Update Framework (TUF): a publisher signs the image digest with their private key, the signature is stored in a Notary server, and clients verify the signature before pulling. AWS also offers ECR image signing with AWS Signer. We enforced immutable tags in ECR so that a v1.2.3 tag could not be overwritten with a different image.

The trade-off is operational complexity. Key management for signing adds overhead -- you need to securely store signing keys, handle key rotation, and manage trust delegation when multiple developers push images. For us, immutable tags plus ECR scanning was the pragmatic middle ground.
