**363.** What is an Nginx server block (virtual host)? How do you configure multiple domains on a single Nginx instance?

========== THIS SECTION IS THE ANSWER ==========

A server block is Nginx's way of hosting multiple domains on a single instance — equivalent to Apache's VirtualHost. Each server block defines a `server_name` (domain), `listen` port, and its own configuration (root directory, proxy rules, SSL certificates). Nginx matches incoming requests to the correct server block based on the Host header.

In our Korean ecommerce project, we had two server blocks on the same Nginx instance: one for the main storefront (`shop.example.kr`) proxying to the Next.js app on port 3000, and one for the admin panel (`admin.example.kr`) proxying to the same Next.js app but with different caching and security headers. Each had its own SSL certificate and access log file. The config looked like: `server { server_name shop.example.kr; location / { proxy_pass http://localhost:3000; } }` and a similar block for admin.

The trade-off: server blocks are simple and efficient for a handful of domains. But if you're managing 50+ domains with different configurations, maintaining Nginx config files becomes unwieldy — at that point, tools like Akamai or an API gateway with dynamic routing are more manageable. For our small project, two server blocks were perfect.
