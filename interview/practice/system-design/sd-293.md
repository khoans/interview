**293.** You need to pass a SOC2 audit. What are the key controls you need to demonstrate?

========== THIS SECTION IS THE ANSWER ==========

SOC2 focuses on five trust service criteria: security, availability, processing integrity, confidentiality, and privacy. The key controls revolve around access management, change management, monitoring, encryption, and incident response.

On our marketplace handling 30M users' data, the controls we demonstrated included: IAM role-based access with MFA for all AWS accounts, Secrets Manager for credential rotation, encryption at rest (RDS, S3, DynamoDB using KMS) and in transit (TLS everywhere), GitHub Actions CI/CD with required PR reviews and SonarCloud quality gating, New Relic and CloudWatch monitoring with alerting, CloudTrail audit trails for all API calls, and a documented incident response playbook.

The trade-off: SOC2 compliance adds significant operational overhead — quarterly access reviews, continuous documentation, and restricted developer workflows. But for an enterprise platform handling consumer and dealer PII, it was non-negotiable for closing B2B partnerships.
