**298.** A developer asks to disable HTTPS for local testing. What is the safe way to handle this?

========== THIS SECTION IS THE ANSWER ==========

Never disable HTTPS in shared or deployed environments. For local development, use Spring Boot profiles that allow HTTP only in the local profile while enforcing HTTPS everywhere else.

On our marketplace, we used Spring profiles: the `local` profile ran HTTP on localhost with a self-signed cert option, but `dev`, `staging`, and `prod` profiles enforced HTTPS with HSTS headers. The key was `application.yml` had TLS enabled by default — only `application-local.yml` overrode it. Our GitHub Actions CI pipeline validated that no PR could merge code disabling TLS in non-local profiles, and SonarCloud flagged any security configuration changes for mandatory review.

Lesson learned: the moment you allow HTTP "just for testing," someone eventually forgets to re-enable HTTPS and it reaches staging. Environment-specific profiles with CI enforcement make this impossible. Five minutes setting up mkcert for local certs prevents days of recovering from a credentials-leaked-over-HTTP incident.
