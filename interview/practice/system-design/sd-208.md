**208.** What is AWS WAF? How do you write rules to block common attacks?

========== THIS SECTION IS THE ANSWER ==========

AWS WAF is a web application firewall that filters HTTP/HTTPS requests at the edge (CloudFront, ALB, API Gateway). Rules inspect request components (URI, headers, body, query strings, IP) and take actions (allow, block, count, CAPTCHA). Rules are organized in Web ACLs with priority-ordered evaluation.

In our marketplace, our WAF Web ACL on CloudFront had: (1) AWS Managed Rule Groups -- AWSManagedRulesCommonRuleSet blocked SQL injection and XSS patterns in query strings and body. AWSManagedRulesKnownBadInputsRuleSet blocked Log4Shell and other known exploits. (2) IP rate-based rule -- 2000 requests per 5 minutes per IP; exceeding IPs were blocked for 30 minutes, preventing listing scraping. (3) Custom regex rule -- blocked requests where the User-Agent matched known scraper patterns targeting our vehicle data. (4) Geo-match rule -- only allowed US traffic. (5) Size constraint -- blocked request bodies over 8KB (our largest legitimate payload was a listing form at ~4KB). Rules were evaluated in priority order; first match wins.

The lesson: deploy rules in Count mode first to analyze matched requests without blocking -- we caught false positives where our own mobile app triggered a SQL injection rule because a VIN number contained characters that looked like SQL syntax. Then switch to Block after tuning.
