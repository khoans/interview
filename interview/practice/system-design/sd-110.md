**110.** What is the difference between encryption, encoding, and hashing?

========== THIS SECTION IS THE ANSWER ==========
Encryption transforms data using a key for confidentiality — reversible only with the key. Encoding transforms data for compatibility/transport (Base64, URL encoding) — reversible by anyone, no key needed, provides zero security. Hashing produces a fixed-size digest for integrity verification — one-way, irreversible by design.

**Real example:** On our automotive marketplace, we used all three distinctly. Encoding: Base64-encoded JWT payload for safe HTTP header transport — anyone can decode it, it is not secret. Encryption: AES-256 via KMS encrypted dealer bank account numbers in DynamoDB — only services with KMS decrypt permission could read them. Hashing: bcrypt hashed user passwords in PostgreSQL — we never needed the original password, just verification that the login attempt matches. A common security mistake I have seen in code reviews: developers Base64-encoding sensitive data and treating it as "encrypted."

**Trade-off:** The key distinction is intent — encoding is for data format compatibility, encryption is for confidentiality, hashing is for integrity/verification. Using the wrong one creates false security. Encoding is fastest (no cryptographic computation), hashing is one-way but fast (except intentionally slow password hashes), and encryption is the most expensive due to key management overhead.
