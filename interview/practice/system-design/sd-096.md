**96.** Design a news feed (like Facebook/Twitter). How do you generate and rank the feed?

========== THIS SECTION IS THE ANSWER ==========
Two main approaches: fan-out-on-write (pre-compute feed at publish time) for low-follower users, and fan-out-on-read (assemble at read time) for high-follower accounts. Ranking uses a scoring function combining recency, engagement signals, and user affinity, applied before serving.

**Real example:** On our automotive marketplace, we built a "recommended listings" feed for buyers. At write time (new listing published), we pushed the listing ID into saved-search result queues in Redis sorted sets, scored by relevance (price match, location proximity, recency). At read time, the feed service merged the user's sorted sets, applied a ranking boost for listings with more photos and dealer ratings, and returned the top 20. Redis sorted sets gave us O(log N) insertion and O(K) retrieval for top-K items.

**Trade-off:** Pre-computing feeds consumes significant write-time resources and storage (each listing pushed to thousands of user feeds). We limited fan-out to users who logged in within the last 30 days — inactive user feeds were computed on-demand at read time. This hybrid approach saved ~60% of Redis memory while keeping active users' feeds fast (under 50ms).
