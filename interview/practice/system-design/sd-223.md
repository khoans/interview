**223.** What is a CI/CD security pipeline? What security gates should each stage have?

A CI/CD security pipeline embeds automated security checks at each stage of code delivery: commit, build, test, deploy, and runtime.

In our GitHub Actions pipeline for the automotive marketplace: at **commit** stage, we had pre-commit hooks for secret detection; at **build**, SonarCloud ran SAST analysis and Dependabot checked dependencies; at **test**, integration tests verified Spring Security rules and OAuth2 flows; at **deploy**, Docker images were scanned before pushing to ECR, and infrastructure templates were validated; at **runtime**, New Relic and Kibana monitored for anomalous patterns. Any critical SonarCloud finding blocked the PR merge.

The trade-off is balancing strictness with velocity. We configured SonarCloud quality gates to block on critical/blocker issues only -- warnings were tracked but did not stop deploys. Too strict and developers route around the system; too loose and it becomes security theater.
