**235.** What is compliance as code? How do you automate SOC2/PCI-DSS/HIPAA checks?

Compliance as code translates regulatory requirements into automated policies that run in CI/CD pipelines, continuously verifying that infrastructure and application configurations meet compliance standards.

On our automotive marketplace handling financial transactions (vehicle pricing, dealer payments), PCI-DSS compliance was relevant. We automated checks like: encryption at rest for PostgreSQL and DynamoDB (AWS KMS), encryption in transit (TLS everywhere via CloudFront and ALB), access logging enabled on S3 buckets, and IAM policies following least privilege. Tools like AWS Config Rules, Open Policy Agent (OPA), or Chef InSpec can codify these checks -- for example, an OPA policy that rejects any Terraform plan creating an unencrypted RDS instance.

The trade-off is that compliance frameworks are written in legal language, not code. Translating "adequate access controls" into a specific OPA rule requires interpretation. You need security and compliance teams to agree on what each rule means technically, which is an ongoing negotiation.
