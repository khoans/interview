**263.** What is a session fixation attack? How do you prevent it?

Session fixation is an attack where the attacker sets a known session ID on the victim browser before authentication. Once the victim logs in, the attacker uses the pre-set session ID to hijack the authenticated session. The key is that the session ID does not change after login.

On our marketplace, Spring Security mitigated this by default with its session fixation protection strategy: upon successful authentication, it creates a new session (new ID) and migrates attributes from the old session, invalidating the original. This means even if an attacker planted a session ID via a crafted URL or XSS, the ID becomes useless after login. We verified this protection was enabled in our security configuration and tested it during penetration testing.

The lesson: session fixation is a surprisingly common vulnerability in custom auth implementations. If you are not using a framework like Spring Security that handles this automatically, you MUST regenerate the session ID after any privilege change (login, role elevation). Relying on developers to remember this manually is a recipe for vulnerability.
