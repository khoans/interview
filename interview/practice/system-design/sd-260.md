**260.** What is a CDN edge server? How does anycast routing direct users to the nearest one?

A CDN edge server is a cache node deployed at geographically distributed locations (Points of Presence) that stores copies of content close to end users. Anycast routing assigns the same IP address to multiple servers worldwide, and BGP routing naturally directs each user request to the nearest server based on network topology.

On our marketplace, CloudFront had 400+ edge locations globally. When a user in New York requested a vehicle listing page, anycast routing directed the DNS query to the nearest CloudFront edge, which served cached static assets (images, CSS, JS) locally. Dynamic API calls were forwarded to our origin (ALB in us-east-1) but still benefited from CloudFront optimized backbone network. We cached vehicle images aggressively (TTL 24 hours) since they rarely changed, reducing origin load by about 80%.

The trade-off: edge caching complicates cache invalidation. When a dealer updated vehicle photos, we had to issue CloudFront invalidation requests, which took 5-10 minutes to propagate globally. For time-sensitive content (pricing, availability), we used short TTLs (60 seconds) or cache-busting query parameters, accepting higher origin load for freshness.
