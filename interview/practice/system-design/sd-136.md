**136.** What is CIDR notation? How do you calculate subnet ranges (e.g., /16, /24, /32)?

========== THIS SECTION IS THE ANSWER ==========
CIDR (Classless Inter-Domain Routing) notation specifies an IP range as base_ip/prefix_length. The prefix length indicates how many bits are fixed for the network portion. /16 = 65,536 IPs (16 fixed bits, 16 for hosts), /24 = 256 IPs, /32 = 1 IP (single host). Each decrease of 1 in prefix doubles the address space.

**Real example:** On our automotive marketplace VPC: 10.0.0.0/16 gave us 65,536 IPs for the entire VPC. We carved it into subnets: 10.0.1.0/24 (256 IPs, 251 usable after AWS reserves 5) for public subnet AZ-a, 10.0.10.0/24 for app subnet AZ-a, 10.0.100.0/24 for data subnet AZ-a. A quick mental math trick: /24 = 256 IPs, /25 = 128, /26 = 64, /27 = 32. For Security Group rules, we used /32 to allow a single bastion IP, and 10.0.0.0/16 to allow any VPC-internal traffic.

**Trade-off:** Smaller subnets (/24, /25) give tighter isolation but fewer IPs per subnet and more subnets to manage. Larger subnets (/20 = 4,096 IPs) are simpler but provide less granular network segmentation. Our initial /24 subnets were too small for auto-scaling workloads — we later migrated app subnets to /22 (1,024 IPs) to handle peak scaling. The lesson: it is much harder to expand subnets than to start with adequate size.
