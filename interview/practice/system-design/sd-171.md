**171.** What is kubectl? Name 10 essential commands you use daily.

========== THIS SECTION IS THE ANSWER ==========

kubectl is the command-line tool for interacting with Kubernetes clusters. It communicates with the API server to manage resources, inspect cluster state, and troubleshoot issues.

Essential commands mapped to tasks from our marketplace operations: (1) kubectl get pods -n production -- check all running service instances. (2) kubectl describe pod <name> -- diagnose why a pod is failing (events, resource limits). (3) kubectl logs <pod> --tail=100 -- view recent application logs, similar to how we used Kibana. (4) kubectl apply -f deployment.yaml -- deploy or update resources declaratively. (5) kubectl rollout status deployment/listing-service -- watch a deployment progress. (6) kubectl rollout undo deployment/listing-service -- rollback a failed deployment. (7) kubectl exec -it <pod> -- /bin/sh -- shell into a container for debugging. (8) kubectl top pods -- check CPU/memory usage, like our New Relic dashboards. (9) kubectl get events --sort-by=.lastTimestamp -- see recent cluster events. (10) kubectl port-forward svc/search-service 8080:80 -- access a service locally for debugging.

The lesson: kubectl is powerful but dangerous in production. We would enforce RBAC so developers can only run read-only commands (get, describe, logs) in production, while write commands (apply, delete, exec) require elevated access or go through CI/CD. Always use --dry-run=client before applying changes.
