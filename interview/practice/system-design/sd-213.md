**213.** What is AWS PrivateLink? How does it avoid exposing traffic to the public internet?

========== THIS SECTION IS THE ANSWER ==========

PrivateLink provides private connectivity between VPCs, AWS services, and on-premises networks through ENIs (Elastic Network Interfaces) in your VPC. Traffic stays entirely within the AWS network -- it never traverses the public internet, avoiding exposure to DDoS, eavesdropping, and data exfiltration risks. You create a VPC endpoint that acts as a private entry point to the target service.

In our marketplace, we used VPC endpoints (powered by PrivateLink) for: (1) Interface endpoints for Secrets Manager, KMS, ECR, and CloudWatch -- our ECS tasks accessed these services without needing a NAT gateway or internet gateway. (2) Gateway endpoint for S3 and DynamoDB -- free, route-table-based access. (3) When integrating with a third-party dealer management platform, they exposed their API via PrivateLink, and we created an interface endpoint in our VPC. This meant our listing sync traffic never touched the internet, satisfying their security compliance requirements.

The trade-off: PrivateLink endpoints cost money ($0.01/GB processed + $0.01/hour per AZ), which adds up at scale. For S3 and DynamoDB, gateway endpoints are free -- always use those. For other services, evaluate whether the security benefit justifies the cost. We saved on NAT gateway costs (which are more expensive at $0.045/GB) by using PrivateLink for high-volume AWS service calls, making it cost-neutral overall.
