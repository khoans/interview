**265.** What is credential stuffing? How does it differ from brute force?

Credential stuffing uses stolen username/password pairs from data breaches to try logging into other services, exploiting password reuse. Unlike brute force (which guesses random passwords for one account), credential stuffing uses known-valid credentials across many accounts -- each attempt looks like a legitimate login, making detection harder.

On our marketplace with 30M monthly users, credential stuffing was a real threat since users often reused passwords from other breached sites. We defended against it with: rate limiting on login endpoints at CloudFront WAF, monitoring login failure rates in New Relic (a sudden spike from many IPs against many accounts signals stuffing, not brute force), requiring MFA for dealer admin accounts, and checking passwords against the HaveIBeenPwned API during registration to warn users if their chosen password was in a known breach.

The lesson: credential stuffing defeats traditional account lockout because each account only sees 1-2 failed attempts -- it is the aggregate pattern that reveals the attack. You need behavioral analysis (impossible travel, device fingerprinting, anomalous login times) alongside per-account protections.
