**226.** What is SCA (Software Composition Analysis)? How does it detect vulnerable dependencies?

SCA scans your project dependencies (direct and transitive) against known vulnerability databases like the NVD and GitHub Advisory Database to flag components with published CVEs.

On our marketplace, we used GitHub Dependabot for SCA. It continuously monitored our Spring Boot Maven dependencies and Next.js npm packages, automatically creating PRs when vulnerabilities were discovered. For example, when Log4Shell hit, Dependabot immediately flagged our transitive Log4j dependency and suggested the patched version. We also ran mvn dependency:tree checks in CI to detect unexpected transitive dependencies.

The lesson is that transitive dependencies are the real danger -- your direct dependencies may be fine, but a library three levels deep can have a critical CVE. SCA tools map the full dependency tree, which is impossible to do manually across dozens of microservices.
