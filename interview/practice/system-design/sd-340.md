**340.** Infrastructure as Code (Terraform) vs ClickOps (AWS Console): When is IaC overhead not worth it?

========== THIS SECTION IS THE ANSWER ==========

IaC is almost always worth it for production infrastructure. ClickOps is acceptable for one-off experiments, prototyping, or resources that exist temporarily and will be deleted within hours.

On our marketplace, we managed production infrastructure with IaC for ECS task definitions, RDS instances, ElastiCache clusters, security groups, and IAM roles. This let us replicate environments (dev, staging, prod), track changes in git, and recover from accidental deletions. But for quick experiments — "let me spin up an EC2 instance to test this configuration" — ClickOps was faster. The danger was when "temporary" ClickOps resources became permanent and nobody knew their configuration.

The trade-off: IaC has upfront cost — writing Terraform/CloudFormation, maintaining state files, handling drift detection. For a small team with 5 AWS resources, it might feel like overkill. But the moment you have multiple environments or need disaster recovery, IaC pays for itself immediately. The rule: if a resource will exist for more than a week or exists in production, it must be in IaC. Everything else is acceptable as ClickOps with a time-boxed cleanup.
