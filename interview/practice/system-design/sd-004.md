**4.** What is a CDN (Content Delivery Network)? How does it improve performance?

========== THIS SECTION IS THE ANSWER ==========

A CDN is a network of edge servers distributed worldwide that caches and serves content from the location closest to the user. Instead of every request traveling to your origin server in Virginia, a user in California gets the response from an edge server in Los Angeles — reducing latency from ~80ms to ~10ms.

In our project, we used CloudFront as our CDN. Vehicle images, JavaScript bundles, and CSS were served from edge locations across the US. For our 30M monthly users spread across the country, this reduced page load time significantly — static assets loaded in under 100ms regardless of user location. We also cached popular search API responses at the edge with a 60-second TTL.

The trade-off: CDN caching means users might see stale content. For static assets, we solved this with content hashing in filenames (`main.a1b2c3.js`) — new deploy means new filename, so the cache naturally refreshes. For API responses, we kept TTLs short (60 seconds) and used cache-control headers carefully. CDN adds cost per GB transferred, but for us the performance gain and origin offloading were well worth it.
