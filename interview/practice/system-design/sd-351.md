**351.** How did you secure your APIs? Walk me through every layer from the client to the database.

========== THIS SECTION IS THE ANSWER ==========

We had six layers of security from client to database on our automotive marketplace.

**Client to CDN**: All traffic over HTTPS via CloudFront with TLS 1.2+. AWS WAF rules blocked common attacks (SQL injection, XSS patterns) and rate-limited per IP. **CDN to Load Balancer**: CloudFront to ALB over HTTPS with origin access control. **Load Balancer to Service**: ALB terminated TLS and forwarded to ECS services in private subnets. Only ALB security group could reach the services. **Authentication/Authorization**: Spring Security with OAuth2 JWT tokens. Every request validated the Bearer token — signature, expiration, audience, and role claims. Method-level `@PreAuthorize` annotations enforced RBAC (dealer, consumer, admin). **Service to Service**: Internal calls used client credentials OAuth2 flow with dedicated service accounts — no shared credentials. **Service to Database**: PostgreSQL on RDS in a private subnet, accessible only from application security groups on port 5432. Credentials stored in Secrets Manager with automatic rotation. Encryption at rest via KMS, in transit via SSL. Parameterized queries everywhere — no string concatenation — preventing SQL injection.

The trade-off: every layer adds latency (JWT validation ~2ms, WAF inspection ~5ms), but with 30M users handling personal and financial data, each layer blocked a real attack category we observed in production logs.
