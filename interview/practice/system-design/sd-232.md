**232.** What is SLSA (Supply-chain Levels for Software Artifacts)? Explain the 4 levels.

SLSA (pronounced "salsa") is a framework for ensuring the integrity of software artifacts throughout the supply chain, with four progressive levels of security guarantees.

**Level 1**: Build process is documented (build exists). **Level 2**: Build service generates provenance (who built what, when, from which source). **Level 3**: Build platform is hardened -- source and build are verified, builds are isolated and reproducible. **Level 4**: Two-person review of all changes, hermetic builds, fully reproducible. On our marketplace, we were effectively at Level 2 -- GitHub Actions built our Docker images with traceable build logs, tagged images with git commit SHAs, and pushed to ECR. To reach Level 3, we would need build isolation and signed provenance attestations.

The trade-off is that higher SLSA levels require significant infrastructure investment. Level 4 is extremely difficult -- even Google only claims Level 3 for most internal projects. For most teams, Level 2-3 provides the best security-to-effort ratio.
