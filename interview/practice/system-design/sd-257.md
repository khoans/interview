**257.** What is the difference between L2 switch, L3 router, and L7 load balancer?

**L2 switch** operates at the data link layer using MAC addresses to forward frames within a LAN. **L3 router** operates at the network layer using IP addresses to route packets between different networks. **L7 load balancer** operates at the application layer, understanding HTTP/HTTPS content to make intelligent routing decisions based on URL paths, headers, cookies, etc.

On our marketplace, AWS handled all three layers: VPC networking provided L2/L3 functionality (subnet routing, internet gateways, NAT gateways), and our Application Load Balancer (ALB) operated at L7. The ALB inspected HTTP requests and routed based on path: /api/vehicles/* went to the vehicle service ECS target group, /api/dealers/* went to the dealer service, and /api/search/* went to the search service backed by Solr. We also used ALB L7 capabilities for sticky sessions, authentication integration, and request-based routing.

The trade-off: L7 load balancers are more expensive and add latency (they must parse HTTP) compared to L4 NLB (which just routes TCP packets). We used ALB for HTTP services needing content-based routing and would use NLB for raw TCP workloads where throughput matters more than routing intelligence.
