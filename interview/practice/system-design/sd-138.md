**138.** What is the difference between Layer 4 and Layer 7 load balancing from a security perspective?

========== THIS SECTION IS THE ANSWER ==========
Layer 4 (NLB) operates on TCP/UDP — it routes based on IP and port without inspecting packet content. Layer 7 (ALB) operates on HTTP — it can inspect headers, URLs, cookies, and request bodies. From a security perspective, L7 can make intelligent decisions (block by URL pattern, inspect for malicious payloads, route by authentication headers), while L4 can only filter by IP/port.

**Real example:** On our automotive marketplace, we used ALB (Layer 7) for all web traffic because it enabled security-critical features. WAF integration on ALB inspected HTTP requests for SQL injection and XSS patterns. Path-based routing directed /api/admin/* to the admin service with stricter Security Group rules. The ALB added X-Forwarded-For headers for real client IP logging (critical for audit trails). We used NLB (Layer 4) only for non-HTTP workloads like direct database connections through VPN, where HTTP inspection was irrelevant.

**Trade-off:** L7 load balancing is more expensive computationally because it must parse HTTP and terminate TLS, adding ~1-2ms latency. L4 passes TCP packets through with minimal overhead and supports millions of connections per second. From a security standpoint, L7 provides much richer protection but requires TLS termination at the LB (the LB sees plaintext HTTP), while L4 can pass encrypted traffic through without termination (but then cannot inspect it). For web applications, L7 is almost always the right choice.
