**121.** Explain the OSI model (7 layers). Which layers are most relevant for security?

========== THIS SECTION IS THE ANSWER ==========
The 7 OSI layers (bottom to top): Physical, Data Link, Network, Transport, Session, Presentation, Application. For security, the most relevant are: Layer 3 (Network) — IP filtering, firewalls, IPsec; Layer 4 (Transport) — TLS/SSL, TCP security; Layer 7 (Application) — authentication, authorization, input validation, WAF.

**Real example:** On our automotive marketplace, we had security controls at multiple layers. Layer 3: AWS Security Groups and NACLs filtered traffic by IP/port — only our VPN IPs could reach management ports. Layer 4: ALB terminated TLS 1.2+ for all traffic, NLB handled TCP-level health checks. Layer 7: AWS WAF on CloudFront inspected HTTP requests for SQL injection, XSS patterns, and blocked IPs from known malicious ranges. Spring Security handled authentication and authorization at the application layer. Each layer caught different attack types — WAF stopped injection attempts that firewalls could not see, and firewalls blocked port scans that WAF was unaware of.

**Trade-off:** Defense in depth (securing multiple layers) increases operational complexity and cost, but no single layer catches all attacks. Layer 7 inspection (WAF, application firewalls) is the most computationally expensive because it must parse full request content. Layer 3/4 filtering is fast (hardware-accelerated) but cannot inspect encrypted payloads. The practical approach: use AWS-managed services (Security Groups, WAF, Shield) to minimize operational burden.
