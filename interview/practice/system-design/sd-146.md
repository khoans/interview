**146.** What is broken authentication? What are the common mistakes in auth implementation?

========== THIS SECTION IS THE ANSWER ==========
Broken authentication encompasses vulnerabilities that allow attackers to compromise user identities: weak password policies, credential stuffing, session fixation, missing MFA, insecure session management, and exposing session tokens in URLs. It is consistently in the OWASP Top 10 because authentication is complex and one mistake can expose all users.

**Real example:** On our automotive marketplace, we addressed common auth mistakes through Spring Security: bcrypt password hashing with work factor 12 (no plain text or weak hashes), rate limiting on login endpoint (5 attempts per minute per IP via WAF rules), secure session management (HttpOnly + Secure + SameSite cookies, 30-minute idle timeout), JWT tokens with RS256 signatures and short expiration (15 minutes) with refresh token rotation. One specific incident: we discovered that password reset tokens were valid for 7 days with no single-use enforcement — an attacker who intercepted a reset email could use the link multiple times. We fixed it by storing reset tokens in Redis with single-use flag and 1-hour TTL.

**Trade-off:** Stricter security measures increase user friction — shorter sessions mean more re-logins, MFA adds steps, aggressive rate limiting can lock out legitimate users with typos. We balanced this by using refresh tokens (users stay logged in without re-entering credentials) and progressive security (MFA required only for sensitive actions like payment or account changes, not for browsing listings).
