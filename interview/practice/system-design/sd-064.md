**64.** What is a firewall? Explain the difference between WAF and network firewall.

========== THIS SECTION IS THE ANSWER ==========

A network firewall operates at Layer 3/4 (IP addresses, ports) — it allows or blocks traffic based on source/destination IP and port rules. A WAF (Web Application Firewall) operates at Layer 7 (HTTP) — it inspects request content, headers, and payloads to block application-level attacks like SQL injection, XSS, and bot traffic.

In our project, we used both. AWS Security Groups acted as our network firewall — the database security group only allowed inbound connections on port 5432 from the application security group (no public access). The ECS services only accepted traffic from the ALB security group. This layered network isolation meant even if an attacker compromised one service, they couldn't directly reach the database. On top of that, we deployed AWS WAF on our ALB and CloudFront distributions. WAF rules blocked: SQL injection patterns in query parameters (car search had user-supplied filters), known malicious IP ranges, and bot traffic that exceeded rate limits (scrapers trying to harvest our vehicle pricing data).

The trade-off: WAF adds latency (~1-2ms per request for rule evaluation) and can produce false positives — legitimate requests blocked because they resemble attacks. We had a case where a dealer's listing description containing SQL keywords ("SELECT trim level...") was blocked by the SQL injection rule. We had to tune the rule to exclude specific content-type fields. The lesson: use both network firewalls (defense in depth at the network layer) and WAF (protection against application attacks). But monitor WAF false positives carefully — blocking legitimate users is worse than most attacks.
