**366.** How do you configure SSL/TLS termination in Nginx? Explain ssl_certificate, ssl_protocols, and how to enforce HTTPS with a redirect.

========== THIS SECTION IS THE ANSWER ==========

SSL termination means Nginx handles the TLS handshake and decrypts traffic, then forwards plain HTTP to the backend — so your application server doesn't deal with encryption overhead. Key directives: `ssl_certificate` points to the cert file, `ssl_certificate_key` to the private key, `ssl_protocols` specifies which TLS versions to accept.

In our project: `server { listen 443 ssl; ssl_certificate /etc/nginx/ssl/cert.pem; ssl_certificate_key /etc/nginx/ssl/key.pem; ssl_protocols TLSv1.2 TLSv1.3; ssl_ciphers HIGH:!aNULL:!MD5; }`. To enforce HTTPS, we added a separate server block: `server { listen 80; return 301 https://$host$request_uri; }` — any HTTP request gets permanently redirected to HTTPS. We also added `add_header Strict-Transport-Security "max-age=31536000; includeSubDomains"` to tell browsers to always use HTTPS.

The trade-off: SSL termination at Nginx means traffic between Nginx and your backend is unencrypted. On a single EC2 where both run on localhost, this is fine — traffic never leaves the machine. For a setup where Nginx and backend are on different servers, you'd want backend TLS too (end-to-end encryption). We kept it simple since everything was on one box.
