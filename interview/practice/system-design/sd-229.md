**229.** What is git secret scanning? How do tools like GitLeaks and TruffleHog detect leaked credentials?

Git secret scanning searches repository history for accidentally committed secrets like API keys, passwords, database connection strings, and tokens using regex patterns and entropy analysis.

On our marketplace, we used multiple layers: GitHub built-in secret scanning detected AWS access keys and OAuth tokens in push events, and we could add GitLeaks as a pre-commit hook for local prevention. GitLeaks uses regex patterns (e.g., matching AKIA followed by 16 alphanumeric characters for AWS keys) while TruffleHog also uses entropy analysis -- if a string has high randomness, it is likely a secret. Both scan the full git history, not just the current state, which is critical because git rm does not remove secrets from previous commits.

The hard lesson: a teammate once committed an AWS key that was caught by GitHub scanning within minutes. Even with fast detection, the key was already in git history and had to be rotated immediately via AWS IAM. Prevention (pre-commit hooks) beats detection (post-push scanning) every time. We stored all secrets in AWS Secrets Manager and KMS.
