**120.** What is end-to-end encryption? How does it work in messaging apps (e.g., Signal protocol)?

========== THIS SECTION IS THE ANSWER ==========
End-to-end encryption (E2EE) means only the communicating users can read the messages — the server that relays messages cannot decrypt them. The Signal protocol achieves this using a combination of identity keys (long-term), signed pre-keys, and ephemeral keys with the Double Ratchet algorithm, which generates new encryption keys for every message, providing forward secrecy at the message level.

**Real example:** On our automotive marketplace, our dealer-buyer messaging did NOT use E2EE — messages were encrypted in transit (TLS) and at rest (AES-256 in PostgreSQL), but the server could read them for content moderation (filtering spam, detecting fraud, complying with legal requests). If we had implemented E2EE like Signal, we would have lost the ability to moderate content server-side, which was a legal requirement for our marketplace. This is the fundamental tension with E2EE in business applications.

**Trade-off:** E2EE provides the strongest privacy guarantee but eliminates server-side capabilities: no search indexing, no content moderation, no message backup without client-side key management. Key management shifts to the user — if they lose their device and have not backed up their keys, messages are lost forever. For consumer privacy apps (Signal, WhatsApp), this trade-off is acceptable. For enterprise/regulated applications, it often is not.
