**255.** What is DHCP? How does a device get an IP address automatically?

DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses to devices on a network through a 4-step process called DORA: **Discover** (client broadcasts looking for DHCP server), **Offer** (server proposes an IP), **Request** (client accepts the offer), **Acknowledge** (server confirms the assignment). The lease has an expiration time and must be renewed.

On our AWS infrastructure, DHCP happens automatically within the VPC. When a new ECS task launches, AWS assigns it a private IP from the subnet CIDR block via an internal DHCP service. We configured VPC DHCP option sets to specify our internal DNS servers and domain names. Each subnet in us-east-1 and us-west-2 had a defined CIDR range (e.g., 10.0.1.0/24 for private subnet A, 10.0.2.0/24 for private subnet B), and DHCP managed allocation within those ranges.

The trade-off: DHCP is convenient but can lead to IP exhaustion if subnets are too small. We once ran into this when auto-scaling launched too many ECS tasks in a /24 subnet (254 usable IPs). The fix was using larger subnets (/20) for ECS services.
