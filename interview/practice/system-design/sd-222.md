**222.** What is shift-left security? How do you integrate security into the development lifecycle?

Shift-left security means moving security testing earlier in the development process -- from production/staging back to the coding and design phase. The earlier you catch a vulnerability, the cheaper it is to fix.

On our platform, we shifted left at multiple stages: pre-commit hooks checked for secrets, SonarCloud ran SAST on every pull request in GitHub Actions, Dependabot opened PRs for vulnerable dependencies automatically, and Spring Security configurations were code-reviewed as part of every PR touching auth endpoints. We also used Flyway migrations with review gates so no unreviewed schema change hit production PostgreSQL.

The lesson is that shift-left does not mean eliminating runtime security -- you still need WAF, rate limiting, and monitoring in production. It means you catch 80% of issues before they ever merge.
