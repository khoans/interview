**221.** What is DevSecOps? How does it differ from traditional security approaches?

DevSecOps integrates security into every phase of the development lifecycle instead of treating it as a final gate before release. Traditional security does a review at the end; DevSecOps automates security checks continuously.

On our automotive marketplace, we embedded security into the GitHub Actions pipeline — SonarCloud scanned every PR for vulnerabilities, Dependabot flagged outdated dependencies, and container images were scanned before pushing to ECR. Security wasn't a separate team's problem; every developer owned it through automated gates.

The trade-off is pipeline speed — adding SAST, dependency checks, and image scanning added ~3 minutes to our CI. But catching a SQL injection in a PR is infinitely cheaper than catching it in production with 30M monthly users.
