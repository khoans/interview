**1.** What is scalability? Explain the difference between horizontal and vertical scaling.

========== THIS SECTION IS THE ANSWER ==========

Scalability is a system's ability to handle increased load by adding resources. Vertical scaling means upgrading the existing machine (more CPU, RAM) — simple but has a ceiling. Horizontal scaling means adding more machines behind a load balancer — no ceiling but requires your application to be stateless.

In our project, we scaled our search service horizontally on ECS — from 4 instances at low traffic to 12 during peak hours (evenings/weekends when car shoppers were active). The service was stateless; session data lived in Redis, so any instance could handle any request. Our PostgreSQL database scaled vertically (bigger RDS instance) because sharding a relational DB is complex and we hadn't hit that threshold yet.

The trade-off: vertical scaling is easier (no code changes) but you'll hit a hardware limit and it creates a single point of failure. Horizontal scaling is limitless but requires stateless design, load balancing, and handling distributed system challenges. Start vertical, go horizontal when you must.
