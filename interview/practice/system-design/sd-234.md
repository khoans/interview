**234.** What is the CIS Benchmark? How do you use it to harden Docker, Kubernetes, and Linux?

CIS (Center for Internet Security) Benchmarks are industry-standard configuration guidelines for hardening systems. They provide specific, actionable security recommendations scored as Level 1 (essential) and Level 2 (defense-in-depth).

On our ECS-based platform, we followed CIS Docker Benchmark recommendations: running containers as non-root users, limiting container capabilities (dropping ALL then adding only needed ones), using read-only filesystems where possible, setting resource limits (CPU/memory) to prevent DoS, and not mounting the Docker socket. For our EC2 hosts, CIS Linux benchmarks guided SSH hardening (key-only auth, no root login) and firewall rules via security groups.

The trade-off is strictness vs functionality. Some CIS recommendations break application functionality -- for example, read-only root filesystem breaks applications that write temp files. You need to audit each recommendation against your specific workloads. We treated CIS as a baseline checklist, accepting about 80% of recommendations and documenting exceptions.
