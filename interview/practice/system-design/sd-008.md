**8.** What is DNS and how does DNS resolution work?

========== THIS SECTION IS THE ANSWER ==========

DNS (Domain Name System) translates human-readable domain names (autotrader.com) into IP addresses (54.239.28.85). Resolution flow: browser cache → OS cache → recursive resolver (ISP) → root nameserver → TLD nameserver (.com) → authoritative nameserver (returns the IP). The result is cached at each level based on TTL.

In our project, we used Route 53 (AWS's DNS service) with health-check-based routing. Our domain pointed to two regions (us-east-1 and us-west-2). If Route 53's health check detected us-east-1 was down, it stopped returning that region's IP — traffic automatically shifted to us-west-2. We set TTL to 60 seconds so DNS changes propagated within a minute.

The trade-off: low TTL (10-60s) means faster failover but more DNS queries (slightly higher latency for the first request). High TTL (3600s) means fewer DNS lookups but slow failover — if your server dies, clients keep hitting the dead IP for up to an hour. For production services, 60 seconds is a good balance. For static marketing sites, higher TTL is fine.
