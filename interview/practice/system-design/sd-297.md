**297.** Your API is being scraped by bots. How do you detect and prevent it without affecting real users?

========== THIS SECTION IS THE ANSWER ==========

Layer multiple defenses: rate limiting, behavioral analysis, and fingerprinting. No single technique works alone because sophisticated bots adapt.

On our automotive marketplace with 30M monthly users, scrapers constantly harvested vehicle listings. We detected them by analyzing CloudFront logs in Kibana — bots showed uniform request intervals, sequential page crawling, and abnormal user-agent strings. We layered: AWS WAF rate limiting per IP (tiered for authenticated vs unauthenticated), CAPTCHA challenges on suspicious sessions, and CloudFront signed cookies for authenticated API access. We also deployed honeypot links invisible to real users that only bots would follow, automatically triggering IP blocks via Lambda.

The trade-off: aggressive bot protection can accidentally block legitimate users, search engine crawlers (hurting SEO), and partner integrations. We whitelisted verified bots like Googlebot via reverse DNS and used progressive escalation — rate limit first, then CAPTCHA, then block — so real users rarely hit the strictest tier.
