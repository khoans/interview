**134.** What is NAT (Network Address Translation)? How does it relate to private/public IPs?

========== THIS SECTION IS THE ANSWER ==========
NAT translates private IP addresses to public IP addresses, allowing instances in private subnets to access the internet (for updates, API calls) while remaining unreachable from the internet. The NAT device (NAT Gateway in AWS) maintains a mapping table: outbound traffic gets the NAT's public IP as the source, and return traffic is mapped back to the private IP.

**Real example:** On our automotive marketplace, our ECS tasks in private subnets needed internet access to call external APIs (payment processors, dealer management systems, vehicle data providers). We deployed NAT Gateways in each public subnet across two AZs (us-east-1a and us-east-1b). ECS tasks had private IPs (10.0.x.x), and all outbound internet traffic went through the NAT Gateway, appearing as the NAT's Elastic IP externally. This let us whitelist our NAT Gateway IPs with external partners (payment processor required a static IP whitelist), while keeping ECS tasks completely unreachable from the internet.

**Trade-off:** NAT Gateways in AWS cost ~$32/month per gateway plus $0.045/GB processed. For high-throughput workloads (we processed ~2TB/month outbound), this adds up. Also, NAT Gateway is a potential bottleneck — each supports up to 55,000 simultaneous connections per destination. We hit this limit once during a bulk data sync and had to spread traffic across multiple NAT Gateways using route table configurations.
