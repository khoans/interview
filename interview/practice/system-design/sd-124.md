**124.** What is TLS 1.3? What improved over TLS 1.2?

========== THIS SECTION IS THE ANSWER ==========
TLS 1.3 reduced the handshake from 2 round trips to 1 (and 0 for resumed connections via 0-RTT), removed insecure algorithms (RSA key exchange, CBC mode ciphers, SHA-1, RC4, DES), mandated perfect forward secrecy, and simplified the cipher suite list to just 5 secure options. The result is faster and more secure connections.

**Real example:** On our automotive marketplace, we configured CloudFront with a TLS 1.2 minimum policy (TLSv1.2_2021). When we evaluated TLS 1.3, the 1-RTT handshake reduced initial page load by ~100ms for users on high-latency mobile connections. The simplified cipher negotiation also meant fewer compatibility issues. We enabled TLS 1.3 on CloudFront by updating the security policy to TLSv1.3, which supported both 1.2 and 1.3 for backward compatibility with older clients. ALB also supported TLS 1.3 starting in 2023.

**Trade-off:** TLS 1.3's 0-RTT resumption mode enables replay attacks — an attacker can capture and replay the first message of a resumed session. Servers must implement replay protection (idempotent endpoints or anti-replay checks) for sensitive operations. We disabled 0-RTT for POST endpoints (payment, write operations) and only allowed it for idempotent GET requests where replay was harmless.
