**292.** An internal service is making requests to an external IP it shouldn't. How do you investigate?

========== THIS SECTION IS THE ANSWER ==========

Immediately isolate and investigate. Check VPC Flow Logs to identify which ECS task is making the outbound call, correlate with the deployment timeline, then trace the code path.

On our marketplace, we noticed unexpected outbound traffic from our listing-ingestion service to a third-party IP. We filtered VPC Flow Logs for that external IP, identified the ECS task, then reviewed recent deployments in GitHub Actions. Turned out a developer added a new library that included telemetry phoning home. The fix was removing the dependency and tightening egress security groups to only allow outbound to known IPs — our APIs, AWS service endpoints, and approved third-party partners.

Lesson learned: default VPC security groups allow all outbound traffic. Always restrict egress in production to a whitelist. We added NAT gateway logging and periodic VPC Flow Log audits to catch unauthorized outbound connections early.
