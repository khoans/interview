**61.** What is OAuth 2.0? Explain the authorization code flow.

========== THIS SECTION IS THE ANSWER ==========

OAuth 2.0 is an authorization framework that lets a third-party application access a user's resources without exposing their password. The authorization code flow: (1) Client redirects user to the authorization server. (2) User authenticates and consents. (3) Authorization server redirects back with a short-lived authorization code. (4) Client exchanges the code for an access token (server-to-server, code never exposed to the browser). (5) Client uses the access token to call the resource API.

In our project, we implemented OAuth 2.0 with Spring Security for dealer authentication. Dealers could log in with their dealership's Google Workspace account (Google as the identity provider). The authorization code flow was critical because the access token exchange happened server-side — the token was never exposed in the browser's URL or JavaScript. We also supported the client credentials flow for service-to-service authentication — when the batch processing Lambda called the listing API, it obtained a token using its client ID and secret, no user involved. Tokens were JWTs with 15-minute expiry, and refresh tokens had 7-day expiry stored securely in HttpOnly cookies.

The trade-off: the authorization code flow is the most secure but requires a server-side component (can't use it in a pure SPA without a backend). The implicit flow (token returned directly to the browser) is simpler but less secure — tokens are exposed in the URL fragment. We used authorization code flow with PKCE for the Next.js frontend, which is the current best practice for SPAs. The lesson: always use authorization code flow. The implicit flow is deprecated for good reason.
