**211.** What is AWS Secrets Manager vs Parameter Store? When do you use each?

========== THIS SECTION IS THE ANSWER ==========

Secrets Manager is purpose-built for secrets -- it provides automatic rotation, cross-account sharing, and native integration with RDS/Redshift for database credential rotation. Parameter Store (part of SSM) stores configuration and secrets in a hierarchy, supports encryption via KMS, but has no built-in rotation. Secrets Manager costs $0.40/secret/month plus $0.05/10K API calls; Parameter Store standard tier is free.

In our marketplace, we used both: Secrets Manager for database credentials (PostgreSQL password, DynamoDB access via IAM roles), third-party API keys (dealer management integrations), and OAuth client secrets -- all benefiting from automatic rotation. We configured Secrets Manager to rotate our RDS password every 30 days using a Lambda rotation function. Parameter Store stored non-sensitive configuration: feature flags, service URLs, environment-specific settings organized hierarchically (/prod/listing-service/solr-url, /prod/listing-service/cache-ttl). Our Spring Boot services read from both at startup via the AWS SDK.

The trade-off: Secrets Manager is more expensive but offers rotation out of the box -- essential for compliance requirements mandating regular credential rotation. Parameter Store is free and sufficient for secrets that do not need automatic rotation. We used the rule: if it needs rotation or is a database credential, use Secrets Manager; if it is configuration or a static secret, use Parameter Store with SecureString type.
