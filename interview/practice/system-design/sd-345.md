**345.** Fan-out-on-write vs fan-out-on-read: How do you choose for a social media feed? What about a mix (hybrid)?

========== THIS SECTION IS THE ANSWER ==========

Fan-out-on-write (push model) precomputes each follower's feed when content is posted — fast reads but expensive writes. Fan-out-on-read (pull model) assembles the feed at read time — cheap writes but expensive reads. The hybrid is usually the right answer at scale.

Drawing from our marketplace analogy: we used a fan-out-on-write approach for saved search notifications. When a new listing matched saved search criteria, we precomputed which users to notify and pushed to their notification queue (SQS to SNS). This was feasible because each listing matched a manageable number of saved searches. If we had done fan-out-on-read, every time a user opened their notifications, we would query all new listings against all their saved searches — slow and expensive at 30M users.

The trade-off: fan-out-on-write is fast for readers but a celebrity posting to 10M followers means 10M writes — hotspot problem. Fan-out-on-read handles celebrities well (one query per reader) but is slow for readers following many accounts. The hybrid approach (used by Twitter): fan-out-on-write for normal users, fan-out-on-read for celebrities. This limits write amplification while keeping reads fast for most users.
