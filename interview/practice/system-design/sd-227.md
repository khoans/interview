**227.** What is container image scanning? Compare Trivy, Snyk, and Grype.

Container image scanning analyzes Docker images for known vulnerabilities in OS packages, application dependencies, and misconfigurations before deployment.

On our platform, we scanned Docker images in GitHub Actions before pushing to Amazon ECR. **Trivy** is open-source, fast, and covers OS packages plus app dependencies -- the most popular for CI pipelines. **Snyk** offers developer-friendly UI, fix suggestions, and IDE integration but is commercial. **Grype** (by Anchore) is also open-source, focuses on vulnerability matching, and pairs well with Syft for SBOM generation. We used ECR built-in scanning which automatically scanned images on push.

The trade-off is scan depth vs speed. Trivy is fast and free but may miss edge cases commercial tools like Snyk catch. The key lesson: scanning on push is not enough -- you need continuous rescanning because new CVEs are published daily against already-deployed images.
