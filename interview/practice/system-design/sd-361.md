**361.** What is Nginx? Explain the difference between Nginx as a web server, reverse proxy, and load balancer.

========== THIS SECTION IS THE ANSWER ==========

Nginx is a high-performance, event-driven server that can serve three roles. As a **web server**, it serves static files (HTML, CSS, JS, images) directly to clients — fast and efficient for static content. As a **reverse proxy**, it sits in front of application servers, forwarding client requests to backend services and returning responses — clients never talk to your app server directly. As a **load balancer**, it distributes incoming requests across multiple backend instances using algorithms like round-robin or least connections.

In our Korean ecommerce project, Nginx served all three roles on a single EC2 instance: it served Next.js static assets directly (web server), proxied API requests to the Spring Boot backend on a different port (reverse proxy), and handled our blue-green deployment by switching between two application instances (load balancer between Blue and Green). On the ATC project, Akamai and ALB handled these roles at scale, but Nginx is the go-to for simpler setups.

The trade-off: Nginx handles all three roles in a single lightweight process, which is perfect for small-to-medium deployments. At large scale, you'd use dedicated services — CloudFront for static serving, ALB for load balancing — because they offer auto-scaling, managed SSL, and health checks out of the box. But for a single-server setup, Nginx does everything you need.
