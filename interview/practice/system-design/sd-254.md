**254.** What is BGP (Border Gateway Protocol)? Why is it called the "internet's routing protocol"?

BGP is the protocol that enables autonomous systems (AS) -- large networks like ISPs, cloud providers, and enterprises -- to exchange routing information and determine the best path for traffic across the internet. It is called the internet routing protocol because every packet crossing network boundaries relies on BGP routing tables.

On our marketplace, BGP was relevant in two ways: Route 53 uses anycast (BGP-based) to route DNS queries to the nearest AWS DNS server, and CloudFront uses BGP to announce its edge location IPs globally so that users connect to the nearest edge. When AWS had a BGP-related incident (like the us-east-1 outage), our multi-region setup (us-east-1 + us-west-2) helped because Route 53 health checks detected the issue and rerouted traffic to the healthy region.

The trade-off: BGP is trust-based -- any AS can announce any IP prefix, leading to BGP hijacking incidents where traffic gets misrouted. AWS uses RPKI (Resource Public Key Infrastructure) to validate route announcements, but not all ISPs do. This is why CDNs like CloudFront are valuable -- they have peering agreements with major ISPs for reliable routing.
