**151.** What is Content Security Policy (CSP)? How does it prevent XSS?

========== THIS SECTION IS THE ANSWER ==========

CSP is an HTTP response header that tells browsers which sources of content (scripts, styles, images) are allowed to load. It prevents XSS by blocking inline scripts and restricting script sources to a whitelist, so even if an attacker injects malicious JavaScript, the browser refuses to execute it.

In our automotive marketplace, we set CSP headers on our Next.js frontend served through CloudFront. We whitelisted only our own domain and CDN for scripts, and S3 for images. When a penetration test attempted reflected XSS through a search parameter, the browser blocked the payload because the inline script violated our CSP directive.

The trade-off: strict CSP breaks legitimate third-party integrations (analytics, ads, chat widgets). We had to carefully whitelist Google Analytics and our dealer chat SDK domains. Start with CSP in report-only mode (Content-Security-Policy-Report-Only) to catch violations without breaking production, then enforce once clean.
