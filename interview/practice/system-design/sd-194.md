**194.** What is supply chain security for containers? Explain image signing (Cosign/Sigstore).

========== THIS SECTION IS THE ANSWER ==========

Supply chain security ensures that every component in your software delivery pipeline -- from source code to base images to deployed containers -- is verified and untampered. Image signing uses cryptographic signatures to prove an image was built by a trusted pipeline. Cosign (part of Sigstore) signs container images and stores signatures in the OCI registry alongside the image. Verifiers (admission controllers) check the signature before allowing deployment.

In our marketplace, we built images in GitHub Actions and pushed to ECR -- we trusted the pipeline but did not cryptographically verify images. With Cosign, the workflow would be: (1) GitHub Actions builds the image, (2) Cosign signs it using a keyless signature tied to the CI identity (OIDC), (3) the signature is stored in ECR alongside the image, (4) a Kubernetes admission controller (Kyverno or Gatekeeper with Cosign verification) rejects any unsigned image. This prevents deploying images built outside the trusted pipeline -- even if someone pushes directly to ECR.

The trade-off: image signing adds a step to the CI pipeline and requires admission controller infrastructure. But it prevents a critical attack vector -- a compromised developer pushing a backdoored image directly to the registry. For our 30M users/month platform, this level of supply chain verification is worth the added complexity.
