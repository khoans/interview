**207.** What is AWS GuardDuty? How does it detect threats?

========== THIS SECTION IS THE ANSWER ==========

GuardDuty is an intelligent threat detection service that continuously analyzes CloudTrail logs, VPC Flow Logs, and DNS logs using machine learning, anomaly detection, and threat intelligence feeds. It identifies suspicious activity like cryptocurrency mining, unauthorized access, compromised instances communicating with known malicious IPs, and unusual API call patterns.

In our marketplace, GuardDuty was enabled across all accounts in our Organization. Key detections it provided: (1) it flagged when an EC2 instance in our dev account started communicating with a known command-and-control IP -- turned out a developer had installed an unvetted tool. (2) It detected an IAM access key being used from an IP in a country where we had no employees (credential compromise from a leaked key). (3) It identified unusual S3 access patterns when a script started downloading large amounts of listing data outside normal business hours. GuardDuty findings were sent to SNS, which triggered a Lambda to create Jira tickets for our security team to investigate.

The trade-off: GuardDuty has a per-event pricing model that scales with your log volume -- at our scale (30M users), the cost was significant but justified. It generates false positives, especially for new API patterns (a new deployment automation might trigger unusual API alerts). Tune severity thresholds and create suppression rules for known patterns to reduce alert fatigue.
