**154.** What is defense in depth? Give an example with multiple security layers.

========== THIS SECTION IS THE ANSWER ==========

Defense in depth is a security strategy that layers multiple independent security controls so that if one layer fails, others still protect the system. No single control is relied upon exclusively.

In our automotive marketplace, a request to view a vehicle listing passed through 6 security layers: (1) AWS Shield and WAF at the edge blocked DDoS and common attack patterns like SQL injection signatures, (2) CloudFront enforced TLS termination, (3) Route 53 DNS had health checks and geo-restrictions, (4) our ALB only accepted traffic from CloudFront via a custom header secret, (5) Spring Security in the application enforced OAuth2 authentication and RBAC authorization, (6) PostgreSQL used row-level permissions so even a compromised app query could not access another dealer data. All data was encrypted at rest via KMS and in transit via TLS.

The lesson: when our WAF rule missed a new XSS variant, the CSP header on the frontend still blocked it. Each layer assumes the others might fail. The cost is operational complexity -- more layers mean more things to configure, monitor, and maintain.
