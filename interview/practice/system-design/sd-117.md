**117.** What is HMAC (Hash-based Message Authentication Code)? When would you use it?

========== THIS SECTION IS THE ANSWER ==========
HMAC combines a cryptographic hash function with a secret key to produce a message authentication code. It verifies both data integrity (message was not tampered with) and authenticity (sender has the shared secret). Unlike a plain hash, HMAC requires the secret key, so an attacker cannot forge a valid code without it.

**Real example:** On our automotive marketplace, we used HMAC-SHA256 for webhook signature verification. When third-party services (payment processors, dealer management systems) sent webhooks to our API, each request included an X-Signature header containing HMAC-SHA256(request_body, shared_secret). Our Spring Boot webhook controller recomputed the HMAC using the stored shared secret and compared it to the header value. If they did not match, the request was rejected — this prevented attackers from sending fake webhook events to our payment processing endpoint.

**Trade-off:** HMAC requires both parties to share the same secret key (symmetric), which means secure key distribution is critical. If the shared secret is compromised, the attacker can forge valid HMACs. For scenarios where you cannot share a secret securely, digital signatures (asymmetric) are better — the sender signs with their private key, and the receiver verifies with the public key without ever possessing the signing key.
