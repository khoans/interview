**141.** What is the OWASP Top 10? Name at least 5 and explain each briefly.

========== THIS SECTION IS THE ANSWER ==========
The OWASP Top 10 is a periodically updated list of the most critical web application security risks. Five key ones: (1) Broken Access Control — users accessing resources they should not, (2) Injection — SQL/NoSQL/command injection through unsanitized input, (3) Cryptographic Failures — weak encryption, cleartext data exposure, (4) Security Misconfiguration — default credentials, open S3 buckets, verbose error messages, (5) Server-Side Request Forgery (SSRF) — tricking the server into making requests to internal resources.

**Real example:** On our automotive marketplace, we actively defended against all of these. Broken Access Control: Spring Security's @PreAuthorize checked dealer_id ownership on every listing mutation. Injection: Spring Data JPA parameterized queries prevented SQL injection; we also had WAF rules blocking injection patterns. Cryptographic Failures: KMS encrypted PII in DynamoDB, TLS everywhere. Security Misconfiguration: SonarCloud scanned for default credentials and exposed endpoints in CI/CD. SSRF: we validated and whitelisted URLs when our service fetched external vehicle data feeds.

**Trade-off:** Defending against all OWASP Top 10 requires defense in depth across multiple layers — WAF, application code, framework configuration, infrastructure. No single tool covers everything. We prioritized based on our threat model: Injection and Broken Access Control were highest risk for a marketplace handling financial transactions, so those got the most rigorous testing (penetration testing, automated security scanning in the pipeline via SonarCloud).
