**122.** What is the TCP/IP model? How does it differ from the OSI model?

========== THIS SECTION IS THE ANSWER ==========
The TCP/IP model has 4 layers: Network Access (combines OSI Physical + Data Link), Internet (OSI Network), Transport (OSI Transport), and Application (combines OSI Session + Presentation + Application). TCP/IP is the practical model that the internet actually runs on; OSI is the theoretical reference model. TCP/IP was designed around real protocols (TCP, IP, HTTP), while OSI was designed around abstract layer concepts.

**Real example:** On our automotive marketplace, mapping our stack to TCP/IP: Network Access layer — AWS VPC networking, ENIs on ECS tasks. Internet layer — Route 53 DNS, IP routing between us-east-1 and us-west-2, NAT gateways for private subnet internet access. Transport layer — TCP connections from ALB to ECS targets, TLS termination. Application layer — HTTP/HTTPS for REST APIs, WebSocket for real-time dealer messaging. In practice, we never discussed OSI layers in team meetings — everything was TCP/IP terminology: "Is the security group allowing TCP port 5432?" not "Is the Layer 4 firewall rule correct?"

**Trade-off:** The OSI model is better for understanding where security controls operate (Layer 3 vs Layer 7 firewall discussions) and for vendor certifications. TCP/IP is better for practical troubleshooting and configuration. Most cloud networking (AWS VPC, Security Groups, NACLs) maps directly to TCP/IP concepts rather than OSI layers.
