**237.** What is a security incident response plan? What are the 6 phases (NIST)?

A security incident response plan defines how an organization detects, responds to, and recovers from security incidents. NIST defines 6 phases: **Preparation**, **Identification**, **Containment**, **Eradication**, **Recovery**, and **Lessons Learned**.

On our marketplace, when we detected suspicious API traffic patterns in New Relic (Identification), we had a runbook: immediately revoke affected API keys and block IPs at CloudFront WAF (Containment), investigate logs in Kibana to find the attack vector (Eradication), restore normal service and verify data integrity in PostgreSQL/DynamoDB (Recovery), then conduct a blameless post-mortem (Lessons Learned). Preparation meant having these runbooks, monitoring dashboards, and on-call rotations ready before incidents happened.

The trade-off is that incident response plans are only useful if practiced. We ran quarterly tabletop exercises simulating scenarios like credential leaks or DDoS. Without practice, even a perfect plan fails under the stress of a real incident.
