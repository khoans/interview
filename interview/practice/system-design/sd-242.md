**242.** What is a MAC address? How does ARP (Address Resolution Protocol) work?

A MAC (Media Access Control) address is a hardware-level 48-bit identifier burned into a network interface card (e.g., AA:BB:CC:DD:EE:FF). ARP translates IP addresses to MAC addresses within a local network -- when a device wants to send a packet to IP 10.0.1.5, it broadcasts "who has 10.0.1.5?" and the device with that IP responds with its MAC address. This mapping is cached in an ARP table.

On our AWS infrastructure, ARP operates transparently within the VPC. When an ECS container communicates with our PostgreSQL RDS instance in the same VPC, the underlying hypervisor handles ARP resolution between the ENIs (Elastic Network Interfaces). AWS also uses ARP for health checks -- if an EC2 instance becomes unhealthy, the ALB removes it from the target group partly by monitoring ARP-level reachability.

The trade-off to know: ARP spoofing is a real attack vector on physical networks where an attacker sends fake ARP replies to redirect traffic. In cloud environments like AWS, the hypervisor prevents ARP spoofing between instances, which is one security benefit of VPC isolation.
