**261.** What is multi-factor authentication (MFA)? Explain TOTP, SMS, WebAuthn/FIDO2.

MFA requires two or more verification factors: something you know (password), something you have (phone, security key), something you are (biometrics). **TOTP** generates time-based one-time codes using a shared secret (Google Authenticator). **SMS** sends codes via text message (least secure -- vulnerable to SIM swapping). **WebAuthn/FIDO2** uses public-key cryptography with hardware tokens or platform biometrics (most secure -- phishing resistant).

On our marketplace, we enforced MFA for all dealer admin accounts and internal team access using TOTP via our OAuth2 provider. Dealer administrators managing vehicle inventory worth millions needed stronger security than just passwords. We chose TOTP over SMS because SMS is vulnerable to SIM-swap attacks and has delivery reliability issues. WebAuthn was considered but not all dealers had compatible hardware at the time.

The trade-off: stronger MFA methods have higher friction and support costs. TOTP requires users to set up an authenticator app (support tickets increase 20-30%). SMS is easiest for users but weakest in security. The right choice depends on the risk profile -- financial/admin operations justify the friction of TOTP or WebAuthn.
