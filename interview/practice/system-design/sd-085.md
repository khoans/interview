**85.** What is request coalescing? How does it reduce duplicate work?

========== THIS SECTION IS THE ANSWER ==========
Request coalescing (or request collapsing) groups multiple identical in-flight requests into a single backend call. Only one request actually executes; all others wait and receive the same result. This eliminates redundant computation and database/API hits.

**Real example:** On our automotive marketplace, when a popular vehicle model page went viral, hundreds of concurrent requests hit the same Solr search query within milliseconds. We used Spring's cache abstraction with a coalescing layer — the first request fetched from Solr while subsequent identical requests waited on the same CompletableFuture. Once the result came back, all waiting callers got the response. This reduced Solr load by 60% during traffic spikes.

**Trade-off:** If the single executing request fails, all coalesced callers fail together. We mitigated this with retry logic on the coalesced group and a short TTL so a subsequent wave could retry independently. Also, coalescing only helps when you have many identical requests in a tight time window — for diverse queries with unique parameters, it provides no benefit.
