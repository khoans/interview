**316.** Pub/Sub vs Point-to-Point messaging: When do you need fan-out to multiple consumers vs exactly one consumer?

========== THIS SECTION IS THE ANSWER ==========

Use pub/sub (fan-out) when multiple independent services need to react to the same event. Use point-to-point when exactly one consumer should process each message to avoid duplicate work.

On our marketplace, we used both patterns via SNS+SQS. When a dealer published a new listing, SNS broadcast the event (pub/sub) to multiple SQS queues — search indexing, analytics, notification service each had their own queue and processed independently. But for payment processing, we used point-to-point SQS — exactly one consumer should process each payment to avoid double-charging. The SQS visibility timeout ensured only one consumer processed each message.

The trade-off: pub/sub enables loose coupling — the producer does not know or care how many consumers exist, and you can add new consumers without changing the producer. But every consumer gets every message, so if you have high-volume events and many consumers, the total message processing multiplies. Point-to-point is simpler and avoids duplication but creates tight coupling — if you later need a second consumer, you have to redesign the flow.
