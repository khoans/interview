**286.** Your TLS certificate expired in production and users see browser warnings. How do you prevent this?

Immediate fix: renew the certificate and deploy it. On AWS, if using ACM (AWS Certificate Manager), certificates auto-renew -- so if this happened, something is misconfigured.

On our marketplace, we prevented this by using ACM certificates exclusively -- ACM automatically renews certificates 60 days before expiry for certificates used with CloudFront and ALB, with zero manual intervention. For the rare case of certificates not managed by ACM (e.g., third-party certificates for partner domains), we set up CloudWatch alarms that triggered SNS notifications 30, 14, and 7 days before expiry. We also monitored certificate expiry dates via New Relic synthetic checks that verified TLS certificate validity as part of our health monitoring.

The trade-off: ACM certificates are free and auto-renewing but only work with AWS services (CloudFront, ALB, API Gateway). If you need certificates for EC2 instances or non-AWS services, you need a different solution like Let us Encrypt with certbot auto-renewal, or a commercial CA with monitoring. The lesson: certificate expiry should never be a manual process -- automate it completely, and monitor as a backup.
