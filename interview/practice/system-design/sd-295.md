**295.** Your Docker images have critical vulnerabilities from the base image. How do you handle it?

========== THIS SECTION IS THE ANSWER ==========

Immediately update the base image, rebuild, scan, and redeploy. Then establish automated scanning to prevent recurrence.

On our marketplace, we ran ECR image scanning on every push via GitHub Actions. When a critical CVE appeared in our openjdk:17-jdk-slim base image, we updated the Dockerfile to the latest patched version, rebuilt all affected service images, verified the ECR scan passed clean, then did a rolling deployment on ECS. We also added a scheduled weekly ECR scan across all repositories with SNS notifications for HIGH/CRITICAL findings. We standardized on a single base image across all Java services so one fix covered everything.

The trade-off: minimal base images like distroless or alpine reduce vulnerability surface dramatically but make debugging harder — no shell, no curl, no common tools. We used slim images for production and full JDK images only in local development. The debugging inconvenience was worth the security benefit.
