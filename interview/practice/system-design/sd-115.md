**115.** What is key management? How do you handle key rotation without downtime?

========== THIS SECTION IS THE ANSWER ==========
Key management covers the lifecycle of cryptographic keys: generation, storage, distribution, rotation, and destruction. Key rotation without downtime requires supporting multiple key versions simultaneously — encrypt with the new key, but decrypt with both old and new keys during a transition period.

**Real example:** On our automotive marketplace, we used AWS KMS for all key management. KMS supports automatic annual key rotation: when rotated, KMS keeps the old key material and uses the new material for new encryptions, but transparently decrypts old data using the previous key version — zero downtime, zero code changes. For application-level secrets (API keys, database passwords), we used AWS Secrets Manager with automatic rotation via Lambda functions. The rotation Lambda would create a new database password, update PostgreSQL, update the secret, and our Spring Boot services fetched the latest secret on next connection pool refresh.

**Trade-off:** Automatic rotation adds complexity in testing — you must verify the rotation Lambda works correctly, or a failed rotation breaks all services. We learned this the hard way when a rotation Lambda had a bug that created the new password in PostgreSQL but failed to update Secrets Manager, causing a 15-minute outage. Always test rotation in staging first and have a rollback procedure (manual secret update).
