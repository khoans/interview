**9.** What are the differences between TCP and UDP? When would you use each?

========== THIS SECTION IS THE ANSWER ==========

TCP (Transmission Control Protocol) guarantees delivery — it establishes a connection (3-way handshake), ensures packets arrive in order, and retransmits lost packets. UDP (User Datagram Protocol) is fire-and-forget — no connection, no ordering, no retransmission, but much faster and lower overhead.

Use TCP when data integrity matters: HTTP/HTTPS (web APIs), database connections, file transfers, email. Use UDP when speed matters more than perfect delivery: video streaming, online gaming, DNS queries, VoIP. Losing one video frame is fine; losing a bank transaction is not.

In our project, all API traffic was TCP (via HTTP/HTTPS) — every search request and response had to be complete and accurate. Our monitoring system (New Relic agent) used UDP for sending metrics to the collector — if a few metrics packets were lost, it didn't affect the dashboard meaningfully, and UDP's lower overhead meant less impact on our application's performance. The trade-off is simple: TCP = reliable but slower setup, UDP = fast but unreliable. Choose based on your tolerance for data loss.
