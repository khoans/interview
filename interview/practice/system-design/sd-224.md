**224.** What is SAST (Static Application Security Testing)? Name tools (SonarQube, Semgrep, Checkmarx).

SAST analyzes source code without executing it to find security vulnerabilities like SQL injection, XSS, and hardcoded secrets. It scans the code structure, data flow, and patterns against known vulnerability rules.

We used SonarCloud (hosted SonarQube) in our GitHub Actions pipeline. Every PR triggered a scan that checked for injection flaws, insecure cryptography, and code smells in our Spring Boot services. Popular alternatives include Semgrep (open-source, fast, custom rules), Checkmarx (enterprise-grade, deep data-flow analysis), and Fortify. SonarCloud was our choice because it integrated natively with GitHub PRs and provided inline comments on vulnerable lines.

The trade-off is false positives -- SonarCloud would sometimes flag parameterized queries as SQL injection risks. We spent time tuning rules and marking false positives, which is ongoing maintenance but necessary to keep developers trusting the tool.
