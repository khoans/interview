**139.** What is HTTP Strict Transport Security (HSTS)? Why should you enable it?

========== THIS SECTION IS THE ANSWER ==========
HSTS is an HTTP response header (Strict-Transport-Security) that tells browsers to ONLY connect to your site via HTTPS for a specified duration, even if the user types http://. It prevents SSL stripping attacks where an attacker downgrades a user's initial HTTP request to intercept traffic before the HTTPS redirect happens.

**Real example:** On our automotive marketplace, we set the HSTS header on all responses: `Strict-Transport-Security: max-age=31536000; includeSubDomains; preload`. This told browsers to enforce HTTPS for one year, including all subdomains (api.*, cdn.*, dealer.*). We also submitted our domain to the HSTS preload list (hstspreload.org), which hardcodes the HSTS policy into Chrome/Firefox/Safari, protecting even the very first visit. Before enabling HSTS, we ensured all subdomains supported HTTPS — a common gotcha is enabling includeSubDomains when some internal subdomain still uses HTTP.

**Trade-off:** HSTS with a long max-age is difficult to undo — if you need to revert to HTTP for any reason (certificate emergency, migration), users' browsers will refuse to connect for the entire max-age duration. Start with a short max-age (300 seconds) during testing, then increase to 1 year once stable. The preload list is even harder to undo (requires browser updates to remove). Always verify ALL subdomains support HTTPS before enabling includeSubDomains.
