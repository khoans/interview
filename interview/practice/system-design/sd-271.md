**271.** What is API authentication? Compare API keys, OAuth tokens, and mutual TLS.

**API keys** are simple static secrets included in headers -- easy to implement but no expiry, no scoping, and hard to rotate. **OAuth tokens** are scoped, time-limited, and support delegation -- the standard for user-facing APIs. **Mutual TLS (mTLS)** uses client certificates for both sides to verify each other -- strongest security, common for service-to-service communication.

On our marketplace, we used different methods for different contexts: public API endpoints used OAuth2 Bearer tokens (JWT with scopes like read:vehicles, write:listings), internal service-to-service calls used OAuth2 client credentials flow, and third-party partner integrations used API keys with rate limiting (simpler for partners to implement). We considered mTLS for high-security dealer payment APIs but the certificate management overhead was significant -- issuing, rotating, and revoking client certificates for hundreds of dealers would require a PKI infrastructure.

The trade-off: OAuth tokens are the most flexible but most complex. API keys are simplest but least secure (no expiry, no scoping). mTLS is strongest but operationally heavy. Choose based on the trust boundary -- public internet gets OAuth, trusted internal network gets client credentials, partner integrations get API keys with monitoring.
