**108.** What is symmetric encryption (AES)? How does it work and when do you use it?

========== THIS SECTION IS THE ANSWER ==========
Symmetric encryption uses the same key for both encryption and decryption. AES (Advanced Encryption Standard) is the most widely used symmetric algorithm, operating on fixed-size blocks (128 bits) with key sizes of 128, 192, or 256 bits. It is fast and efficient, ideal for encrypting large volumes of data at rest or in transit.

**Real example:** On our automotive marketplace, we used AES-256 through AWS KMS for encrypting sensitive data at rest. DynamoDB tables storing PII (dealer tax IDs, bank account numbers) used AWS-managed encryption with AES-256 under the hood. S3 buckets containing vehicle inspection documents used SSE-KMS (server-side encryption with KMS-managed keys). The encryption/decryption was transparent to our Spring Boot services — KMS handled key management, and the AWS SDK handled the encrypt/decrypt calls.

**Trade-off:** The main challenge with symmetric encryption is key distribution — both parties need the same secret key, so how do you share it securely? In practice, symmetric encryption is used for bulk data encryption, while asymmetric encryption (RSA) is used to exchange the symmetric key. This hybrid approach (used in TLS) gives you the speed of AES with the key exchange security of RSA.
