**155.** What is OAuth 2.0 vs OpenID Connect (OIDC)? When do you need which?

========== THIS SECTION IS THE ANSWER ==========

OAuth 2.0 is an authorization framework -- it grants third-party applications limited access to resources without sharing passwords by issuing access tokens. OpenID Connect (OIDC) is an identity layer built on top of OAuth 2.0 -- it adds authentication by providing an ID token (JWT) that contains user identity claims like name, email, and subject.

In our marketplace, we used both. OAuth 2.0 handled authorization: when a third-party dealer management tool needed to update listings on behalf of a dealer, it got an access token with listings:write scope via the authorization code flow. OIDC handled authentication: when users logged into our platform through Sign in with Google, we received an ID token confirming their identity, then created or mapped their internal account. Our Spring Security configuration used spring-security-oauth2-resource-server to validate access tokens and spring-security-oauth2-client for the OIDC login flow.

The trade-off: if you only need to know who the user is, OIDC is sufficient. If you need delegated API access, you need OAuth 2.0. We needed both because we had both consumer login and third-party integrations.
