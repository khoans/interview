**225.** What is DAST (Dynamic Application Security Testing)? How does it differ from SAST?

DAST tests a running application by sending actual HTTP requests and analyzing responses for vulnerabilities -- it works from the outside in, like an attacker would. SAST analyzes source code statically; DAST analyzes runtime behavior dynamically.

On our automotive marketplace, SAST (SonarCloud) caught code-level issues in PRs, while DAST would run against our staging environment to detect runtime issues like misconfigured CORS headers, missing security headers on CloudFront responses, or exposed Spring Boot Actuator endpoints. Tools like OWASP ZAP or Burp Suite crawl the application, test for XSS, injection, and authentication bypasses against live endpoints.

The key trade-off: DAST finds real exploitable issues (fewer false positives) but runs slower and later in the pipeline. SAST is fast and early but has more false positives. You need both -- SAST for shift-left speed, DAST for runtime confidence.
