**82.** What is a content-addressable storage? How does deduplication work?

========== THIS SECTION IS THE ANSWER ==========
Content-addressable storage (CAS) uses the hash of the content itself as the storage key. If two files produce the same hash, they are the same content and stored only once — this is deduplication.

**Real example:** On our automotive marketplace, vehicle listing images were stored in S3. Dealers frequently uploaded the same stock photos across multiple listings. We implemented a deduplication layer: before uploading, the service computed a SHA-256 hash of the image, checked if that key already existed in S3, and if so, just linked the new listing to the existing object. This reduced our S3 storage costs by roughly 30% across millions of vehicle images.

**Trade-off:** The dedup check adds latency to every upload (hash computation + S3 HEAD request). For our use case the savings justified it, but for write-heavy workloads with unique content, it is pure overhead. Also, you cannot delete an object until all references are gone, so we needed a reference-counting mechanism with periodic garbage collection.
