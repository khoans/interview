**228.** What is Infrastructure as Code (IaC) scanning? How do tools like Checkov and tfsec work?

IaC scanning analyzes infrastructure definition files (Terraform, CloudFormation, Kubernetes manifests) for security misconfigurations before deployment -- like SAST but for infrastructure.

On our AWS infrastructure, tools like Checkov or tfsec would scan CloudFormation templates to catch issues like S3 buckets without encryption, security groups with 0.0.0.0/0 ingress on port 22, or RDS instances without encryption at rest. These tools have built-in policy libraries mapped to CIS benchmarks and AWS best practices. Checkov supports multiple frameworks (Terraform, CF, Kubernetes, Docker); tfsec is Terraform-specific but very fast.

The lesson from our experience: we once had a staging environment where an ECS task definition allowed broader network access than needed. IaC scanning would have caught that at PR time. The trade-off is policy customization -- out-of-box rules are too strict for many teams, so you spend time tuning which checks are enforced vs advisory.
