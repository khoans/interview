**273.** What is an identity provider (IdP)? Compare Keycloak, Auth0, AWS Cognito, and Okta.

An IdP manages user identities and authentication, issuing tokens that applications trust. **Keycloak** is open-source, self-hosted, highly customizable but you manage the infrastructure. **Auth0** is a managed SaaS with great developer experience and extensive integrations but expensive at scale. **AWS Cognito** integrates natively with AWS services and is cost-effective but has limited customization. **Okta** is enterprise-focused with strong SSO/SAML support but the most expensive.

On our marketplace on AWS, we used a combination approach: AWS Cognito for consumer authentication (cost-effective at 30M user scale, native integration with ALB and Lambda), with federated identity for social logins (Google, Apple). For dealer/enterprise authentication, we integrated with Okta-based corporate IdPs via SAML. Spring Security OAuth2 Resource Server validated tokens from both providers.

The trade-off: Cognito is cheap and AWS-native but its customization for complex flows (custom MFA, progressive profiling) is limited compared to Auth0 or Keycloak. If we needed advanced features like adaptive MFA or breached password detection, Auth0 would be worth the premium. Build vs buy: Keycloak gives full control but you own the uptime and security patching of the IdP itself.
