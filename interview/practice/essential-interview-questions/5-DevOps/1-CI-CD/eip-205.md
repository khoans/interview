## 205. What is Jenkins?

**Jenkins is an open-source automation server for CI/CD — orchestrates builds, tests, and deployments with 1500+ plugins for integrations.**

It's the most widely used CI/CD tool — flexible, extensible, but requires maintenance.

**Key characteristics:**
- Pipeline as code — Jenkinsfile defines build/test/deploy stages
- Plugin ecosystem — integrates with Git, Docker, Kubernetes, AWS, Slack
- Distributed builds — run jobs on multiple agents (Linux, Windows, Mac)
- Self-hosted — you manage the infrastructure, upgrades, backups

**Trade-off:**
- Flexible — customize anything, integrate with everything
- But: maintenance overhead — upgrades break plugins, backup management
- Steep learning curve — Groovy scripting, pipeline syntax, agent management

**Real-world example:**
Our team used manual deployments — Jenkins automated everything: build on commit, run 300 tests, deploy to staging, manual approval → production. Deployment errors: 30% → 5%. But Jenkins server required dedicated maintenance — monthly upgrades, plugin conflicts, disk space management. Jenkins outage during critical deployment — 4-hour downtime. Added backup Jenkins instance, automatic failover. Now running 500+ pipelines, 10,000 builds/month. Maintenance: 1 day/week for upgrades, plugin management.

**Rule of thumb:** Jenkins for complex, customized pipelines with diverse tooling. Consider managed alternatives (GitHub Actions, GitLab CI) for simpler needs — less maintenance. Always have backup Jenkins instance for critical deployments.
