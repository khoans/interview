## 207. What is GitHub Actions?

**GitHub Actions is GitHub's built-in CI/CD platform — workflows triggered by GitHub events (push, pull request, release) with YAML configuration.**

It's CI/CD integrated directly into GitHub — no separate server, minimal setup.

**Key characteristics:**
- YAML workflows — .github/workflows/ci.yml defines jobs
- GitHub-hosted runners — Ubuntu, Windows, Mac runners provided
- Marketplace actions — reusable steps (checkout, setup-node, deploy)
- Tight GitHub integration — status checks, branch protection, release automation

**Trade-off:**
- Zero setup — works out of the box, no server management
- But: vendor lock-in — workflows tied to GitHub, harder to migrate
- Cost at scale — free for public repos, paid minutes for private repos

**Real-world example:**
Our team used Jenkins — maintenance heavy, dedicated engineer 1 day/week. Migrated to GitHub Actions: workflows in YAML, GitHub-hosted runners. Maintenance: 1 day/week → 0. Build times: 10 minutes (Jenkins queue) → 3 minutes (dedicated runners). But costs surprised us — 50,000 minutes/month = $400/month. Optimized: cached dependencies (npm, Maven), parallelized tests, reduced from 50K → 20K minutes. Cost: $400 → $160/month.

**Rule of thumb:** GitHub Actions for GitHub-hosted projects — seamless integration, minimal setup. Self-hosted runners for cost optimization at scale. Cache aggressively — dependencies, build artifacts.
