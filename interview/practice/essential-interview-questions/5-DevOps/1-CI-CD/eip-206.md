## 206. What is a Jenkins pipeline?

**Jenkins pipeline is code that defines the entire build-test-deploy workflow — stages, conditions, parallel execution, all in a Jenkinsfile.**

It's infrastructure as code for CI/CD — version controlled, reviewable, repeatable.

**Key concepts:**
- **Pipeline** — entire workflow from checkout to deployment
- **Stage** — logical grouping (Build, Test, Deploy)
- **Step** — individual command (sh 'mvn test', docker build)
- **Agent** — where pipeline runs (Docker, Kubernetes, specific machine)

**Trade-off:**
- Pipelines as code — review changes, rollback bad pipeline updates
- But: Groovy learning curve — developers must learn pipeline syntax
- Complexity — complex pipelines become hard to maintain

**Real-world example:**
Our Jenkins jobs were click-ops — configured via UI, no version control. Developer deleted production job by mistake — 2 hours to recreate. Converted to pipelines: Jenkinsfile in Git, peer-reviewed, versioned. Pipeline deletion: restore from Git in 5 minutes. But complex pipeline (20 stages, parallel tests, conditional deploys) became 500 lines of Groovy — hard to debug. Refactored: extracted shared libraries for common logic, pipelines down to 50 lines. Maintenance: 500 lines → 50 lines + reusable library.

**Rule of thumb:** Pipeline as code for all jobs — version control, peer review. Keep pipelines under 100 lines — extract shared libraries for common logic. Test pipeline changes in staging first.
