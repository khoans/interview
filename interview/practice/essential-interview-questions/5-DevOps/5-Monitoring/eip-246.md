## 246. What is Grafana?

**Grafana is a visualization and dashboard platform — connects to data sources (Prometheus, Elasticsearch, CloudWatch) and displays metrics, logs, traces.**

It's the window into your system — dashboards for operations, business, executives.

**Key features:**
- Multi-source — query Prometheus, ES, MySQL, InfluxDB in single dashboard
- Alerting — thresholds, notifications (Slack, PagerDuty, email)
- Templating — dynamic dashboards (select service, environment)
- Sharing — public dashboards, snapshots, reports

**Trade-off:**
- Unified view — one dashboard for all metrics, logs, traces
- Beautiful visualizations — executives understand system health
- But: dashboard sprawl — 100+ dashboards, nobody maintains
- Query complexity — bad PromQL crashes dashboards

**Real-world example:**
Our team had 5 monitoring tools — 5 dashboards, different metrics, confusion during incidents. Consolidated to Grafana: single pane of glass — infrastructure (Prometheus), logs (Loki), business metrics (MySQL). Incident response: 5 screens → 1 dashboard. But we created 50 dashboards — nobody knew which was authoritative. Reduced to 5: system health, service health, business metrics, error tracking, capacity planning. Dashboard load time: 10 seconds → 1 second.

**Rule of thumb:** Grafana for all visualization — single source of truth. Limit dashboards — 5-10 per team. Alert from Grafana — single alerting system. Use folders, naming conventions — findable, maintainable.
