## 245. What is Prometheus?

**Prometheus is an open-source monitoring system — time-series database for metrics, pull-based collection, alerting.**

It's the de facto standard for Kubernetes monitoring — cloud-native, scalable, queryable.

**Key characteristics:**
- Pull-based — Prometheus scrapes metrics from endpoints (/metrics)
- Time-series data — timestamped numeric values (CPU, memory, request count)
- PromQL — query language for aggregations, alerting
- Multi-dimensional — metrics labeled (service, environment, instance)

**Trade-off:**
- Cloud-native — Kubernetes integration, service discovery
- Powerful queries — aggregations, joins, alerting rules
- But: pull-only — can't monitor push-only sources without gateway
- Storage limits — not for long-term retention (use Thanos, Cortex)

**Real-world example:**
Our microservices had no monitoring — outages detected by user reports, 30-minute MTTR. Implemented Prometheus: each service exposed /metrics, Prometheus scraped every 15 seconds. Alerting: CPU >90%, error rate >1%, latency p99 >500ms. MTTR: 30 minutes → 5 minutes. But Prometheus storage filled in 7 days — 500GB. Added retention policy (7 days), shipped old data to S3 with Thanos. Now 90-day retention, $50/month S3 cost.

**Rule of thumb:** Prometheus for Kubernetes, microservices metrics. Expose /metrics in all services. Alert on SLOs (error rate, latency), not infrastructure (CPU, memory). Use Thanos/Cortex for long-term storage.
