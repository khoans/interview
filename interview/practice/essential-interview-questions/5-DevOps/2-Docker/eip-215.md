## 215. What is a Docker container?

**A Docker container is a runnable instance of a Docker image — isolated process with its own filesystem, network, and resources.**

It's the execution unit — images are static, containers are dynamic (start, stop, move, delete).

**Key characteristics:**
- Ephemeral — designed to be disposable, stateless
- Isolated — own filesystem, network stack, process tree
- Resource-limited — CPU, memory limits enforced
- Fast lifecycle — start/stop in seconds

**Trade-off:**
- Fast scaling — spin up 10 containers in seconds for traffic spikes
- But: stateless by design — persistent data requires volumes
- Debugging complexity — containers are transient, logs disappear on delete

**Real-world example:**
Our API ran on EC2 instances — scaling took 10 minutes (boot, install, start). Migrated to Docker containers on ECS: scaling 1 → 10 containers took 30 seconds. Black Friday traffic spike: auto-scaled to 50 containers, handled 10x load. Cost: 50 containers × 2 hours = $20 vs. 50 EC2 instances × 24 hours = $200. But initial debugging was hard — containers crashed, logs gone. Added centralized logging (ELK stack), container insights. Debug time: 30 minutes → 5 minutes.

**Rule of thumb:** Containers for stateless applications — web servers, APIs, workers. Use volumes for stateful data (databases, caches). Always configure logging driver — send logs to external system before container dies.
