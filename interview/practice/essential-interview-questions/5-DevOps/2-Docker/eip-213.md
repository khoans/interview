## 213. What is Docker?

**Docker is a containerization platform that packages applications with all dependencies into lightweight, portable containers — run anywhere, consistently.**

It solves "works on my machine" by eliminating environment differences between dev, test, and production.

**Key benefits:**
- Consistency — same container runs on laptop, CI, production
- Isolation — each app has its own dependencies, no conflicts
- Fast startup — containers start in seconds vs. minutes for VMs
- Resource efficiency — share host kernel, no VM overhead

**Trade-off:**
- Fast deployment — seconds to start, stop, scale containers
- But: security isolation weaker than VMs — shared kernel
- Learning curve — Docker concepts, networking, storage management

**Real-world example:**
Our team had 3 environments: dev (Mac), test (Ubuntu), prod (CentOS). Dependency conflicts constant — "works on my Mac" failures in production. Setup for new developers: 2 days. Adopted Docker: single image for all environments. Setup: docker pull + docker run, 10 minutes. Environment bugs: 20/month → 0. But Docker daemon became critical infrastructure — outage blocked all deployments. Added Docker registry mirror, local caching. Zero deployment blocks since.

**Rule of thumb:** Docker for all applications — consistency outweighs learning curve. Use official base images, keep images small, scan for vulnerabilities. Not for high-security isolation needs — use VMs or specialized container security tools.
