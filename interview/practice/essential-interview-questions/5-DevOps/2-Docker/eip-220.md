## 220. What is Docker Compose?

**Docker Compose is a tool for defining and running multi-container applications — single YAML file describes all services, networks, volumes.**

It's local development for microservices — one command starts your entire stack.

**Key features:**
- YAML configuration — docker-compose.yml defines services, networks, volumes
- Single command — docker-compose up starts all containers
- Service discovery — containers find each other by service name
- Environment management — different configs for dev, test, prod

**Trade-off:**
- Simple local development — 10 services start with one command
- But: not for production — use Kubernetes, ECS, or Swarm for orchestration
- Single host — all containers run on same machine

**Real-world example:**
Our microservices app had 8 services: API, auth, users, orders, payments, notifications, PostgreSQL, Redis. Developers manually started each container — 20 commands, wrong networks, configuration drift. Onboarding: 2 days. Adopted Docker Compose: single docker-compose.yml, one command. Onboarding: 30 minutes. But team tried using Compose for production — no auto-scaling, no health checks, single point of failure. Migrated production to Kubernetes, kept Compose for local development.

**Rule of thumb:** Docker Compose for local development, testing, CI environments. Not for production — use proper orchestration (Kubernetes, ECS). Version control docker-compose.yml — single source of truth for local setup.
