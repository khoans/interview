## 224. What is .dockerignore?

**.dockerignore is a file that excludes files and directories from Docker build context — like .gitignore for Docker builds.**

It prevents unnecessary files from being copied into images, slowing down builds, or leaking secrets.

**What to ignore:**
- Version control — .git, .svn (megabytes of history)
- Dependencies — node_modules, __pycache__, .venv (reinstalled in container)
- Build artifacts — target/, dist/, *.log (regenerated in container)
- Secrets — .env, *.pem, credentials (security risk)
- IDE files — .idea/, .vscode/, *.swp

**Trade-off:**
- Faster builds — smaller context to send to Docker daemon
- Smaller images — unnecessary files not copied
- Security — secrets not accidentally included in image
- But: wrong ignores break builds — needed files excluded

**Real-world example:**
Our Node.js app had no .dockerignore. docker build copied everything: node_modules (500MB), .git (200MB), logs (50MB). Build time: 5 minutes (most copying). Image size: 1GB. Developer committed .env with AWS keys — keys ended up in image, pushed to public registry. Security incident: keys rotated, image deleted. Added .dockerignore: node_modules, .git, .env, logs, *.log. Build time: 5 min → 30 seconds. Image size: 1GB → 150MB. Zero secrets in images since.

**Rule of thumb:** Always create .dockerignore on day one. Start with template: node_modules, .git, .env, logs, IDE files. Add as you discover large directories. Treat like .gitignore — version control, team awareness.
