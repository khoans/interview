## 441. What is Schema Registry?

**Schema Registry is a centralized service for managing Avro/Protobuf schemas — ensures compatibility between producers and consumers.**

It's schema governance — prevent breaking changes from breaking consumers.

**Key features:**
- **Schema storage** — versioned schemas per topic
- **Compatibility checking** — reject breaking changes
- **Schema evolution** — add fields, deprecate fields safely
- **Serialization** — producers/consumers fetch schemas on demand

**Trade-off:**
- Schema governance — prevent breaking changes
- But: central SPOF — Schema Registry outage blocks producers
- Compatibility modes — backward, forward, full — choose based on needs

**Real-world example:**
Our team had schema drift — producer added field, 5 consumers broke (didn't handle new field). Incident: 4-hour outage. Implemented Schema Registry — backward compatibility enforced. Breaking changes: rejected at produce time. Zero consumer breakages since. But Schema Registry single point of failure — outage blocked all producers. Added Schema Registry cluster (3 nodes), high availability. Schema Registry outage: eliminated. Also, compatibility mode: backward — new consumers can read old data, old consumers ignore new fields.

**Rule of thumb:** Schema Registry for all Avro/Protobuf — prevent schema drift. Backward compatibility — new producers, old consumers work. Schema Registry cluster — high availability. Test compatibility — CI/CD check before deploy.
