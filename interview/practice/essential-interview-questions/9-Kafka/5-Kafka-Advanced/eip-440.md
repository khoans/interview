## 440. What is KSQL?

**KSQL is SQL-like query language for Kafka — process streams with SQL syntax, no coding required.**

It's for analysts and developers who prefer SQL over Java/Scala.

**Key features:**
- **SQL syntax** — SELECT, JOIN, GROUP BY on streams
- **Real-time** — queries run continuously on streaming data
- **Materialized views** — create tables from streams
- **Push queries** — results pushed as new data arrives

**Trade-off:**
- SQL simplicity — no coding, declarative
- But: limited flexibility — complex logic requires Kafka Streams
- Resource intensive — KSQL cluster separate from Kafka brokers

**Real-world example:**
Our analytics team wanted real-time dashboards — Kafka Streams required Java developers, slow iteration. KSQL: analysts wrote SQL queries, dashboards updated in real-time. Time to insight: 2 weeks (Java dev) → 2 hours (SQL query). But KSQL cluster had 50% CPU overhead — separate cluster needed. Also, complex sessionization logic couldn't be expressed in KSQL — fell back to Kafka Streams for that use case. Hybrid: KSQL for simple aggregations, Kafka Streams for complex logic.

**Rule of thumb:** KSQL for simple aggregations, filters, joins — analysts can self-serve. Kafka Streams for complex logic — sessionization, custom processing. Separate KSQL cluster — don't run on Kafka brokers.
