## 439. What is Kafka Streams?

**Kafka Streams is a stream processing library — process data in real-time from Kafka topics, write results back to Kafka.**

It's application-level stream processing — no separate cluster, just a library.

**Key features:**
- **Stateless processing** — filter, map, transform
- **Stateful processing** — aggregations, joins, windowing
- **Exactly-once** — transactional processing
- **Fault tolerance** — state stored in changelog topics

**Trade-off:**
- Simple deployment — just a library, no cluster
- But: scales with application — need multiple instances for parallelism
- State management — state stores require changelog topics

**Real-world example:**
Our real-time analytics needed sessionization — group user clicks into sessions. Batch processing (Spark): 1-hour latency. Kafka Streams: real-time, sub-second. Session state stored in RocksDB (local state store), changelog to Kafka for fault tolerance. But state store grew unbounded — 100GB per instance. Added state retention — sessions older than 30 days deleted. State size: 100GB → 10GB. Also, exactly-once processing — duplicate clicks deduplicated.

**Rule of thumb:** Kafka Streams for real-time processing — aggregations, joins, sessionization. State retention — prevent unbounded growth. Exactly-once for critical processing — deduplication, accurate counts.
