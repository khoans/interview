## 443. What is dead letter queue in Kafka?

**Dead letter queue (DLQ) is a topic for messages that fail processing — poison pills isolated, main stream continues.**

It's error handling — don't let bad messages block entire stream.

**How it works:**
- Consumer processes message — exception thrown
- Retry N times — still fails
- Send to DLQ topic — message isolated for investigation
- Continue processing — main stream not blocked

**Trade-off:**
- Error isolation — bad messages don't block stream
- But: DLQ management — need process to handle DLQ messages
- Monitoring — alert on DLQ growth

**Real-world example:**
Our order processor crashed on invalid messages — missing required field, deserialization failed. Entire consumer group stuck — 100K messages/hour backlog. Implemented DLQ — invalid messages sent to "orders-dlq" topic after 3 retries. Main stream: unblocked. DLQ: 100 messages/day — investigated, fixed root causes (producer bugs). Also added DLQ monitoring — alert if DLQ > 1000 messages/day. Caught producer regression — 500 invalid messages in 1 hour, alerted, fixed within 30 minutes.

**Rule of thumb:** DLQ for poison pills — messages that fail processing. Retry 3 times — transient failures may recover. Monitor DLQ — alert on growth. Process DLQ — fix root causes, reprocess if fixable.
