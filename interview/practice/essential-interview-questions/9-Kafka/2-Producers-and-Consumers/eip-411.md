## 411. What is a Kafka producer?

**A producer is an application that publishes messages to Kafka topics.**

It's responsible for serialization, partitioning, batching, and delivery guarantees.

**Key responsibilities:**
- **Serialization** — convert objects to bytes (JSON, Avro, Protobuf)
- **Partitioning** — decide which partition message goes to
- **Batching** — group messages for efficiency
- **Delivery guarantees** — configure acks for at-most/at-least/exactly-once

**Trade-off:**
- Batching — higher throughput, but increased latency
- Compression — smaller messages, but CPU overhead
- Acks configuration — durability vs. latency trade-off

**Real-world example:**
Our producer sent messages one-by-one — 10K msg/s, high network overhead. Enabled batching (100 messages or 10ms) — throughput 10K → 100K msg/s. Latency: 1ms → 11ms (batch wait). Acceptable trade-off. But producer had no retry logic — transient broker failure caused message loss. Added retries (3 attempts, exponential backoff). Message loss: 1% → 0%. Also added compression (lz4) — network bandwidth: 100MB/s → 30MB/s.

**Rule of thumb:** Enable batching — 10-100 messages or 10-100ms. Compression for large messages — lz4 or snappy. Configure retries — transient failures shouldn't cause data loss.
