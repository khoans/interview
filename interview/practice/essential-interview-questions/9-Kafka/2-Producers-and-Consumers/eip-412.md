## 412. What is a Kafka consumer?

**A consumer is an application that subscribes to Kafka topics and processes messages.**

It's responsible for deserialization, offset management, and processing logic.

**Key responsibilities:**
- **Subscription** — subscribe to topics, join consumer groups
- **Deserialization** — convert bytes back to objects
- **Offset management** — track progress, commit offsets
- **Processing** — business logic for each message

**Trade-off:**
- Batch processing — process multiple messages, higher throughput
- But: increased latency — wait for batch before processing
- Offset commit frequency — frequent commits = more safety, more overhead

**Real-world example:**
Our consumer processed messages one-by-one — 1K msg/s, high overhead. Switched to batch processing (poll 500 messages) — throughput 1K → 10K msg/s. But offset commit after every batch — if consumer crashed mid-batch, entire batch reprocessed. Added idempotent processing — track processed message IDs, skip duplicates. Reprocessing: 100% → 0% (duplicates detected, skipped). Also, commit offsets after every message within batch — safety without throughput loss.

**Rule of thumb:** Batch processing for throughput — poll 100-500 messages. Idempotent processing — handle duplicates gracefully. Commit offsets frequently — balance safety vs. overhead.
