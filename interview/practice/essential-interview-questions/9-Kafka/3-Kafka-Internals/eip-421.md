## 421. How does Kafka store messages?

**Kafka stores messages in segment files on disk — sequential writes, memory-mapped for reads.**

It's optimized for sequential I/O — append-only logs, not random access.

**Storage structure:**
- **Partition** — directory with multiple segment files
- **Segment** — 1GB files (configurable), rotated periodically
- **Index** — offset-to-position index for fast lookup
- **Log** — actual message data

**Trade-off:**
- Sequential writes — disk throughput near RAM speed
- But: random reads require index — index fits in memory
- Retention — old segments deleted based on policy

**Real-world example:**
Our Kafka broker had 10TB of messages — single partition, single segment. Segment grew unbounded — broker restart took 30 minutes (rebuild index). Fixed: segment.bytes=1GB — segments rotate every 1GB. Restart: 30 minutes → 2 minutes (only active segment indexed). But too many segments — 10,000 segments, index overhead. Optimized: segment.ms=1 hour — time-based rotation + size-based. Segments: 10,000 → 1,000. Performance: optimal.

**Rule of thumb:** Segment size 1-4GB — balance index overhead vs. rotation frequency. Retention policy — delete old segments based on time or size. Monitor disk usage — alert at 70%.
