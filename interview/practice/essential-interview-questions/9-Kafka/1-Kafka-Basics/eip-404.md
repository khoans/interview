## 404. What is a Kafka topic?

**A Kafka topic is a category for messages — producers publish to topics, consumers subscribe to topics.**

It's like a database table — but append-only, ordered, partitioned.

**Key characteristics:**
- Append-only — messages added to end, never modified
- Ordered — messages within partition are ordered
- Durable — messages persisted to disk
- Multi-subscriber — multiple consumers can read same topic

**Trade-off:**
- Durability — messages survive broker restarts
- But: storage grows — need retention policy to delete old messages
- Ordering only within partition — not across partitions

**Real-world example:**
Our order events went to single topic "orders". Worked until 1M orders/day — single topic became bottleneck. Split to "orders-created", "orders-updated", "orders-cancelled". Consumers subscribe only to relevant events. But schema drift — "orders-created" format changed, broke 5 consumers. Added Schema Registry — enforce schema compatibility. Breaking changes: rejected at produce time, not consume time.

**Rule of thumb:** One topic per event type — orders-created, user-signed-up. Partition for parallelism — more partitions = more parallel consumers. Use Schema Registry — prevent schema drift breaking consumers.
