## 410. What is Kafka Connect?

**Kafka Connect is a framework for streaming data between Kafka and external systems — databases, file systems, APIs.**

It's pre-built connectors — no custom code for common integrations.

**Key components:**
- **Source connectors** — import data from external systems to Kafka
- **Sink connectors** — export data from Kafka to external systems
- **Connect workers** — run connectors, manage scaling, fault tolerance

**Trade-off:**
- Pre-built connectors — JDBC, S3, Elasticsearch, MongoDB out of box
- But: connector quality varies — some are community-maintained
- Operational complexity — Connect cluster to manage, monitor

**Real-world example:**
Our team wrote custom code to sync PostgreSQL to Kafka — 2 weeks development, bugs, maintenance burden. Switched to Kafka Connect JDBC Source — 2 hours config, production-ready. CDC (Change Data Capture): database changes → Kafka in real-time. But Connect worker crashed — no monitoring, 4-hour gap in data. Added Connect monitoring (lag, task failures), auto-restart. Data gaps: 4 hours → 0 (auto-restart in 30 seconds).

**Rule of thumb:** Kafka Connect for common integrations — databases, file systems, search engines. Don't write custom code if connector exists. Monitor Connect workers — task failures, lag.
