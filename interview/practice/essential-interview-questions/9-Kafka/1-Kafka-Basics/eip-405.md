## 405. What is a Kafka partition?

**A partition is a subdivision of a topic — topics are split into partitions for parallelism and scalability.**

Each partition is an ordered, immutable log — messages appended sequentially.

**Key characteristics:**
- Ordered — messages within partition are strictly ordered
- Parallel — multiple consumers can read different partitions simultaneously
- Distributed — partitions spread across brokers
- Leader-follower — one leader handles reads/writes, followers replicate

**Trade-off:**
- Parallelism — more partitions = more parallel consumers
- But: ordering only within partition — can't order across partitions
- Too many partitions — broker memory overhead, slower leader election

**Real-world example:**
Our orders topic had 3 partitions — 3 consumers processed in parallel. Black Friday: 10x traffic, consumers lagged 1 hour. Increased to 30 partitions — added 27 consumers. Lag eliminated. But ordering issue — order created then updated went to different partitions, consumer saw update before create. Fixed: partition by order_id — same order always goes to same partition, ordering preserved.

**Rule of thumb:** Partitions = max parallel consumers. Partition by key (order_id, user_id) for ordering. Don't exceed 100 partitions per broker — memory overhead.
