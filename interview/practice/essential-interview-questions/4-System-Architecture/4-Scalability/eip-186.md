## 186. What is CDN (Content Delivery Network)?

**CDN is a geographically distributed network of edge servers that cache static content closer to users — reducing latency and origin server load.**

It's about serving content from the nearest location, not a single data center.

**What CDNs cache:**
- Static assets — images, CSS, JavaScript, fonts
- Media — videos, audio files, large downloads
- Dynamic content — API responses (with short TTL), personalized pages (edge computing)

**Trade-off:**
- Latency reduction — 500ms → 50ms for users far from origin
- Origin offload — 90% of requests served from edge, 10% hit origin
- But: cache invalidation — updating content takes time to propagate
- Cost — CDN pricing based on bandwidth, requests, features

**Real-world example:**
Our e-commerce site: images, CSS, JS served from us-east-1. European users: 2-3 second load times. Implemented CloudFront CDN: assets replicated to 50+ edge locations. European users: served from London/Frankfurt edges. Page load: 3s → 800ms. Conversion rate increased 15%. But image updates took 5 minutes to propagate — old images shown briefly. Added cache invalidation API — critical images (product photos) invalidated on update. Zero customer complaints since.

**Rule of thumb:** CDN for any static content with global users. Long TTL (1 year) for versioned assets (app.v1.2.3.js). Short TTL (5 min) for frequently changing content. Invalidate on updates for critical content.
