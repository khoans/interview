## 194. What is 12-Factor App methodology?

**12-Factor App is a methodology for building cloud-native, scalable applications — codebase, dependencies, config, backing services, build/release/run, processes, port binding, concurrency, disposability, dev/prod parity, logs, admin processes.**

It's the blueprint for modern SaaS applications that scale.

**Key factors:**
- **One codebase, many deploys** — same code in dev, staging, prod
- **Dependencies explicitly declared** — no implicit system-wide dependencies
- **Config in environment** — no config in code, different per environment
- **Backing services as attached resources** — DB, cache, queue are pluggable
- **Stateless processes** — no sessions on server, everything in backing services
- **Logs as streams** — app doesn't manage logs, just writes to stdout

**Trade-off:**
- Cloud-ready — deploy anywhere, scale easily
- But: discipline required — no shortcuts, no "just this once"
- Learning curve — teams used to traditional apps need retraining

**Real-world example:**
Our legacy app: config in database, sessions in memory, logs to files, manual deployments. Moving to cloud: 3 months of refactoring. Applied 12-Factor: config to environment variables, sessions to Redis, logs to stdout, Docker containers. Deployment: 2 hours → 10 minutes. Scaling: manual (4 hours) → auto-scaling (instant). Dev/prod parity: 80% different → 95% same. Bugs in production dropped 70% — "works on my machine" eliminated.

**Rule of thumb:** 12-Factor for new cloud applications. For legacy apps, factor by factor migration — start with config, then logs, then statelessness. Don't boil the ocean — pick 2-3 factors per sprint.
