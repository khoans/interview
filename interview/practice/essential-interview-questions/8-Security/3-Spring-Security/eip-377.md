## 377. What is AuthenticationManager?

**AuthenticationManager is Spring Security's authentication orchestrator — delegates to AuthenticationProviders, handles authentication flow.**

It's the entry point for authentication — receives authentication request, returns authenticated user.

**How it works:**
- Receives Authentication token (username/password)
- Delegates to AuthenticationProvider (DaoProvider, LDAP, OAuth2)
- Provider validates credentials, returns authenticated Authentication
- AuthenticationManager sets security context

**Trade-off:**
- Pluggable — multiple providers, chain of responsibility
- Centralized — one place to configure authentication
- But: abstraction layer — debugging requires understanding provider chain
- Default config may not fit — custom providers needed for custom auth

**Real-world example:**
Our app needed multi-source authentication — users in database OR LDAP. Default AuthenticationManager: one provider. Configured ProviderManager with two providers — DaoAuthenticationProvider (database), LdapAuthenticationProvider. Authentication: database first, if not found, try LDAP. Users: seamless, don't know which source. But LDAP was slow (2 seconds) — database users experienced delay. Added provider order — database first (fast), LDAP only if not in database. Average auth time: 2s → 100ms.

**Rule of thumb:** AuthenticationManager for orchestrating multiple providers. Order providers by speed — fast providers first. Custom providers for non-standard authentication (JWT, API keys).
