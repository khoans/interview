## 381. What is method security?

**Method security is authorization at method level — not just URL-based, but service/method-level access control.**

It's defense in depth — even if URL security is bypassed, method security catches unauthorized access.

**How it works:**
- Annotations on methods — @PreAuthorize, @Secured, @PostAuthorize
- Spring Security intercepts method calls
- Evaluates authorization expression
- Allows or denies method execution

**Trade-off:**
- Fine-grained — access control based on method arguments, return values
- Defense in depth — multiple layers of security
- But: performance overhead — AOP interception on every call
- Complexity — security logic scattered across methods

**Real-world example:**
Our app had URL-based security only — /users/{id} required authentication. But service methods had no security — internal calls bypassed URL security. Bug: controller called service directly, no authentication check. Added method security — @PreAuthorize on all service methods. Internal calls: now secured. Unauthorized access: eliminated. But 200 service methods to annotate — 2 days effort. Prioritized: critical methods first (user data, payments), then rest.

**Rule of thumb:** Method security for all service methods — not just controllers. URL security + method security — defense in depth. Start with critical methods — payments, user data, admin operations.
