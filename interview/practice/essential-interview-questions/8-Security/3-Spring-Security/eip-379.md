## 379. What is @PreAuthorize?

**@PreAuthorize is method-level security annotation — SpEL expression evaluated before method execution, access denied if false.**

It's fine-grained authorization — not just URL-based, but method-level.

**How it works:**
- Annotation on method — @PreAuthorize("hasRole('ADMIN')")
- Spring Security evaluates expression before method call
- If true — method executes
- If false — AccessDeniedException thrown

**Trade-off:**
- Fine-grained — access control based on user, resource, context
- SpEL flexibility — complex expressions (user matches resource owner)
- But: SpEL learning curve — syntax, available variables
- Performance — expression evaluation on every call (1-5ms overhead)

**Real-world example:**
Our API had URL-based security — /admin/** requires ADMIN role. But resource-level security missing — any ADMIN could access any user's data. Added @PreAuthorize on service methods — @PreAuthorize("#userId == authentication.principal.id or hasRole('ADMIN')"). Users can access own data, admins access all. Unauthorized access: eliminated. But SpEL expressions were complex — 50+ characters, hard to read. Extracted to custom annotations — @IsOwnerOrAdmin. Readability: improved. Functionality: same.

**Rule of thumb:** @PreAuthorize for resource-level security — not just role-based. Keep expressions simple — extract complex logic to custom annotations. Test security expressions — don't assume they work.
