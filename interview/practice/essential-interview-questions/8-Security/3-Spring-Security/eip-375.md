## 375. What is Spring Security?

**Spring Security is a security framework for Spring applications — authentication, authorization, attack protection.**

It's the standard for Spring Boot security — comprehensive, configurable, widely adopted.

**Key features:**
- **Authentication** — username/password, OAuth2, LDAP, SAML
- **Authorization** — role-based, method-level, URL-based
- **Attack protection** — CSRF, XSS, clickjacking, session fixation

**Trade-off:**
- Comprehensive — one framework for all security needs
- Spring integration — seamless with Spring Boot
- But: complexity — steep learning curve, verbose configuration
- Over-engineering risk — default config may be more than needed

**Real-world example:**
Our Spring Boot app had custom security — manual session management, role checks in every controller. Bug: one controller missed role check, unauthorized access. Migrated to Spring Security — centralized security config, method-level authorization with @PreAuthorize. Unauthorized access: eliminated. But initial config was complex — 500 lines of security config. Simplified: Spring Boot auto-configuration + minimal overrides. Config: 500 lines → 50 lines. Functionality: same.

**Rule of thumb:** Spring Security for Spring applications — don't roll your own. Use Spring Boot auto-configuration — minimal overrides. Method-level security (@PreAuthorize) — not just URL-based.
