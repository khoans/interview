## 383. What is the flow of Spring Security?

**Spring Security flow: request → filter chain → authentication → authorization → resource access.**

It's a pipeline — each stage processes request, passes to next stage.

**Flow steps:**
1. **Request arrives** — SecurityFilterChain intercepts
2. **Authentication** — UsernamePasswordAuthenticationFilter extracts credentials
3. **AuthenticationManager** — validates credentials against UserDetailsService
4. **SecurityContext** — authenticated user stored in context
5. **Authorization** — FilterSecurityInterceptor checks access rules
6. **Resource access** — if authorized, request reaches controller

**Trade-off:**
- Modular — each stage has single responsibility
- Customizable — replace any stage with custom implementation
- But: complexity — debugging requires understanding full flow
- Performance — 10+ filters, 5-10ms overhead per request

**Real-world example:**
Our app had authentication working, but authorization failed — all requests allowed. Debug: traced flow. Found FilterSecurityInterceptor missing from chain — authorization stage skipped. Added interceptor — authorization working. Also, SecurityContextPersistenceFilter loaded context from session — but session expired, context null. Added re-authentication logic — expired session → redirect to login. User experience: seamless re-login, not "access denied" errors.

**Rule of thumb:** Understand full flow — authentication before authorization. Debug with filter logging — each filter logs execution. Custom filters for non-standard flows — JWT, API keys.
