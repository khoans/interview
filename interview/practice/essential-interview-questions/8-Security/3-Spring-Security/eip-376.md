## 376. What is SecurityFilterChain?

**SecurityFilterChain is Spring Security's core mechanism — chain of filters processing every request for authentication and authorization.**

It's the request processing pipeline — each filter handles specific security concern.

**Key filters:**
- **SecurityContextPersistenceFilter** — loads security context from session
- **UsernamePasswordAuthenticationFilter** — processes login forms
- **BasicAuthenticationFilter** — processes Basic auth headers
- **FilterSecurityInterceptor** — enforces authorization rules

**Trade-off:**
- Modular — each filter has single responsibility
- Customizable — add, remove, reorder filters
- But: complexity — filter order matters, debugging requires understanding chain
- Performance — each filter adds overhead (1-2ms per filter)

**Real-world example:**
Our app needed custom JWT authentication — not supported by default filters. Added custom filter before UsernamePasswordAuthenticationFilter — extracts JWT, sets security context. JWT auth: working. But filter order was wrong — ran after authorization filter, security context not set. Moved filter earlier in chain — before authorization. Auth: working. Also, added logging filter to debug chain — each filter logs execution time. Found one filter taking 50ms (database lookup). Optimized: cached result, 50ms → 1ms.

**Rule of thumb:** Custom filters for non-standard auth (JWT, API keys). Order matters — authentication before authorization. Log filter execution — identify performance bottlenecks.
