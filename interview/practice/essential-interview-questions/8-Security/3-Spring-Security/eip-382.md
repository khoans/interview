## 382. What is BCrypt password encoder?

**BCrypt is a password hashing function — slow by design, salted, adaptive (cost factor increases over time).**

It's the gold standard for password storage — resistant to brute force, rainbow tables.

**Key characteristics:**
- **Salted** — random salt per password — no rainbow table attacks
- **Slow** — intentionally slow (100ms per hash) — brute force expensive
- **Adaptive** — cost factor increases — stays slow as hardware improves

**Trade-off:**
- Secure — slow hashing makes brute force impractical
- Salted — identical passwords have different hashes
- But: slow — 100ms per hash impacts login performance
- Cost factor tuning — too low (insecure), too high (slow login)

**Real-world example:**
Our app stored passwords with MD5 — fast, unsalted. Breach: 100K passwords cracked in 1 hour (rainbow tables). Migrated to BCrypt — cost factor 10 (100ms per hash). Password cracking: 1 hour → 1000 years (estimated). But login slowed — 1ms (MD5) → 100ms (BCrypt). User complaints: "slow login". Tuned cost factor to 12 (200ms) — acceptable trade-off. Also, added progress indicator — users see "verifying password..." during hash.

**Rule of thumb:** BCrypt for all password storage — never MD5, SHA1, or plain text. Cost factor 10-12 — balances security and performance. Progress indicator for slow hashing — user experience.
