## 357. What is JWT (JSON Web Token)?

**JWT is a compact token format for securely transmitting claims between parties — JSON data with digital signature.**

It's self-contained — token contains all necessary information, no database lookup required.

**Key characteristics:**
- **Stateless** — server doesn't store session, token contains all data
- **Signed** — HMAC or RSA signature prevents tampering
- **Compact** — small enough for HTTP headers, URLs

**Trade-off:**
- No session storage — scales horizontally without session sync
- Fast validation — verify signature, no database query
- But: can't revoke easily — token valid until expiration
- Size limits — don't store large data in token (keep under 4KB)

**Real-world example:**
Our API used session-based auth — server stored sessions in Redis. Scaling issue: 1M users = 1M sessions, Redis memory full. Switched to JWT — user ID, roles, expiration in token. Server storage: 1M sessions → 0. But token revocation issue — user logged out, token still valid for 1 hour. Added token blacklist (Redis) for logged-out users. Memory: 1M sessions → 10K blacklisted tokens (only active logouts).

**Rule of thumb:** JWT for stateless authentication, microservices. Keep payload small — user ID, roles, expiration only. Implement token blacklist for logout functionality.
