## 352. What is OAuth2?

**OAuth2 is an authorization framework — allows third-party applications to access user resources without sharing passwords.**

It's about delegated access — user grants limited access to third parties.

**Key roles:**
- **Resource Owner** — the user who owns the data
- **Client** — application requesting access
- **Resource Server** — API hosting protected resources
- **Authorization Server** — issues access tokens

**Trade-off:**
- No password sharing — users don't give passwords to third parties
- Limited scope — grant read-only access, not full access
- But: complexity — 4 roles, multiple flows, token management
- Token security — stolen tokens grant access until expiration

**Real-world example:**
Our app allowed "Login with Google" — OAuth2 authorization code flow. Users didn't create passwords — Google handles authentication. Signup conversion: 30% → 60% (no password to remember). But token expiration caused issues — access tokens expired after 1 hour, users logged out mid-session. Added refresh tokens — silent re-authentication, seamless experience. Token theft concern: short-lived access tokens (1 hour), long-lived refresh tokens (30 days) with rotation.

**Rule of thumb:** OAuth2 for third-party access, SSO. Use authorization code flow for web apps, PKCE for mobile/SPA. Short-lived access tokens, rotating refresh tokens.
