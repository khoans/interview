## 354. What is OAuth2 implicit flow?

**Implicit flow returns access token directly in browser — no authorization code exchange. Deprecated for security reasons.**

It was designed for SPAs (single-page applications) — no server-side component to exchange codes.

**How it worked:**
1. User clicks "Login" — redirected to authorization server
2. User authenticates at provider
3. Provider redirects back with access token in URL fragment
4. SPA extracts token from URL — uses token directly

**Why deprecated:**
- Token exposed in URL — browser history, server logs, referer headers
- No client secret — anyone can use stolen token
- Token theft via XSS — JavaScript can access token

**Real-world example:**
Our SPA used implicit flow — tokens in URL, stored in localStorage. XSS vulnerability stole tokens — attacker accessed 1000 user accounts. Migrated to authorization code flow with PKCE — code exchanged server-side, tokens never in browser. Token theft: 100% → 0%. But required backend changes — added proxy endpoint for token exchange. Development effort: 3 days. Security improvement: worth it.

**Rule of thumb:** Don't use implicit flow — deprecated since 2017. Use authorization code flow with PKCE for SPAs/mobile. PKCE replaces client secret for public clients.
