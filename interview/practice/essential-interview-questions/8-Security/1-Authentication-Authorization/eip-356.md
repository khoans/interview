## 356. What is OpenID Connect (OIDC)?

**OpenID Connect is OAuth2 extension for authentication — adds identity layer on top of OAuth2 authorization.**

OAuth2 answers "what can you access?" OIDC answers "who are you?"

**Key additions over OAuth2:**
- **ID Token** — JWT containing user identity (name, email, picture)
- **UserInfo endpoint** — get additional user profile data
- **Standard claims** — consistent user data across providers

**Trade-off:**
- Standardized authentication — same flow for Google, GitHub, Microsoft
- User profile data — email, name, picture without separate API calls
- But: more complexity — ID token validation, claim mapping
- Provider differences — not all providers support all claims

**Real-world example:**
Our app had custom authentication — username/password, user profiles in database. Migrated to OIDC with Google, GitHub providers. Users: "Login with Google" instead of creating passwords. Signup: 30% → 60%. User profile: ID token has email, name, picture — no separate profile API call. But claim mapping varied — Google uses "email", GitHub uses "emails[0].value". Added claim normalization layer — consistent user object regardless of provider.

**Rule of thumb:** OIDC for user authentication with third-party providers. Validate ID token signature — don't trust token without verification. Normalize claims — different providers use different claim names.
