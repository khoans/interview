## 362. What is LDAP?

**LDAP is a protocol for accessing directory services — query user information, authenticate against central directory.**

It's the enterprise standard for user directories — Active Directory is LDAP-compatible.

**Common uses:**
- **Authentication** — verify username/password against directory
- **User lookup** — get user attributes (email, department, manager)
- **Group membership** — check if user is in specific group

**Trade-off:**
- Centralized user management — one directory for all systems
- Enterprise standard — integrates with Active Directory
- But: complex protocol — ASN.1 encoding, DN (Distinguished Names)
- Legacy — newer systems prefer REST APIs, OIDC

**Real-world example:**
Our enterprise app integrated with customer LDAP (Active Directory). Users authenticate with corporate credentials — no separate accounts. But LDAP queries were complex — DN: "CN=John Doe,OU=Users,DC=company,DC=com". One typo, authentication fails. Added LDAP library with builder pattern — simplified DN construction. Authentication failures: 20% (typos) → 1% (invalid credentials only). Also, LDAP connection pooling — 1000 auth requests/second overwhelmed LDAP server. Added connection pool (10 connections), queue excess requests.

**Rule of thumb:** LDAP for enterprise Active Directory integration. Use LDAP libraries — don't construct DNs manually. Connection pooling essential — LDAP servers have connection limits.
