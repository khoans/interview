## 361. What is SAML?

**SAML is an XML-based standard for SSO — Security Assertion Markup Language, enterprise SSO before OIDC.**

It's the older SSO standard — still widely used in enterprise, government.

**Key components:**
- **Identity Provider (IdP)** — authenticates users, issues SAML assertions
- **Service Provider (SP)** — application accepting SAML assertions
- **SAML Assertion** — XML document with user identity, attributes

**Trade-off:**
- Enterprise standard — widely supported (Active Directory, Okta)
- Rich attributes — user roles, departments in assertion
- But: XML complexity — verbose, harder to parse than JWT
- Legacy — newer systems prefer OIDC/OAuth2

**Real-world example:**
Our enterprise app required SAML integration — customers used Active Directory. Implemented SAML SSO — users login with corporate credentials, no separate accounts. Support tickets: "forgot password" eliminated for enterprise customers. But SAML assertions were 10KB+ (XML verbosity) — vs. 1KB JWT. Network overhead: 10x. Parsing: 5ms (JWT) → 20ms (SAML). Acceptable for enterprise contracts, but OIDC preferred for new integrations.

**Rule of thumb:** SAML for enterprise SSO, Active Directory integration. OIDC for new implementations, mobile/SPA. Support both if serving enterprise + modern customers.
