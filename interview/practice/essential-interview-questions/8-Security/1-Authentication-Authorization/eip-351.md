## 351. What is authentication vs authorization?

**Authentication verifies who you are. Authorization verifies what you're allowed to do.**

Authentication comes first — you can't authorize an unknown user.

**Authentication:**
- Verifies identity — username/password, biometrics, security token
- Answers: "Who are you?"
- Example: Login with email and password

**Authorization:**
- Verifies permissions — admin, user, read-only access
- Answers: "What can you do?"
- Example: Admin can delete users, regular users cannot

**Trade-off:**
- Authentication: security focus — prevent unauthorized access
- Authorization: business logic — enforce access control policies
- Both required — authentication without authorization exposes everything

**Real-world example:**
Our API had authentication (JWT tokens) but weak authorization — any authenticated user could access admin endpoints by guessing URLs. Security audit: critical vulnerability. Added role-based authorization — @PreAuthorize("hasRole('ADMIN')"). Unauthorized access attempts: 100/day → 0. But over-authorized initially — gave all users admin access in dev, forgot to remove in production. Added security tests verifying authorization rules.

**Rule of thumb:** Always implement both — authentication gates access, authorization gates actions. Test authorization rules — don't assume authentication is enough.
