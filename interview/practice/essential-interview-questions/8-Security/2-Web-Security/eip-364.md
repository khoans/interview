## 364. What is SSL/TLS?

**SSL/TLS is cryptographic protocol for secure communication — SSL is deprecated, TLS is current standard.**

TLS ensures confidentiality (encryption), integrity (tamper detection), and authentication (server identity).

**Versions:**
- **SSL 2.0/3.0** — deprecated, vulnerable (POODLE attack)
- **TLS 1.0/1.1** — deprecated, vulnerable
- **TLS 1.2** — current minimum, widely supported
- **TLS 1.3** — latest, faster (1-RTT handshake), more secure

**Trade-off:**
- TLS 1.3: faster handshake, better security — but older clients don't support
- TLS 1.2: universal support — but slower, older cryptography
- Disabling old versions: better security — but legacy clients break

**Real-world example:**
Our server supported TLS 1.0 for legacy clients. Security scan: vulnerable to BEAST attack. Disabled TLS 1.0/1.1, minimum TLS 1.2. 500 legacy clients (old Android apps) broke — couldn't connect. Support tickets: 200/day. Compromise: TLS 1.2 minimum for web, TLS 1.0 for mobile API (with extra monitoring). 6 months later, forced mobile app update — TLS 1.2 minimum everywhere. Vulnerability: eliminated. Legacy support: 6-month migration window.

**Rule of thumb:** TLS 1.2 minimum, TLS 1.3 preferred. Disable SSL 2.0/3.0, TLS 1.0/1.1 — known vulnerabilities. Plan legacy client migration — don't support insecure protocols indefinitely.
