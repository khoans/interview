## 371. What is SQL Injection?

**SQL Injection is an attack where attacker manipulates SQL queries by injecting malicious SQL through user input.**

It's about code injection — attacker's SQL executes against your database.

**How it works:**
- User input concatenated into SQL query — string concatenation
- Attacker enters: `' OR '1'='1` — query logic changed
- Result: authentication bypass, data extraction, data modification

**Impact:**
- Data breach — extract entire database
- Authentication bypass — login without credentials
- Data modification — update/delete records

**Trade-off:**
- Any unsanitized input is vulnerable — search, login, forms
- Legacy code often uses string concatenation — high risk
- Prevention required — parameterized queries, ORM

**Real-world example:**
Our legacy login query: `SELECT * FROM users WHERE username = '` + input + `' AND password = '` + hash + `'`. Attacker entered username: `admin' --`. Query became: `SELECT * FROM users WHERE username = 'admin' --' AND password = '...'`. Comment ignored password check. Attacker logged in as admin. Fixed: parameterized queries — `SELECT * FROM users WHERE username = ? AND password = ?`. SQL injection: eliminated. But 50 queries to refactor — 2 weeks effort. Security improvement: worth it.

**Rule of thumb:** Parameterized queries for all SQL — never string concatenation. ORM (Hibernate, JPA) uses parameters by default — but watch for native queries. Regular security scans — SQL injection detection.
