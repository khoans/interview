## 373. What is Content Security Policy (CSP)?

**CSP is a security header restricting resource loading — where scripts, styles, images can load from.**

It's defense in depth — even if XSS exists, CSP limits damage.

**How it works:**
- Server sends Content-Security-Policy header — allowed sources
- Browser enforces — blocks resources from non-allowed sources
- Example: `script-src 'self'` — only scripts from same origin

**Trade-off:**
- Strong XSS protection — inline scripts blocked, external sources limited
- But: misconfiguration breaks legitimate functionality
- Reporting mode available — test before enforcing

**Real-world example:**
Our app had multiple third-party scripts — analytics, ads, widgets. Implemented CSP: script-src 'self' https://analytics.com. Broke: ads (5 different domains), widgets (3 domains). Added all domains to CSP — 10 domains total. But CSP blocked inline script — "onclick" handlers didn't work. Refactored: inline handlers → addEventListener. CSP: enforced. Also enabled report-uri — collected violations for 2 weeks. Found 3 missing domains — added before enforcement. Zero broken functionality.

**Rule of thumb:** Start with CSP report-only — collect violations, fix before enforcing. Allow specific domains — not wildcard (*). Block inline scripts — 'unsafe-inline' defeats purpose. Regular violation monitoring — new third-party scripts may break.
