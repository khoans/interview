## 369. What are the types of XSS?

**Three types of XSS: Reflected (immediate), Stored (persistent), DOM-based (client-side).**

Different attack vectors, same impact — malicious scripts execute in victim's browser.

**Reflected XSS:**
- Input immediately reflected in response — search query, error message
- Requires user action — click malicious link
- Example: search?q=<script>alert('XSS')</script>

**Stored XSS:**
- Input stored in database — comments, profiles, forums
- Automatic execution — any user viewing stored content
- Example: comment with malicious script, all viewers affected

**DOM-based XSS:**
- Client-side vulnerability — JavaScript processes input unsafely
- Server never sees payload — all in browser
- Example: location.hash passed to innerHTML without encoding

**Trade-off:**
- Reflected: easier to prevent — encode output, no storage
- Stored: more dangerous — affects all viewers, persistent
- DOM-based: hardest to detect — server-side scanning misses it

**Real-world example:**
Our app had all three types. Reflected: search page echoed query without encoding. Fixed: output encoding. Stored: user profile allowed script tags. Fixed: input sanitization + output encoding. DOM-based: JavaScript copied URL parameter to page. Fixed: use textContent, not innerHTML. Security audit: 10 XSS vulnerabilities → 0. Quarterly scans now — catch regressions early.

**Rule of thumb:** Output encoding prevents reflected/stored XSS. Input sanitization for stored data. DOM XSS prevention: safe JavaScript APIs (textContent, not innerHTML). Regular security scans — automated + manual.
