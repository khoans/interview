## 370. How do you prevent XSS attacks?

**XSS prevention: output encoding, input sanitization, Content Security Policy — multiple layers of defense.**

No single defense is perfect — defense in depth.

**Prevention methods:**
- **Output encoding** — HTML entities for HTML context, URL encoding for URLs
- **Input sanitization** — whitelist allowed tags/attributes, strip everything else
- **Content Security Policy (CSP)** — restrict script sources, inline scripts disabled

**Trade-off:**
- Output encoding: most effective — but must match context (HTML, JavaScript, URL)
- Input sanitization: good for stored data — but whitelist maintenance required
- CSP: strong additional layer — but misconfiguration breaks legitimate scripts

**Real-world example:**
Our app implemented output encoding — all user input encoded as HTML entities. XSS attacks: blocked. But encoding broke functionality — user couldn't post code snippets with <code> tags. Added input sanitization — whitelist <code>, <pre>, <b>, <i>. Code snippets: working. XSS: still blocked. Also added CSP — report-only mode first, monitored violations for 2 weeks. Fixed violations, then enforced. CSP blocked one legitimate analytics script — added to allowed sources.

**Rule of thumb:** Output encoding as primary defense — context-aware (HTML, JS, URL). Input sanitization for rich text — whitelist, don't blacklist. CSP in report-then-enforce mode — don't break production.
