## 366. What is CSRF (Cross-Site Request Forgery)?

**CSRF is an attack where malicious site tricks authenticated user into performing unwanted actions — user is logged in, site submits hidden form.**

It exploits trust — site trusts user's browser, doesn't verify intent.

**How it works:**
1. User logs into bank.com — authenticated session
2. User visits evil.com — has hidden form: POST bank.com/transfer
3. Browser automatically sends bank.com cookies with request
4. Bank processes transfer — thinks it's user's intentional action

**Trade-off:**
- Attack exploits browser behavior — cookies sent automatically
- User unaware — action happens without their knowledge
- Defense required — CSRF tokens, SameSite cookies, origin checking

**Real-world example:**
Our e-commerce had no CSRF protection. Attacker created page with hidden form: POST /api/change-address. Victim visited page while logged in — address changed to attacker's address. Order shipped to attacker. Financial loss: $50K. Added CSRF tokens — unique per session, required for state-changing operations. Attacker can't guess token — attack blocked. But CSRF tokens broke third-party integrations — added separate API key authentication for integrations (no CSRF token required).

**Rule of thumb:** CSRF tokens for all state-changing operations (POST, PUT, DELETE). SameSite cookies — prevent cross-site cookie sending. API authentication (OAuth, API keys) exempt from CSRF — different threat model.
