## 389. What is output encoding?

**Output encoding converts data to safe format for specific context — HTML encoding for HTML, URL encoding for URLs, JavaScript encoding for JavaScript.**

It's XSS prevention — user input displays as text, doesn't execute as code.

**Context-specific encoding:**
- **HTML** — < becomes &lt;, > becomes &gt;
- **JavaScript** — quotes escaped, newlines encoded
- **URL** — spaces become %20, special characters encoded
- **CSS** — special characters escaped

**Trade-off:**
- Prevents XSS — scripts display as text, don't execute
- Context matters — HTML encoding doesn't prevent XSS in JavaScript context
- But: over-encoding breaks functionality — rich text needs some HTML
- Performance — encoding adds overhead (minimal, but measurable)

**Real-world example:**
Our app HTML-encoded all output — XSS blocked. But users couldn't post formatted text — <b>bold</b> displayed as &lt;b&gt;bold&lt;/b&gt;. Added rich text support — whitelist <b>, <i>, <a> tags, encode everything else. Rich text: working. XSS: still blocked (script tags encoded). Also, JavaScript context had XSS — user input in onclick handler. Added JavaScript encoding — quotes escaped, scripts don't break out. XSS vectors: 5 → 0.

**Rule of thumb:** Encode output for specific context — HTML, JavaScript, URL, CSS. Whitelist allowed tags for rich text — encode everything else. Use framework encoding — don't roll your own (easy to miss edge cases).
