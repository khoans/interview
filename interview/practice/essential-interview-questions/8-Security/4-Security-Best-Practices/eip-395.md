## 395. What is two-factor authentication (2FA)?

**2FA requires two forms of authentication — something you know (password) + something you have (phone, token).**

It's defense in depth — compromised password isn't enough for access.

**Common factors:**
- **Knowledge** — password, PIN (something you know)
- **Possession** — phone (SMS, authenticator app), hardware token (something you have)
- **Inherence** — fingerprint, face recognition (something you are)

**Trade-off:**
- Strong security — password breach doesn't compromise account
- But: user friction — extra step for every login
- Adoption: users may skip 2FA if optional

**Real-world example:**
Our app had password-only auth. Credential stuffing attack — 1000 accounts compromised (users reused passwords from other breaches). Implemented 2FA — TOTP (Google Authenticator). Compromised accounts: attacker had password, but no TOTP code. Access blocked. Adoption: 60% (optional). Made mandatory for admin accounts — 100% adoption. Admin compromise: 50 → 0. But support tickets increased — users lost phone, couldn't login. Added backup codes — print and store safely. Support tickets: reduced 50%.

**Rule of thumb:** 2FA for all users — optional with incentives, mandatory for admins. TOTP (authenticator app) — more secure than SMS. Backup codes — recovery when phone lost.
