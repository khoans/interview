## 398. What is penetration testing?

**Penetration testing is authorized simulated attack — ethical hackers try to breach your systems.**

It's adversarial testing — find vulnerabilities before real attackers do.

**Types:**
- **Black box** — no prior knowledge, simulates external attacker
- **White box** — full knowledge (architecture, source code), thorough
- **Gray box** — partial knowledge, balanced approach

**Trade-off:**
- Real-world testing — finds vulnerabilities automated tools miss
- Human creativity — attackers think outside the box
- But: point-in-time — snapshot, vulnerabilities introduced after test not caught
- Cost — $10K-$100K per engagement

**Real-world example:**
Our app passed all automated scans — no vulnerabilities. Penetration test: 3 days, found 5 critical vulnerabilities (business logic flaws, race conditions). Automated tools: missed all 5 (not signature-based). Fixed vulnerabilities before production. Breach prevented: estimated $1M+ (customer data, reputation). Now: annual pen tests, plus after major releases. Cost: $50K/year. Breach cost avoided: 20x ROI. Also, developers observed pen test — learned attacker mindset. Code quality: improved.

**Rule of thumb:** Annual penetration testing — plus after major releases. White box for thoroughness — source code access. Developers observe — learning opportunity. Fix findings before production.
