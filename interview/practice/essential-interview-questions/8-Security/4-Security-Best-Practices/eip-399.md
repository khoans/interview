## 399. What is OWASP Top 10?

**OWASP Top 10 is a list of most critical web application security risks — updated every 3-4 years based on real-world data.**

It's the industry standard — prioritize security efforts on most common, most impactful vulnerabilities.

**Current Top 10 (2021):**
1. Broken Access Control
2. Cryptographic Failures
3. Injection
4. Insecure Design
5. Security Misconfiguration
6. Vulnerable Components
7. Authentication Failures
8. Software/Data Integrity Failures
9. Logging/Monitoring Failures
10. SSRF

**Trade-off:**
- Prioritized — focus on most critical risks
- Industry standard — common language for security
- But: not exhaustive — other vulnerabilities exist
- Generic — may not apply to all applications

**Real-world example:**
Our security program was ad-hoc — fixed whatever seemed urgent. Adopted OWASP Top 10 as framework — quarterly reviews against each category. Found: #1 Broken Access Control (users accessing other users' data), #3 Injection (SQL injection in legacy code), #6 Vulnerable Components (outdated libraries). Fixed top 3 — 80% risk reduction. Now: OWASP Top 10 checklist for every release. Vulnerabilities in production: 10/quarter → 1/quarter.

**Rule of thumb:** OWASP Top 10 as security framework — quarterly reviews. Checklist for releases — verify each category. Training for developers — understand each vulnerability type.
