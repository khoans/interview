## 397. What is security audit?

**Security audit is systematic evaluation of security controls — policies, procedures, technical controls.**

It's about finding gaps — what's secure, what's vulnerable, what's missing.

**Audit types:**
- **Internal** — self-assessment, continuous monitoring
- **External** — third-party auditors, compliance (SOC2, PCI-DSS)
- **Automated** — security scanning tools (SAST, DAST)
- **Manual** — penetration testing, code review

**Trade-off:**
- Finds vulnerabilities — before attackers do
- Compliance — required for certifications
- But: cost — external audits $50K-$500K
- Frequency — annual audits miss issues between audits

**Real-world example:**
Our startup had no security audits — "we're too small". Breach: SQL injection, 10K records exposed. Cost: $500K (remediation, legal, reputation). Post-breach audit: 50 vulnerabilities found. Fixed top 10 — 90% risk reduction. Now: quarterly internal audits, annual external. Cost: $50K/year. Breach cost avoided: $500K+. ROI: clear. Also, automated scanning — weekly SAST/DAST. Vulnerabilities caught: within days, not months.

**Rule of thumb:** Regular audits — internal quarterly, external annually. Automated scanning — weekly. Prioritize findings — fix critical first, then high. Track remediation — don't let findings linger.
