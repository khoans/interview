## 124. What are ACID properties? Explain each.

**ACID is Atomicity, Consistency, Isolation, Durability — the four guarantees databases provide for reliable transactions.**

Break any of these, and you risk data corruption that's hard to recover from.

**Atomicity — all or nothing:**
- All operations in transaction commit, or all roll back
- No partial commits — database never in half-updated state
- Implemented via rollback logs — undo incomplete transactions

**Consistency — valid state to valid state:**
- Constraints enforced (foreign keys, unique, check constraints)
- Business rules preserved — account balance never negative (if constrained)
- Application + database together ensure consistency

**Isolation — concurrent transactions don't interfere:**
- Transaction A doesn't see uncommitted changes from Transaction B
- Controlled by isolation levels (READ_COMMITTED, SERIALIZABLE, etc.)
- Higher isolation = more consistency, lower concurrency

**Durability — committed data survives crashes:**
- Once committed, data is on disk (write-ahead log)
- Power failure after commit — data still there on restart
- Replication adds durability across multiple machines

**Trade-off:**
- Full ACID compliance costs performance — 20-50% slower than non-ACID systems
- NoSQL databases often relax ACID for scale (BASE model)
- Distributed systems struggle with ACID — CAP theorem limits

**Real-world example:**
Our inventory system had race conditions — two orders could reserve same item. No isolation. Fixed with SERIALIZABLE transactions — zero oversells. But checkout slowed 3x, timeouts increased. Compromise: READ_COMMITTED + optimistic locking (`@Version`). 99.9% success on first retry, checkout fast, 0.1% failures handled gracefully. We traded perfect isolation for performance — acceptable for our business (oversell rate dropped from 5% to 0.001%).

**Rule of thumb:** ACID is non-negotiable for financial data. For less critical data, consider eventual consistency. Test isolation under concurrent load — bugs only appear with multiple users.
