## 118. What is the difference between INNER JOIN and LEFT JOIN?

**INNER JOIN returns only matching rows from both tables. LEFT JOIN returns all rows from the left table, with NULLs for non-matching right table rows.**

Choose based on whether you need unmatched rows from the left table.

**Key differences:**
- **INNER JOIN** — only rows with matches in both tables
- **LEFT JOIN** — all left table rows, matched right rows (or NULL)
- **Result size** — INNER JOIN ≤ LEFT JOIN (INNER never returns more)
- **NULL handling** — LEFT JOIN introduces NULLs for non-matches

**Why it matters:**
- INNER JOIN is typically faster — optimizer can filter earlier
- LEFT JOIN preserves data — critical for "find missing" queries
- Wrong choice causes missing rows or unexpected NULLs

**Trade-off:**
- INNER JOIN is more efficient but loses unmatched rows
- LEFT JOIN is necessary for outer queries but can be slower
- LEFT JOIN with WHERE clause on right table converts to INNER JOIN

**Real-world example:**
Our customer report needed "all customers with their orders." Used INNER JOIN — 50K customers became 30K (lost customers without orders). Business complained: inactive customers disappeared from reports. Switched to LEFT JOIN — all 50K customers appeared, NULL for order columns where no orders existed. Query slowed 20% but results were correct. For "find customers without orders" query, used LEFT JOIN with `WHERE order.id IS NULL` — found 20K inactive customers for re-engagement campaign.

**Rule of thumb:** Use INNER JOIN when you only care about matching rows. Use LEFT JOIN when you need all rows from the left table. Never put right-table conditions in WHERE clause with LEFT JOIN — move them to ON clause.
