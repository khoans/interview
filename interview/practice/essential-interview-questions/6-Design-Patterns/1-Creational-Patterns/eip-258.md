## 258. What is the difference between deep copy and shallow copy?

**Shallow copy copies the object but not nested objects — references are shared. Deep copy copies everything — original and copy are completely independent.**

The difference matters when objects contain references to other objects.

**Shallow copy:**
- Copies primitive fields (int, boolean) — independent
- Copies references to objects — shared between original and copy
- Fast — just copy memory
- Risk: modify copy's nested object, original changes too

**Deep copy:**
- Copies everything — primitives and nested objects
- Complete independence — modify copy, original unchanged
- Expensive — recursive copy of entire object graph
- Complexity: circular references cause infinite loops

**Trade-off:**
- Shallow: fast, simple — but shared state causes bugs
- Deep: safe, independent — but slow, complex implementation

**Real-world example:**
Our ShoppingCart clone used shallow copy — cart items list reference copied, not the list contents. User cloned cart to save for later, removed item from clone — item removed from original too. Customer complaint: saved cart lost items. Fixed with deep copy — clone cart items list, clone each CartItem. Zero cart corruption since. But deep copy added 10ms per clone — acceptable for cart operations (user-initiated, not in hot path).

**Rule of thumb:** Deep copy for mutable nested objects — lists, maps, custom objects. Shallow copy acceptable for immutable nested objects (String, Integer) or when shared state is intentional. Document which copy your clone method provides.
