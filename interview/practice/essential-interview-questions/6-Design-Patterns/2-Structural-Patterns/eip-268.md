## 268. What are the types of Proxy pattern?

**Main proxy types: virtual (lazy loading), protection (access control), remote (RPC stubs), caching (memoization).**

Each type solves different access control problems.

**Virtual Proxy:**
- Delays creation until first use
- Example: load high-res image only when user clicks thumbnail
- Benefit: faster startup, less memory

**Protection Proxy:**
- Checks permissions before allowing operation
- Example: admin-only operations check role before delegating
- Benefit: centralized security, single responsibility

**Remote Proxy:**
- Local representative for remote object
- Example: web service client, RPC stub
- Benefit: hides network complexity

**Caching Proxy:**
- Stores results, returns cached if available
- Example: expensive query results cached
- Benefit: faster repeated access

**Trade-off:**
- Each proxy type adds specific behavior — focused, single responsibility
- But: multiple proxy types can stack — debugging which proxy does what
- Performance: protection + caching + virtual = 3 layers of overhead

**Real-world example:**
Our document management system used all four: virtual proxy (load document content on demand), protection proxy (check read/write permissions), remote proxy (documents stored on separate server), caching proxy (cache frequently accessed documents). Document access: secure, efficient, lazy-loaded. But debugging access issues required tracing through 4 proxy layers. Added logging at each proxy level — "ProtectionProxy: user X has read access", "CacheProxy: miss, fetching from remote".

**Rule of thumb:** Use specific proxy type for specific problem. Don't combine all four unless needed — each adds complexity. Log proxy decisions for debugging.
