## 281. What is the Template Method pattern?

**Template Method pattern defines skeleton of algorithm in base class — subclasses override specific steps without changing algorithm structure.**

It's about invariant algorithm structure with variant steps — base class controls flow, subclasses provide details.

**When to use:**
- Algorithm framework — same steps, different implementations per step
- Framework development — base class defines lifecycle, users extend
- Avoid code duplication — common steps in base class

**Trade-off:**
- Reusable algorithm structure — don't repeat framework code
- But: inverted control — base class calls subclass methods (Hollywood principle)
- Inflexible — can't change algorithm order without changing base class

**Real-world example:**
Our report generator had 10 report types — each did: fetch data, filter, sort, format, export. 80% same code, 20% varies (filter, format). Each report: 200 lines, 160 duplicated. Implemented Template Method: ReportGenerator base class with template method generate(). Subclasses override filterData(), formatOutput(). Duplicated code: 160 lines × 10 → 160 lines in base. Adding new report: 40 lines (only overrides). But debugging required understanding base class flow — added comments documenting each step, hook methods for optional customization.

**Rule of thumb:** Template Method for algorithm frameworks with common structure. Not for one-off algorithms — over-engineering. Document hook methods — which are required, which optional.
