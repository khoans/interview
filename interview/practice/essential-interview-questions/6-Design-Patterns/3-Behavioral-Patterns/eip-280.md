## 280. What is the difference between State and Strategy?

**State and Strategy have same structure (context + interchangeable implementations), different intent. State changes behavior based on internal state. Strategy changes behavior based on client choice.**

Same pattern, different purpose — State is automatic, Strategy is manual.

**State:**
- Intent: behavior changes when state changes
- Who chooses: object itself transitions states
- Example: Order state (NEW → PAID → SHIPPED) — order transitions automatically

**Strategy:**
- Intent: behavior varies by algorithm choice
- Who chooses: client sets strategy explicitly
- Example: Shipping strategy (FedEx, UPS, USPS) — client chooses which to use

**Trade-off:**
- State: automatic transitions — object manages its own state
- Strategy: explicit selection — client controls algorithm
- Both: same structure, different semantics

**Real-world example:**
Our document editor had both. State: Document (DRAFT → REVIEW → PUBLISHED) — document transitions based on workflow actions. Strategy: ExportStrategy (PDF, Word, HTML) — user chooses export format. Same structure: DocumentContext + DocumentState/ExportStrategy. Different usage: state transitions automatic (publish() moves DRAFT → PUBLISHED), strategy explicit (setExportStrategy(new PDFExport())). Clear separation prevented confusion.

**Rule of thumb:** State for automatic state-driven behavior. Strategy for client-chosen algorithms. Don't mix — state should be internal, strategy should be external choice.
