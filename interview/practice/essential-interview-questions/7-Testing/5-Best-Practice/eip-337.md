## 337. What is AAA pattern (Arrange-Act-Assert)?

**AAA pattern structures tests in three sections — Arrange (setup), Act (execute), Assert (verify).**

It's about readable, maintainable tests — clear separation of concerns.

**Three sections:**
- **Arrange** — setup test data, mocks, preconditions
- **Act** — execute code under test
- **Assert** — verify outcome, check results

**Trade-off:**
- Readable — clear what's setup, what's tested, what's expected
- Maintainable — easy to find setup, execution, verification
- But: can be verbose — explicit sections add lines
- Rigid — not all tests fit neatly into three sections

**Real-world example:**
Our tests had mixed sections — setup, assert, act, setup more, assert again. Reading test: what's being tested? Refactored to AAA — all setup first, one act, all asserts. Test readability improved — new developers understood tests in 30 seconds vs. 5 minutes. But some tests had multiple acts — tested workflow. Used Given-When-Then pattern for workflows, AAA for simple tests.

**Rule of thumb:** AAA for simple tests — one setup, one action, multiple asserts. Given-When-Then for workflows — multiple steps. Keep sections visually separated — blank lines between Arrange, Act, Assert.
