## 346. How do you handle test data?

**Test data should be isolated, repeatable, and representative — created per test, cleaned after, production-like.**

Good test data catches bugs. Bad data gives false confidence.

**Approaches:**
- **Create per test** — @BeforeEach, fresh data each test
- **Cleanup after** — @AfterEach, delete test data
- **Production-like** — anonymized production data or realistic generators

**Trade-off:**
- Fresh data: isolated tests — no interference
- But: setup overhead — each test creates data
- Shared data: fast setup — but tests interfere

**Real-world example:**
Our tests shared data — one setup for all tests. Test A created user, Test B expected no users. Random failures depending on test order. Fixed: @BeforeEach creates fresh data, @AfterEach cleans up. Tests isolated — order doesn't matter. But setup time: 10ms → 50ms per test. Acceptable — reliable tests worth 40ms overhead. For slow setup (100ms+), used @BeforeAll shared data with careful isolation.

**Rule of thumb:** Fresh data per test — @BeforeEach create, @AfterEach cleanup. Production-like data — anonymized or realistic generators. Shared data only for slow setup — with careful isolation.
