## 343. What is @ValueSource annotation?

**@ValueSource provides simple parameterized test inputs — arrays of primitives, strings, classes.**

It's the simplest parameterized test — inline values, no external data source.

**Usage:**
- @ValueSource(ints = {1, 2, 3}) — multiple int inputs
- @ValueSource(strings = {"a", "b", "c"}) — multiple string inputs
- @ValueSource(booleans = {true, false}) — boolean inputs

**Trade-off:**
- Simple — inline values, no setup
- Readable — values visible in test
- But: limited types — primitives, strings, classes only
- Not for complex objects — can't pass User, Order objects

**Real-world example:**
Our validation tested email formats — 20 valid emails, 20 invalid. @ValueSource(strings = {...}) with 40 values. Test method: 10 lines. Coverage: all formats. But 40 values made test 200 lines — hard to read. Extracted to CSV file — @CsvFileSource. Test method: 5 lines. Data file: 40 rows, readable. But @ValueSource better for simple cases — 3-5 values inline, clear and simple.

**Rule of thumb:** @ValueSource for 3-5 simple values — inline, readable. @CsvSource for 5-20 values — structured inline. @CsvFileSource for 20+ values — external file, maintainable.
