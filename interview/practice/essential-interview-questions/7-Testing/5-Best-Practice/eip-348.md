## 348. What is the difference between @Before and @BeforeEach?

**@Before is JUnit 4 — runs before each test. @BeforeEach is JUnit 5 — same behavior, clearer name.**

Different versions, same concept — setup code before every test.

**JUnit 4:**
- @Before — runs before each test method
- @After — runs after each test method
- @BeforeClass — runs once before all tests
- @AfterClass — runs once after all tests

**JUnit 5:**
- @BeforeEach — runs before each test method
- @AfterEach — runs after each test method
- @BeforeAll — runs once before all tests
- @AfterAll — runs once after all tests

**Trade-off:**
- JUnit 5: clearer names — Each vs. Class scope
- JUnit 5: more features — nested tests, parameterized tests
- But: migration effort — annotations, extensions

**Real-world example:**
Our codebase had JUnit 4 — @Before, @After, @BeforeClass. JUnit 5 released: better extensions, parallel execution, parameterized tests. Migrated: @Before → @BeforeEach, @BeforeClass → @BeforeAll. 500 tests, 2 days migration. Benefits: parallel execution (50% faster), better parameterized tests. But some third-party extensions didn't support JUnit 5 — verified all before migration.

**Rule of thumb:** JUnit 5 for new projects — @BeforeEach, @BeforeAll. Migrate from JUnit 4 — 2 days for 500 tests, worth it for parallel execution. Verify extensions support JUnit 5 before migration.
