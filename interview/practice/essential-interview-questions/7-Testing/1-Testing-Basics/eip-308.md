## 308. What is smoke testing?

**Smoke testing is a minimal test suite to verify basic functionality — "does the application start and respond?"**

It's about failing fast — catch deployment failures before running full test suite.

**When to use:**
- After deployment — verify application is healthy
- Before full test suite — don't waste time on broken builds
- Production health checks — periodic smoke tests

**Trade-off:**
- Fast feedback — 2 minutes vs. 30 minutes for full suite
- But: limited coverage — only critical paths
- False confidence — smoke passes, deeper bugs exist

**Real-world example:**
Our deployment pipeline ran full test suite (30 minutes) before realizing application couldn't start — database connection string wrong. Wasted 30 minutes per failed deployment. Added smoke test: application starts, health endpoint responds, database connection works. 2 minutes. Failed deployments caught immediately. But smoke test passed, checkout broken (logic bug). Smoke test is necessary, not sufficient — still run full suite.

**Rule of thumb:** Smoke test for every deployment — under 5 minutes. Include: application starts, health check passes, critical path works. Not a replacement for full testing — gate before full suite.
