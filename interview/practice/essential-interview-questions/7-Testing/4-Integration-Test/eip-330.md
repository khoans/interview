## 330. What is @WebMvcTest?

**@WebMvcTest is Spring Boot test slice for MVC controllers — loads only web layer, mocks services and repositories.**

It's about testing controllers in isolation — not full application context.

**What it loads:**
- Controllers — @RestController, @Controller
- Web configuration — converters, filters, interceptors
- Mocks — @Service, @Repository auto-mocked

**Trade-off:**
- Fast — loads 10% of full context
- Isolated — controller bugs not hidden by service bugs
- But: limited scope — doesn't test service integration
- Mocking required — services mocked, may not match real behavior

**Real-world example:**
Our integration tests used @SpringBootTest — full context, 30 seconds startup. @WebMvcTest: 3 seconds. Test suite: 30 minutes → 3 minutes. But @WebMvcTest passed, production failed — mock service returned different data than real service. Added contract tests — verify mock behavior matches real service. Now @WebMvcTest for controller logic, integration tests for service integration.

**Rule of thumb:** @WebMvcTest for controller logic — request mapping, validation, response format. @SpringBootTest for full integration — controller + service + database. Use both — fast feedback + confidence.
