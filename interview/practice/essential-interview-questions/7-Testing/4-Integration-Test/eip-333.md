## 333. What is context slice? What is Test Slice?

**Test slice loads only part of application context — @WebMvcTest (web layer), @DataJpaTest (database layer), not full application.**

It's about faster, isolated tests — load 10% of context, test one layer.

**Common slices:**
- @WebMvcTest — controllers, web config, mocks services
- @DataJpaTest — JPA, repositories, in-memory database
- @JsonTest — JSON serialization, mocks everything else
- @RestClientTest — REST client testing, mocks server

**Trade-off:**
- Fast — partial context, 3 seconds vs. 30 seconds
- Isolated — layer-specific bugs visible
- But: integration issues missed — layers work alone, fail together
- Multiple test types — need slice tests + full integration tests

**Real-world example:**
Our tests used @SpringBootTest — full context, 30 seconds per test, 100 tests = 50 minutes. Split into slices: @WebMvcTest (3s), @DataJpaTest (2s), @SpringBootTest (30s, 10 critical paths only). Test suite: 50 minutes → 10 minutes. But slice tests passed, full integration failed — controller expected different data than service provided. Added 5 full integration tests for critical paths. Caught integration issues early.

**Rule of thumb:** Test slices for layer-specific logic — 80% of tests. Full context for critical paths — 20% of tests. Pyramid: many slice tests, few full integration tests.
