## 331. What is @DataJpaTest?

**@DataJpaTest is Spring Boot test slice for JPA — loads only database layer, mocks services.**

It's about testing repositories and database operations — not full application.

**What it loads:**
- JPA entities — @Entity classes
- Repositories — @Repository, Spring Data repositories
- In-memory database — H2 by default, transactional

**Trade-off:**
- Fast — loads only JPA configuration
- Isolated — database bugs not hidden by service bugs
- But: in-memory DB differs from production — H2 vs. PostgreSQL
- Limited scope — doesn't test service integration

**Real-world example:**
Our @DataJpaTest passed — H2 database, all queries worked. Production failed — PostgreSQL specific syntax (JSON extraction), H2 doesn't support. Tests gave false confidence. Switched to TestContainers — real PostgreSQL in Docker for tests. Test time: 2 seconds → 10 seconds per test. But caught PostgreSQL-specific bugs in staging. Zero database-related production failures since.

**Rule of thumb:** @DataJpaTest for repository logic — custom queries, entity mappings. TestContainers for database-specific features — stored procedures, JSON, full-text search. Don't rely on H2 for production PostgreSQL.
