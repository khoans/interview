## 335. How do you test with real data?

**Test with production-like data — anonymized production dumps, synthetic data generators, or carefully curated test datasets.**

Real data catches bugs synthetic data misses — edge cases, data quality issues.

**Approaches:**
- **Production dump** — anonymized real data, most realistic
- **Synthetic generators** — create realistic data programmatically
- **Curated datasets** — hand-crafted edge cases

**Trade-off:**
- Production data: most realistic — catches real-world bugs
- But: privacy concerns — must anonymize PII
- Synthetic: safe, repeatable — but may miss real-world edge cases

**Real-world example:**
Our tests used synthetic data — "John Doe", "test@example.com". Production had special characters — "José García", "test+filter@example.com". Tests passed, production failed (encoding issues, email parsing). Started using anonymized production data — real names, real emails, real edge cases. Caught 10 encoding bugs, 5 email parsing bugs. But data anonymization took 2 days — one-time cost. Privacy-compliant, realistic tests.

**Rule of thumb:** Production data for integration tests — anonymize PII. Synthetic data for unit tests — fast, repeatable. Curated datasets for edge cases — null, empty, special characters.
