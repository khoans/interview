## 332. What is TestContainers?

**TestContainers is a library that runs real dependencies in Docker containers for tests — databases, message queues, external services.**

It's about realistic integration tests — real PostgreSQL, not H2; real Kafka, not embedded.

**When to use:**
- Database testing — real PostgreSQL/MySQL, not in-memory
- External services — Kafka, Redis, Elasticsearch
- Consistent environments — same version as production

**Trade-off:**
- Real dependencies — catches database-specific bugs
- Production parity — same version, same configuration
- But: slower — Docker startup overhead (5-10 seconds)
- Docker required — CI/CD must support Docker

**Real-world example:**
Our tests used H2 — passed, production PostgreSQL failed (JSON queries, case-sensitive comparisons). TestContainers: real PostgreSQL container. Test time: 2s → 10s per test. Caught 5 PostgreSQL-specific bugs before production. But CI didn't have Docker — added Docker-in-Docker. CI time: 10 minutes → 20 minutes. Trade-off accepted — zero database bugs in production vs. 5/month before.

**Rule of thumb:** TestContainers for integration tests — databases, message queues, external services. Not for unit tests — too slow. Cache containers across tests — don't start new container per test.
