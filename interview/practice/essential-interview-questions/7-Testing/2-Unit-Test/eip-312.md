## 312. What is @Test annotation?

**@Test annotation marks a method as a test case — JUnit runs all @Test methods as individual tests.**

It's the basic building block of JUnit tests — every test method has this annotation.

**Key attributes:**
- **timeout** — test fails if exceeds time limit (for performance tests)
- **expectedException** — test passes if specific exception thrown (JUnit 4)
- **displayName** — custom name for test reports (JUnit 5)

**Trade-off:**
- Simple — just annotate method, JUnit handles rest
- But: no parameters — test methods can't accept arguments
- One test per method — don't test multiple things in one @Test

**Real-world example:**
Our team had @Test methods with 20 assertions each — testing entire workflow in one test. First assertion failed, rest skipped. Debugging: which scenario failed? Refactored: one @Test per scenario — testPositiveNumbers, testNegativeNumbers, testZero. Failed test name showed exact scenario. But test duplication — same setup in 10 tests. Extracted setup to @BeforeEach method.

**Rule of thumb:** One @Test per scenario — not per method. Test method name describes scenario — testWithdrawal_InsufficientFunds_ThrowsException. Keep tests under 10 assertions — if more, split into multiple tests.
