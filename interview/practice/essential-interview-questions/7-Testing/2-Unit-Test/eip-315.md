## 315. What is assert vs verify?

**Assert checks conditions — test fails if assertion false. Verify checks mock interactions — test fails if expected interaction didn't happen.**

Different purposes: assert validates state, verify validates behavior.

**Assert:**
- Validates state — assertEquals(expected, actual)
- Did the code produce correct output?
- Example: assert balance == 100 after deposit

**Verify:**
- Validates behavior — verify(mock).method()
- Did the code call the right methods?
- Example: verify(emailService).sendConfirmation(user)

**Trade-off:**
- Assert: tests outcome — more stable, implementation-agnostic
- Verify: tests implementation — fragile, breaks on refactoring
- Overuse verify: tests break when implementation changes, even if behavior correct

**Real-world example:**
Our tests used verify for everything — verify(repository).save(user). Refactored repository to saveAsync — all tests broke, even though behavior correct. Switched to assert — assert user saved (query database). Tests passed after refactoring. But verify useful for side effects — can't assert email sent. Use verify for side effects (send email, publish event), assert for state changes.

**Rule of thumb:** Assert for state changes — database, return values. Verify for side effects — external calls, events. Prefer assert — more stable, less fragile.
