## 313. What is @BeforeEach and @AfterEach?

**@BeforeEach runs before each test method — setup code. @AfterEach runs after each test method — cleanup code.**

They ensure tests are isolated — each test starts fresh, cleans up after.

**When to use:**
- @BeforeEach — create fresh objects, reset mocks, initialize state
- @AfterEach — close resources, delete temp files, reset static state

**Trade-off:**
- Isolation — tests don't interfere with each other
- DRY — setup code in one place, not duplicated
- But: hidden setup — test doesn't show all setup inline
- Overuse — @BeforeEach grows to 50 lines, tests hard to understand

**Real-world example:**
Our tests had duplicated setup — 10 lines in each of 100 tests. Moved to @BeforeEach — 10 lines once. But @BeforeEach grew to 50 lines — create user, create order, create payment, mock services. New developers didn't understand test setup. Refactored: small @BeforeEach (10 lines), helper methods for complex setup (createTestUser(), createTestOrder()). Test readability restored.

**Rule of thumb:** @BeforeEach for common setup — under 10 lines. Use helper methods for complex setup — createTestOrder(), mockPaymentService(). Don't share state between tests — fresh objects each test.
