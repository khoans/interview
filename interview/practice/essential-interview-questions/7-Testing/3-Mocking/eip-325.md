## 325. What is @InjectMocks annotation?

**@InjectMocks annotation creates tested object and injects @Mock dependencies — constructor, setter, or field injection.**

It's about automatic wiring — test class with all mocks injected.

**How it works:**
- Creates instance of annotated class
- Injects @Mock fields into constructor/setter/fields
- Priority: constructor > setter > field injection

**Trade-off:**
- Automatic wiring — no manual new MyClass(mock1, mock2)
- But: field injection hides dependencies — can't see what's injected
- Multiple constructors — Mockito picks one, may not be intended

**Real-world example:**
Our test had manual injection — new UserService(userMock, emailMock, paymentMock). @InjectMocks: one annotation. But class had 3 constructors — Mockito picked wrong one (no-arg), fields null, tests failed. Fixed: single constructor injection, @InjectMocks works correctly. But @InjectMocks hid that UserService had 8 dependencies — code smell. Refactored to 3 smaller services.

**Rule of thumb:** @InjectMocks for simple classes — 1-3 dependencies. Multiple constructors: use single constructor injection. 5+ dependencies: refactor class, not test.
