## 324. What is @Mock annotation?

**@Mock annotation creates mock objects automatically — no need for manual Mockito.mock() calls.**

It's syntactic sugar — cleaner test setup, less boilerplate.

**How it works:**
- @Mock creates mock — Mockito initializes before each test
- @InjectMocks injects mocks into tested class — constructor or field injection
- MockitoExtension — JUnit 5 extension for automatic initialization

**Trade-off:**
- Less boilerplate — annotation vs. manual mock creation
- But: magic — mocks created implicitly, not visible in test code
- Overuse — @Mock on every field, tests mock everything

**Real-world example:**
Our tests had Mockito.mock() in every @BeforeEach — 5 mocks × 50 tests = 250 lines. @Mock annotation: 50 lines total. But @Mock on 20 fields — test mocked entire dependency graph. Hard to understand what's real, what's fake. Limited @Mock to 2-3 external dependencies per test. Internal dependencies: real objects.

**Rule of thumb:** @Mock for external dependencies — 2-3 per test max. @InjectMocks for tested class. Don't @Mock internal dependencies — use real objects.
