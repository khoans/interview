## 323. What is the difference between mock and spy and stub?

**Mock is fake object with no real implementation. Spy is real object with selective mocking. Stub is simplified implementation returning fixed values.**

Different tools for different isolation needs.

**Mock:**
- No real code executed — all methods return default/null
- Use: isolate completely, verify interactions
- Example: mock(UserService.class) — all methods fake

**Spy:**
- Real code executed — unless stubbed
- Use: test most of real class, mock one method
- Example: spy(new UserService()) — real except stubbed methods

**Stub:**
- Simplified real implementation — returns fixed data
- Use: predictable behavior, no verification
- Example: new InMemoryUserRepository() — real code, in-memory

**Trade-off:**
- Mock: complete isolation — but can drift from reality
- Spy: mostly real — but partial mocking can hide bugs
- Stub: real behavior — but maintenance cost for stub implementations

**Real-world example:**
Our service had expensive calculateTax() method. Mock: all methods fake — missed bug in other methods. Spy: real service, stub calculateTax() — caught bug in validateUser(). Stub: InMemoryUserService — real behavior, fast. But stub grew to 200 lines — maintenance burden. Used spy for most tests, stub for integration-like tests.

**Rule of thumb:** Mock for external dependencies. Spy for testing most of class with one expensive method. Stub for simple, stable interfaces.
