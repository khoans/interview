## 322. What is Mockito?

**Mockito is Java's most popular mocking framework — create mocks, stub behaviors, verify interactions with simple API.**

It's the standard for Java unit testing — integrates with JUnit, widely adopted, rich features.

**Key features:**
- Mock creation — mock(Class), @Mock annotation
- Stubbing — when(mock.method()).thenReturn(value)
- Verification — verify(mock).method(), verify(mock, times(3))
- Argument matching — any(), eq(), contains()

**Trade-off:**
- Simple API — one-liner mocks, readable tests
- Ecosystem — integrates with JUnit, Spring
- But: can encourage over-mocking — mock everything, test nothing real
- Silent failures — wrong method name on mock returns null, no error

**Real-world example:**
Our team used manual stubs — 50 lines per stub class. Mockito: one line mock(UserService.class). Stubbing: 10 lines → 1 line. But over-mocked: mocked UserService, UserRepository, EmailService, PaymentGateway. Test passed, integration failed (5 mocks drifted from reality). Limited mocking to external boundaries — EmailService, PaymentGateway. Real classes for internal logic.

**Rule of thumb:** Mockito for external dependencies only. Internal logic: real objects. Verify mock behavior matches real API — contract tests or integration tests.
