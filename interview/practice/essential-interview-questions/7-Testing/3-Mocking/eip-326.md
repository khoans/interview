## 326. What is when().thenReturn()?

**when().thenReturn() is Mockito's stubbing syntax — define what happens when mock method is called.**

It's the core of mock behavior — arrange mock responses before test execution.

**Syntax:**
- when(mock.method()).thenReturn(value) — return value when method called
- when(mock.method(arg)).thenReturn(value) — return for specific argument
- when(mock.method()).thenThrow(exception) — throw exception when called

**Trade-off:**
- Readable — English-like syntax, clear intent
- But: brittle — change method signature, stubbing breaks silently
- Over-stubbing — stub every method, mock becomes complex

**Real-world example:**
Our test stubbed 10 methods — when(mock.getUser()).thenReturn(user), when(mock.getOrder()).thenReturn(order)... Test failed: method renamed, stubbing didn't update (returns null). Debugging: 2 hours. Added strict stubbing — Mockito fails test if stubbed method not called. Caught unused stubbing immediately. But strict mode too harsh — removed stubbing for optional calls.

**Rule of thumb:** Stub only methods used in test. Use strict stubbing — catch unused stubbing. Don't stub everything — mock should be simple.
