# Responsibilities

I am not responsible for the entire system, but I was heavily participating in several key areas:

- Decoupling the Spring Framework monolith into microservices based on Java technologies

- Search Engine Optimization (SEO) and AI Search Optimization (AIO)

- Natural Language Search using AWS Bedrock

- Development of an Analytics Engine to collect user information and behavior

- Integration with third-party SEO services to reduce the burden of maintaining metadata for a large number of webpages, which previously required significant developer resources

- Heavy involvement in developing a new UI library with accessibility features to support people with visual impairments in the US market

- Optimizing Web Vital scores of the customer-facing sites on both frontend and backend

Besides these key initiatives, I also maintain the current system, including the search services written in Spring Boot and the custom Node.js framework for Next.js on the client side. Maintenance includes developing new APIs, performance optimization, and troubleshooting production issues.

On the backend, I implemented features following the client's requirements, utilizing DynamoDB and PostgreSQL as database management systems. We used Java frameworks including Spring Boot, Spring Data, Spring Security, and OpenAPI to implement features and provide APIs. Since the project is heavily integrated with AWS, many of our components and services use serverless platforms such as AWS Lambda, AWS SQS, and AWS SNS.

I also participated in daily stand-up meetings, consistently providing valuable updates and insights such as identifying critical bugs in the backend code, raising problems that impact the whole flow, suggesting solutions, and asking for help. Additionally, I actively contributed to code reviews by thoroughly examining the codebase, identifying potential issues, applying design patterns, and cleaning code to ensure the delivery of high-quality, maintainable code throughout the development process.
